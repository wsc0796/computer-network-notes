### 90分红线

出现以下任一错误，不能认定为90分水平：SYN/FIN 不占序号；把 ACK 理解成已经收到的最后一个字节；rwnd 和 cwnd 写反；UDP 数据长度未减 8；CIDR 网络地址或广播地址越界；VLSM 地址块未按边界对齐；最长前缀选择较短路由；RIP 距离未先加 1；RIP 同一下一跳变差时没有更新；IP 分片用总长度计算偏移；片偏移没有除以 8；最后一片 MF 写成 1。

# 一、网络层知识与训练

## 1. 网络层非大题知识链

| 知识点 | 一句话结论 | 标准简答 | 易错辨析 | 立即自测 |
|---|---|---|---|---|
| 网络层任务 | 跨网络主机到主机交付 | 网络层使用 IP 地址和路由协议，在多个网络之间为分组选择路径并转发到下一跳。 | 不是进程到进程；端口是运输层 | 1. 网络层PDU？2. 网络层地址？ |
| 转发与路由 | 转发查表，路由算表 | 转发是路由器对每个分组查转发表并送出端口；路由是全网运行协议生成路由表。 | 不要把二者写成同义词 | 1. 哪个每包发生？2. RIP属于哪类？ |
| 数据报与虚电路 | IP是数据报思想 | 数据报无连接，每个分组携带目的地址；虚电路先建连，分组携带VC号并沿固定路径。 | 数据报路径可不同 | 1. IP属于？2. 虚电路是否先建连？ |

## 2. 网络层大题模板

### 2.1 普通CIDR计算

题型信号：给 `192.168.10.77/26`，求网络地址、广播地址、可用范围。公式：块大小 `2^(32-前缀)`；最后一段步长为 64。例题：`192.168.10.77/26` 位于 64-127，网络 `192.168.10.64`，广播 `192.168.10.127`，可用 `.65-.126`。练习 N-C1：`192.168.5.130/27`。变式 N-C2：`10.1.2.199/28`。红线：地址段边界不能凭感觉四舍五入。

### 2.2 跨字节CIDR

步骤：确定前缀结束在哪个字节；该字节掩码值；块大小 `256-掩码值`；网络边界 `floor(地址字节/块大小)×块大小`；广播边界 `网络边界+块大小-1`。例题：`172.16.88.150/18`，/18 在第三字节，掩码 192，块大小 64，88 落在 64-127，网络 `172.16.64.0`，广播 `172.16.127.255`。练习 N-X1：`10.20.37.100/20`。变式 N-X2：`172.30.201.9/21`。

### 2.3 等长子网划分

步骤：子网数 n，借 k 位使 `2^k>=n`；新前缀=原前缀+k；按步长列网络。例题：`192.168.1.0/24` 划 4 个子网，新前缀 /26，网络 `.0,.64,.128,.192`。练习 N-S1：`192.168.8.0/24` 划 8 个子网。变式 N-S2：`10.0.0.0/16` 划 16 个等长子网。

### 2.4 VLSM

步骤：需求从大到小排序；对每个主机数 h 找最小 m 使 `2^m-2>=h`；前缀 `32-m`；从低地址按块边界分配。例题：`192.168.10.0/24` 给 60、30、12 主机：60 -> /26 `.0-.63`；30 -> /27 `.64-.95`；12 -> /28 `.96-.111`。练习 N-V1：80、40、20。变式 N-V2：要求给路由器接口也预留地址。

### 2.5 路由聚合

步骤：写二进制找共同前缀，后面补0，检查是否引入额外网络。例题：`192.168.0.0/24` 到 `192.168.3.0/24` 可聚合为 `192.168.0.0/22`。练习 N-A1：`10.0.4.0/24` 到 `10.0.7.0/24`。变式 N-A2：`192.168.0.0/24` 与 `192.168.2.0/24` 能否精确聚合？

### 2.6 最长前缀匹配

模板：列出所有匹配项，选 `/x` 最大。例题：目的 `192.168.1.130`，路由 `/16`、`192.168.1.0/24`、`192.168.1.128/25`，选 `/25`。练习 N-L1：目的 `10.1.5.200` 匹配 `/8,/16,/24,/25`。变式 N-L2：有默认路由和黑洞路由时先按最长前缀。

### 2.7 RIP路由表更新

模板：邻居通告距离先 +1；新网络添加；更短更新；同一下一跳强制更新。例题：本表到 N1 距离 5 下一跳 R2；R2 通告 N1 距离 7，新距离 8，同一下一跳，更新为 8。练习 N-R1：R3 通告新网络 N4 距离2。变式 N-R2：不同下一跳距离变差不更新。

### 2.8 IPv4分片

模板：数据长度=总长-首部；每片最大数据=MTU-首部；非末片取8的倍数；偏移=起始数据字节/8；末片MF=0。例题：总长4000，首部20，MTU1500。数据3980，每片最大1480；片1数据1480偏移0 MF1；片2数据1480偏移185 MF1；片3数据1020偏移370 MF0。练习 N-F1：总长3020，MTU1000。变式 N-F2：末片非整齐长度。

### 2.9 跨网ARP

例题：A `192.168.1.10/24` 访问 B `10.0.0.5`，默认网关 `192.168.1.1`。A判断不同网段，ARP查询网关MAC，把目的IP仍写B，目的MAC写网关MAC。练习 N-ARP1：同网通信。变式 N-ARP2：ARP缓存已有网关但无目标主机。

### 2.10 NAT转换表

例题：内网 `192.168.1.10:5000` 访问 `8.8.8.8:53`，NAT公网 `203.0.113.9` 分配端口 `40001`，表项为 `192.168.1.10:5000 <-> 203.0.113.9:40001`。返回包目的 `203.0.113.9:40001`，NAT查表改回 `192.168.1.10:5000`。练习 N-NAT1：两个内网主机同访80端口。变式 N-NAT2：返回端口不存在表项则丢弃。

### 2.11 路由器完整转发流程

标准流程：检查IP首部 -> TTL减1 -> TTL为0则丢弃并产生ICMP -> 重算首部校验和 -> 最长前缀匹配 -> 确定下一跳 -> ARP获得下一跳MAC -> 重新封装链路层帧。红线：转发时源/目的IP通常不变，源/目的MAC每跳变化。

# 二、运输层知识与训练

## 3. 运输层非大题知识链

### 3.1 作用、端口、复用分用

运输层实现进程到进程通信。复用：多个应用进程的数据交给运输层发送；分用：运输层根据端口/四元组交给正确进程。TCP连接由源IP、源端口、目的IP、目的端口四元组标识；UDP通常按目的端口分用。

### 3.2 TCP与UDP

| 对比 | TCP | UDP |
|---|---|---|
| 连接 | 面向连接 | 无连接 |
| 可靠性 | 确认、重传、有序 | 尽最大努力 |
| 数据边界 | 字节流 | 面向报文 |
| 控制 | 流量和拥塞控制 | 无 |
| 首部 | 20-60B | 8B |

### 3.3 UDP首部

源端口16、目的端口16、长度16、校验和16。长度字段=UDP首部+数据，数据长度必须减8。例：`04 00 00 35 00 18 AB CD`：源1024，目的53，长度24，数据16B。

### 3.4 TCP首部速查表

| 字段 | 作用 | 常见考法 |
|---|---|---|
| 源端口、目的端口 | 标识进程 | 端口与分用 |
| seq | 当前段首字节序号 | 序号计算 |
| ack | 下一个期望字节 | 确认号计算 |
| 首部长度 | TCP首部长度 | 最小20字节 |
| ACK | 确认号有效 | 握手与数据传输 |
| SYN | 建立连接 | 占1个序号 |
| FIN | 释放连接 | 占1个序号 |
| RST | 复位连接 | 判断题 |
| window | 接收窗口 | 流量控制 |
| checksum | 差错检测 | 首部字段题 |

### 3.5 RDT演进

| 版本 | 信道问题 | 新机制 |
|---|---|---|
| RDT1.0 | 可靠信道 | 无需额外机制 |
| RDT2.0 | 数据可能出错 | ACK/NAK |
| RDT2.1 | ACK/NAK也可能出错 | 序号 |
| RDT2.2 | 不用NAK | 重复ACK |
| RDT3.0 | 数据还可能丢失 | 定时器、超时重传 |

顺序判断：先处理差错，再处理 ACK/NAK 出错，最后处理丢失。

### 3.6 Karn、SACK、零窗口

Karn：不使用重传报文段的RTT样本更新SRTT。SACK：接收方告知已收到的非连续数据块。零窗口：rwnd=0时发送方启动持续计时器，发送零窗口探测，避免死锁。

### 3.7 握手挥手与TIME_WAIT

三握：SYN(x) -> SYN+ACK(y, x+1) -> ACK(y+1)。四挥：FIN(u) -> ACK(u+1) -> FIN(v) -> ACK(v+1)。TIME_WAIT等待2MSL，保证最后ACK可重传并让旧报文消失。

### 3.8 RTT/RTO统一口径

考试首先按题目公式。题目未说明时，本文统一采用：使用旧SRTT计算偏差；更新RTTVAR；更新SRTT；`RTO=SRTT+4×RTTVAR`。

### 3.9 拥塞控制课程口径

若题目说明TCP Reno按Reno；未说明采用课程简化模型：超时 -> `ssthresh=当前cwnd/2, cwnd=1 MSS`；3个重复ACK -> `ssthresh=当前cwnd/2, cwnd降至阈值附近，随后拥塞避免`。同一道题不得混用模型。

## 4. 运输层大题模板

### 4.1 TCP普通seq/ack

铁律：SYN/FIN各占1；数据按字节占序号；纯ACK不占序号；ACK=下一个期望字节。例：seq=1000发800B，ACK=1800。练习 T-S1：seq=500发1200B。变式 T-S2：含纯ACK不推进序号。

### 4.2 SYN、数据、FIN连续

例：C发SYN seq=100；S回SYN seq=800 ack=101；C发ACK并从seq=101发300B，则S确认ack=401；C再发FIN seq=401，S回ack=402。练习 T-C1：x=3000,y=9000,数据700B。变式 T-C2：服务端也发送200B后FIN。

### 4.3 握手挥手填图

填图顺序：SYN、SYN+ACK、ACK；FIN、ACK、FIN、ACK。标注CLOSED、SYN-SENT、SYN-RCVD、ESTABLISHED、FIN-WAIT、CLOSE-WAIT、LAST-ACK、TIME-WAIT。

### 4.4 RTT/RTO

例：旧SRTT=100ms，RTTVAR=20ms，Sample=140ms，α=1/8，β=1/4。偏差=40；新RTTVAR=0.75×20+0.25×40=25；新SRTT=0.875×100+0.125×140=105；RTO=105+4×25=205ms。

### 4.5 停止等待与利用率

```text
T发送 = L/R
U = T发送 / (T发送 + RTT)  （忽略ACK发送和处理）
若 a=传播时延/发送时延，U=1/(1+2a)
```

例：L=1000B，R=1Mbps，RTT=40ms。发送时延8ms，U=8/(8+40)=16.7%。含ACK发送时间时，分母加ACK发送时间。

### 4.6 GBN丢帧

例：发送0-4，2丢失。GBN接收0、1后ACK1；3、4乱序丢弃并重复ACK1；超时后重传2、3、4。

### 4.7 SR乱序

同题SR：0、1确认；3、4缓存并分别ACK3、ACK4；只重传2；2到达后交付2、3、4。

### 4.8 流量控制与零窗口

发送窗口=`min(rwnd,cwnd)`。rwnd为0后发送方不能持续发送正常数据，需持续计时器发送探测。练习：rwnd=0,cwnd=10，发送窗口为0。

### 4.9 拥塞控制

例：ssthresh=16，cwnd从1开始：1,2,4,8,16后拥塞避免17,18。若cwnd=20超时，则ssthresh=10，cwnd=1；若3重复ACK，则ssthresh=10，cwnd降至阈值附近并拥塞避免。

### 4.10 UDP十六进制解析

按2字节一组转十进制。例：`13 88 00 35 00 20 12 34`：源5000，目的53，长度32，数据24B，校验和0x1234。

# 四、核心大题训练

N6 最长前缀：目的 `10.1.5.200`，路由 `10.0.0.0/8->A`，`10.1.0.0/16->B`，`10.1.5.0/24->C`，`10.1.5.128/25->D`，选谁？

## 运输层8题

T1 普通seq/ack：seq=500发1200B，对方ACK？
T2 SYN+数据+FIN：C SYN seq=3000，S SYN seq=9000；C建立后发700B再FIN，求相关ACK与FIN序号。
T3 RTT/RTO：旧SRTT=80ms，RTTVAR=10ms，Sample=120ms，按本文口径求新RTTVAR、新SRTT、RTO。
