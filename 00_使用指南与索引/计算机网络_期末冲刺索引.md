
用途：

```text
补强三次握手、四次挥手状态图、拓扑分析、滑动窗口、流量控制、拥塞控制计算、HTTP/HTTPS区别。

```

必会：

```text
三次握手：SYN -> SYN+ACK -> ACK
四次挥手：FIN -> ACK -> FIN -> ACK
发送窗口 = min(rwnd, cwnd)
慢开始指数增长，拥塞避免线性增长
HTTP明文80端口，HTTPS=HTTP+TLS，443端口
端口号16位；TCP面向字节流，UDP面向报文；停止等待信道利用率低
```

# 7. 第七优先级：薄弱点专项补强 + 配套练习（新增）

文件：

```text
01_考前冲刺复习/计算机网络_薄弱点专项补强（4）.md       ← 先读这个（讲解/速背）
01_考前冲刺复习/计算机网络_薄弱点配套练习（5）.md       ← 再闭卷做（每题附详细解析）
```

用途：

```text
补强现有资料覆盖不到的四个薄弱区 + 配套练习巩固：
1. 物理层编码识图（NRZ/曼彻斯特等）+ 奈奎斯特/香农计算
2. GBN与SR协议详解 + 停止等待对比
3. 应用层协议细节（DNS递归/迭代、FTP双连接、SMTP/POP3/IMAP、MIME/base64）
4. UDP/TCP首部十六进制解析实战
5. IPv6基础（地址格式、分片行为差异）

练习题覆盖：编码识图×3、奈奎斯特/香农×4、GBN/SR×5、UDP解析×3、base64×2、10Base×2、IPv6×2、综合自测×5
```

必会：

```text
四种编码看图辨类型：NRZ、NRZ-I、曼彻斯特、差分曼彻斯特
10Base命名：5=粗缆500m, 2=细缆185m, T=双绞线
奈奎斯特（无噪声）：C=2W·log₂(V)
香农（有噪声）：C=W·log₂(1+S/N)，30dB对应S/N=1000
GBN：收方窗口=1，乱序丢弃，累计确认，超时回退N帧
SR：收方缓存乱序，逐一确认，只重传丢失帧
FTP：双连接（控制21+数据20），主动PORT/被动PASV
SMTP发邮件25，POP3收邮件110，IMAP在线管理143
MIME/base64：3字节→4字符，膨胀约33%
UDP首部8字节解析：十六进制→源端口+目的端口+长度+校验和
IPv6：128位，路由器不分片，MTU过大就丢弃+ICMPv6报告
```

# 9. 80+最低完成线

```text
1. 平时作业高频20条能闭卷默写关键词。
2. 模拟卷3套至少做完选择、判断、填空，并订正错题。
3. 大题至少能独立完成：CRC、子网、RIP更新、IP分片、拥塞控制各1题。
4. TCP/HTTP专题能写出：三次握手、四次挥手、发送窗口、拥塞控制、HTTP/HTTPS区别。
5. 薄弱点配套练习至少做完综合自测 + UDP解析 + GBN/SR + 奈奎斯特/香农。
6. 错题要复做，不只是看解析；复做正确才算掌握。
```

如果时间不够，优先顺序是：

```text
平时作业高频 > 大题专项 > TCP/HTTP专题 > 模拟卷客观题 > 薄弱点综合自测
```
