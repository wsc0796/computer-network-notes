### 流量控制与可靠传输关系（修订版）

```
发送控制与流水线机制（解决"一次能发多少"）：
├─ 停止等待：发一个→等ACK→发下一个
│     出现位置：数据链路层基础、运输层RDT教学
└─ 滑动窗口：窗口范围内连续发送
      出现位置：TCP滑动窗口、数据链路层窗口协议

可靠传输协议（在发送控制基础上+序号+确认+定时器+重传）：
├─ 停止等待ARQ：W_s=1, W_r=1
│     出现位置：数据链路层§停止等待ARQ, RDT 3.0
├─ 单帧滑动窗口协议：W_s=1, W_r=1（窗口化表达的停止等待）
│     出现位置：数据链路层
├─ GBN（多帧滑动窗口）：W_s>1, W_r=1，累计确认，乱序丢弃
│     出现位置：**数据链路层**§GBN（薄弱点补强明确标注）
└─ SR（多帧滑动窗口）：W_s>1, W_r>1，独立确认，缓存乱序
      出现位置：**数据链路层**§SR

TCP流量控制（解决"接收方能不能吃下"）：
├─ rwnd：接收方通告窗口
├─ 零窗口 + 持续计时器
└─ 与拥塞控制（cwnd）共同决定：发送窗口 = min(rwnd, cwnd)

注意：
- TCP的"滑动窗口"是流量控制机制（以字节为单位，rwnd驱动）
- 数据链路层的"滑动窗口协议"是可靠传输机制（以帧为单位，序号+确认驱动）
- 两者原理相通但属于不同层，不要混为一谈
- GBN/SR的可靠传输原理对理解TCP可靠传输（累计确认、选择确认SACK）有帮助
  但GBN/SR本身属于数据链路层，不应在运输层验收中占过高权重
```
