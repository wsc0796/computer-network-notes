## 二、流量控制与可靠传输的完整关系

### 2.1 分类体系

```
流量控制方法
├─ 1. 停止—等待
│     原理：发送方每发一个分组就停下来等ACK，收到ACK后再发下一个
│     特点：简单、效率低、信道利用率低（RTT越大越明显）
│     出现位置：运输层RDT教学、数据链路层ARQ基础
│
└─ 2. 滑动窗口
      原理：发送方可在窗口范围内连续发送多个分组，不必等一个ACK再发下一个
      特点：提高信道利用率、支持流水线传输
      出现位置：TCP滑动窗口、数据链路层滑动窗口协议

可靠传输协议（在流量控制基础上加入序号、确认、定时器、重传）
├─ 1. 停止—等待ARQ
│     = 停止等待流量控制 + 序号 + 确认 + 超时重传
│     发送窗口 = 1，接收窗口 = 1
│     出现位置：RDT 3.0、数据链路层停止等待ARQ
│
├─ 2. 单帧滑动窗口协议
│     发送窗口 = 1，接收窗口 = 1
│     可视为停止等待协议的窗口化表达
│     出现位置：数据链路层
│
└─ 3. 多帧滑动窗口协议
   ├─ 3.1 GBN（后退N帧 / Go-Back-N）
   │     发送窗口 > 1，接收窗口 = 1（通常）
   │     方式：累计确认、乱序丢弃、超时从丢失处回退重传所有已发未确认帧
   │     出现位置：数据链路层§GBN，运输层RDT教学中作为对比
   │
   └─ 3.2 SR（选择重传 / Selective Repeat）
         发送窗口 > 1，接收窗口 > 1
         方式：独立确认、缓存乱序帧、只重传丢失帧
         出现位置：数据链路层§SR，运输层SACK机制的底层思想
```

### 2.3 数据链路层与运输层的边界说明

```text
1. 停止等待既可以描述一种流量控制方式，也可以在加入确认、
   序号、定时器和重传后形成可靠传输协议。

2. 单帧滑动窗口的发送窗口和接收窗口通常为1，
   可视为停止等待协议的窗口化表达。

3. GBN和SR属于多帧滑动窗口协议。

4. 这些内容在很多教材中主要放在数据链路层（如谢希仁教材第3章），
   但其可靠传输思想也会在运输层RDT/TCP学习中出现。

5. 仓库中GBN/SR最详细的讲解在：
   - 01_考前冲刺复习/计算机网络_薄弱点专项补强（4）.md §2
   - 该文件明确标注为"数据链路层：GBN与SR协议"
   
6. 运输层相关文件中GBN/SR出现在：
   - 07_考研预备_分层学习包/02_运输层/03_原资料包对照补强.md（引用薄弱点文件）
   - 07_考研预备_分层学习包/02_运输层/04_题型训练.md（§4 GBN与SR题，作为运输层题型训练）
   - 01_考前冲刺复习/计算机网络_大题补充_时延CSMACD_TCP序号_GBN窗口_交换机.md（GBN窗口题）

7. 当前6小时任务是运输层和网络层复习，因此只学习GBN/SR的原理联系；
   数据链路层的帧级窗口移动、帧编号和链路级ARQ题在数据链路层单独复习时再深入学习。
```
