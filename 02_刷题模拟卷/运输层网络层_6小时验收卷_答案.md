### 2. 停止等待与滑动窗口（6分）

（1）停止等待（2分）

**答案**：发送方每发送一个分组后必须等待接收方 ACK，收到 ACK 后才能发送下一个分组。主要缺点是信道利用率低，RTT 越大等待浪费越明显。

**评分点**：等待 ACK 1分；信道利用率低 1分。

（2）GBN 与 SR（2分）

**答案**：

| 协议 | 接收窗口 | 乱序帧处理 |
|---|---|---|
| GBN | 通常为 1 | 丢弃乱序帧，重复确认最后按序收到的帧 |
| SR | 大于 1 | 缓存乱序帧，分别确认已收到的帧 |

**评分点**：GBN 1分；SR 1分。

（3）计时器（2分）

**答案**：B。GBN 为最早未确认的分组维护计时器，超时后从该分组开始回退重传。

**常见错误**：把 SR 的每帧独立计时器误写成 GBN。

### 3. TCP seq/ack 计算（10分）

（1）数据段确认号（3分）

**公式**：

```text
ack = seq + 数据长度
```

**代入**：

```text
ack = 1200 + 350 = 1550
```

**单位**：TCP 序号按字节编号。
**最终答案**：`ack=1550`。

**反向验证**：收到的最后一个字节序号为 `1200+350-1=1549`，所以下一个期望字节是 `1550`。
**常见错误**：写成 `1549`，忘记 ACK 表示下一个期望字节。

（2）下一个数据报文段 seq（2分）

**公式**：

```text
下一个 seq = 上一个 seq + 数据长度
```

**代入**：

```text
1200 + 350 = 1550
```

**最终答案**：`seq=1550`。
**反向验证**：与上题 ACK 相同，说明连续发送无空洞。

（3）SYN 不携带数据（3分）

**规则**：SYN 占用 1 个序号，即使不携带数据。

**代入**：

```text
B 回复 ack = 4000 + 1 = 4001
A 后续第一个数据报文段 seq = 4001
```

**最终答案**：`ack=4001`，后续数据段 `seq=4001`。
**反向验证**：SYN 使用了序号 4000，所以下一个可用序号是 4001。
**常见错误**：认为 SYN 没有数据所以不占序号。

（4）FIN 不携带数据（2分）

**规则**：FIN 占用 1 个序号，即使不携带数据。

**代入**：

```text
ack = 8000 + 1 = 8001
```

**最终答案**：`ack=8001`。
**反向验证**：FIN 使用了序号 8000，下一个期望序号是 8001。

### 4. RTT/RTTVAR/RTO 计算（8分）

已知：

```text
旧 SRTT = 60 ms
旧 RTTVAR = 20 ms
R = 100 ms
alpha = 1/8
beta = 1/4
K = 4
```

本题按 RFC 6298 顺序计算：**先使用旧 SRTT 计算 RTTVAR，再更新 SRTT，最后计算 RTO**。本题忽略时钟粒度 `G` 和 RTO 最小值限制。

（1）新的 RTTVAR（3分）

**公式**：

```text
RTTVARnew = (1 - beta) * RTTVARold + beta * |SRTTold - R|
```

**代入**：

```text
RTTVARnew = (1 - 1/4) * 20 + (1/4) * |60 - 100|
           = 0.75 * 20 + 0.25 * 40
           = 15 + 10
           = 25 ms
```

**最终答案**：`RTTVARnew=25 ms`。
**反向验证**：新样本比旧 SRTT 大 40ms，波动量从 20ms 增大到 25ms，方向合理。
**常见错误**：先用新 SRTT=65ms 计算 RTTVAR，这是本题要求排除的错误口径。

（2）新的 SRTT（3分）

**公式**：

```text
SRTTnew = (1 - alpha) * SRTTold + alpha * R
```

**代入**：

```text
SRTTnew = (1 - 1/8) * 60 + (1/8) * 100
        = 0.875 * 60 + 0.125 * 100
        = 52.5 + 12.5
        = 65 ms
```

**最终答案**：`SRTTnew=65 ms`。
**反向验证**：新样本 100ms 大于旧均值 60ms，因此新 SRTT 上升到 65ms，方向合理。

（3）RTO（2分）

**公式**：

```text
RTO = SRTT + 4 * RTTVAR
```

**代入**：

```text
RTO = 65 + 4 * 25 = 165 ms
```

**最终答案**：`RTO=165 ms`。
**反向验证**：`165ms > 65ms`，RTO 高于平均 RTT，能覆盖波动。
**常见错误**：把 4 倍 RTTVAR 漏掉，写成 90ms。

### 5. 流量控制与零窗口（8分）

（1）发送窗口（3分）

**公式**：

```text
发送窗口 = min(rwnd, cwnd)
```

**答案**：`rwnd` 是接收方通告窗口，反映接收方还能接收多少数据；`cwnd` 是拥塞窗口，反映网络拥塞控制允许注入多少数据。

**评分点**：公式 1分；rwnd 解释 1分；cwnd 解释 1分。

（2）零窗口（5分）

**答案**：零窗口表示接收方缓存已满，通告 `rwnd=0`，发送方暂时不能继续发送普通数据。发送方启动持续计时器；计时器到期后发送零窗口探测报文，接收方回复当前窗口大小，防止窗口更新报文丢失导致双方永久等待。

**反向验证**：如果没有持续计时器，接收方窗口恢复的通知一旦丢失，发送方会一直以为窗口为 0。
**常见错误**：把零窗口理解为 TCP 连接已经断开。

### 6. 拥塞控制（6分）

（1）当前阶段（2分）

**判断**：

```text
cwnd = 24 MSS
ssthresh = 16 MSS
cwnd > ssthresh
```

**最终答案**：处于拥塞避免阶段。
**常见错误**：只看到 cwnd 较大就写慢开始；慢开始看的是与 ssthresh 的关系。

（2）超时重传（2分）

**公式与代入**：

```text
ssthresh = cwnd / 2 = 24 / 2 = 12 MSS
cwnd = 1 MSS
```

**最终答案**：`ssthresh=12 MSS`，`cwnd=1 MSS`，重新进入慢开始。
**反向验证**：超时被视为较重拥塞，因此窗口退回 1 MSS。

（3）3 个重复 ACK（2分）

**课程简化模型**：

```text
ssthresh = cwnd / 2 = 12 MSS
cwnd = ssthresh = 12 MSS
```

**最终答案**：`ssthresh=12 MSS`，`cwnd=12 MSS`，进入拥塞避免。
**补充**：若按 TCP Reno，快恢复入口可为 `cwnd=ssthresh+3*SMSS`。考试以教师口径为准。

### 7. TCP连接管理（6分）

（1）三次握手（4分）

**答案**：三次握手用于同步双方初始序号，确认双方收发能力，并避免旧的重复 SYN 造成错误连接。服务器收到 SYN 后会进入 SYN-RECEIVED 状态并维护半连接状态；第三次 ACK 证明客户端确实收到了服务器的 SYN 和初始序号，使双方完成初始序号同步并正式进入 ESTABLISHED 状态。

**评分点**：同步初始序号 1分；确认双方收发能力 1分；避免旧 SYN 1分；准确说明 SYN-RECEIVED 到 ESTABLISHED 1分。
**常见错误**：写成“服务器收到最终 ACK 后才分配资源”。这不准确，服务器收到 SYN 后已经维护半连接状态。

（2）TIME-WAIT（2分）

**答案**：主动关闭方发送最后一个 ACK 后进入 TIME-WAIT 状态，等待 2MSL。

**常见错误**：把 TIME-WAIT 写成被动关闭方状态。

### 8. CIDR 计算（8分）

已知 `172.16.88.150/18`。

（1）子网掩码（2分）

**公式/代入**：

```text
/18 = 11111111.11111111.11000000.00000000
```

**最终答案**：`255.255.192.0`。

（2）网络地址（2分）

**公式**：

```text
网络地址 = IP & 子网掩码
```

**代入**：

```text
第三字节 88 落在 /18 的 64-127 段
网络地址 = 172.16.64.0
```

**最终答案**：`172.16.64.0/18`。

（3）广播地址（2分）

**代入**：

```text
/18 地址块范围：172.16.64.0 - 172.16.127.255
```

**最终答案**：`172.16.127.255`。

（4）可用主机数（2分）

**公式/代入**：

```text
主机位 = 32 - 18 = 14
可用主机数 = 2^14 - 2 = 16382
```

**最终答案**：`16382`。
**反向验证**：地址总数为 16384，去掉网络地址和广播地址后为 16382。
**常见错误**：把第三字节块大小误算成 128，而 `/18` 第三字节块大小是 64。

### 11. RIP 路由更新（8分）

**规则**：

```text
经邻居的新距离 = 邻居报告距离 + 1
同一下一跳：无论变好变差都更新
不同下一跳：只有更优才更新
```

**计算过程**：

- N1：当前下一跳是 R1，R1 报告距离 3，经 R1 新距离 `3+1=4`，同一下一跳，更新。
- N2：当前经 R2 距离 5，若经 R1 新距离 `3+1=4`，更优，更新为 R1。
- N3：R1 未报告，保持直连距离 1。
- N4：新网络，经 R1 新距离 `1+1=2`，加入。

**最终答案**：

| 目的网络 | 距离 | 下一跳 | 理由 |
|---|---:|---|---|
| N1 | 4 | R1 | 同一下一跳，按 R1 新通告更新 |
| N2 | 4 | R1 | 经 R1 距离 4 小于原距离 5 |
| N3 | 1 | 直连 | 未收到更新，保持 |
| N4 | 2 | R1 | 新网络，1+1=2 |

**反向验证**：所有经 R1 的新距离都比 R1 报告值多 1。
**常见错误**：忘记加 1；N1 因变差而不更新。

### 12. IP 数据报分片（10分）

（1）原始数据长度与每片最大数据长度（2分）

**公式/代入**：

```text
原始数据长度 = 总长度 - IP首部 = 5000 - 20 = 4980 字节
每片最大数据长度 = floor((MTU - IP首部)/8) * 8
                 = floor((1300 - 20)/8) * 8
                 = floor(1280/8) * 8
                 = 1280 字节
```

**最终答案**：原始数据 `4980` 字节；每片最大数据 `1280` 字节。

（2）分片数（2分）

**代入**：

```text
4980 = 1280*3 + 1140
```

**最终答案**：4片。

（3）分片表（6分）

| 分片 | 数据长度(字节) | MF | 片偏移 |
|---|---:|---:|---:|
| 第1片 | 1280 | 1 | 0 |
| 第2片 | 1280 | 1 | 160 |
| 第3片 | 1280 | 1 | 320 |
| 第4片 | 1140 | 0 | 480 |

**偏移计算**：

```text
第2片偏移 = 1280/8 = 160
第3片偏移 = 2560/8 = 320
第4片偏移 = 3840/8 = 480
```

**反向验证**：

```text
1280 + 1280 + 1280 + 1140 = 4980 = 5000 - 20
```

**常见错误**：片偏移写成字节数；最后一片 MF 写成 1；把 IP 首部算进数据长度。

### 13. ARP 与 NAT（6分）

（1）ARP 跨网（3分）

**答案**：目的 MAC 地址是默认网关 `192.168.1.1` 对应接口的 MAC 地址，不是服务器 `8.8.8.8` 的 MAC。

**原因**：ARP 只在本局域网内解析 MAC。A 发现目的 IP 不在 `192.168.1.0/24` 内，因此把帧交给默认网关。IP 目的地址仍是 `8.8.8.8`，但链路层目的 MAC 是下一跳网关 MAC。

**常见错误**：认为主机会 ARP 查询远程服务器的 MAC。

（2）NAT 作用（3分）

**答案**：

1. 将内网私有地址和端口转换为公网地址和端口，使内网主机访问 Internet。
2. 多台内网主机共享少量公网 IPv4 地址，缓解 IPv4 地址不足。

**常见错误**：把 NAT 写成路由协议；NAT 是地址/端口转换机制。
