### 2. 停止等待与滑动窗口（10分）

（1）停止等待协议的主要缺点是什么？（3分）

**答案**：信道利用率低。发送方每发送一个分组就必须停下来等待确认，在RTT较大时，大部分时间都在等待确认，信道几乎空转。

（2）滑动窗口相比停止等待的核心改进是什么？（3分）

**答案**：滑动窗口允许发送方在窗口范围内连续发送多个分组，不必每发一个就等一个ACK，从而实现流水线传输，大大提高信道利用率。

（3）三者的发送/接收窗口大小（4分）

```
停止等待  发送窗口 =1,  接收窗口 =1
GBN      发送窗口 >1,  接收窗口 =1（通常）
SR       发送窗口 >1,  接收窗口 >1
```

### 4. TCP seq/ack计算（10分）

**已知**：seq=200，数据长度=300字节

（1）B正确接收后回复的ack是多少？（3分）

**步骤**：
- B收到的最后一个字节序号 = 200 + 300 - 1 = 499
- ack = 期望收到的下一个字节 = 499 + 1 = **500**

**答案**：ack = 500

（2）A紧接着发送下一段，seq是多少？（3分）

**步骤**：
- 下一段第一个字节 = 200 + 300 = **500**

**答案**：seq = 500

（3）若该报文段是SYN报文段（SYN=1），seq=200（4分）

**步骤**：
- SYN报文段消耗1个序号（即使不携带数据）
- B回复的ack = 200 + 1 = **201**
- A下一段的seq = 200 + 1 = **201**

**答案**：ack = 201，下一段seq = 201

### 5. TCP拥塞控制（8分）

已知：cwnd=16 MSS，ssthresh=8 MSS

（1）当前处于什么阶段？（2分）

**判断**：cwnd(16) > ssthresh(8)，所以处于**拥塞避免**阶段。

（2）若发生超时重传（3分）

**步骤**：
- ssthresh = cwnd / 2 = 16 / 2 = 8 MSS
- cwnd = 1 MSS
- 重新进入慢开始

**答案**：新ssthresh=8 MSS，新cwnd=1 MSS

（3）若收到3个重复ACK（3分）

**步骤**：
- ssthresh = cwnd / 2 = 16 / 2 = 8 MSS
- cwnd = ssthresh = 8 MSS
- 进入拥塞避免阶段

**答案**：新ssthresh=8 MSS，新cwnd=8 MSS，之后进入拥塞避免

### 6. 三次握手/四次挥手（8分）

（1）为什么需要三次握手而不是两次？（4分）

**答案**：
- 三次握手可以确认双方的发送能力和接收能力都正常。
- 双方交换并确认初始序号。
- 防止已失效的连接请求报文段突然到达服务器，导致服务器错误建立连接、浪费资源。
- 如果只有两次握手，旧的SYN报文可能在网络中延迟很久后到达服务器，服务器误以为客户端要建立新连接。

（2）TIME-WAIT的两个作用和等待时间（4分）

**答案**：
- 作用一：保证最后一个ACK能到达对方。如果最后一个ACK丢失，被动关闭方会重传FIN，主动关闭方仍处于TIME-WAIT状态可以再次回复ACK。
- 作用二：让旧连接中的报文段从网络中自然消失，避免影响后续使用相同四元组的新连接。
- 等待时间：**2MSL**（两倍最长报文段寿命）。

## 网络层

### 7. CIDR计算（10分）

已知：192.168.10.130/26

（1）子网掩码点分十进制形式（2分）

/26 = 前26位为1 = 11111111.11111111.11111111.11000000

**答案**：255.255.255.192

（2）网络地址（3分）

**步骤**：
```
192.168.10.130    → 11000000.10101000.00001010.10 000010
255.255.255.192   → 11111111.11111111.11111111.11 000000
AND ─────────────────────────────────────────────────────
                  = 11000000.10101000.00001010.10 000000
                  = 192.168.10.128
```

**答案**：192.168.10.128/26

（3）广播地址（3分）

**步骤**：网络地址的主机位（后6位）全部置1
```
192.168.10.128    → 11000000.10101000.00001010.10 000000
全部置1           → 11000000.10101000.00001010.10 111111
                  = 192.168.10.191
```

**答案**：192.168.10.191

（4）可用主机地址数（2分）

**步骤**：2^6 - 2 = 64 - 2 = 62

**答案**：62个

### 8. 等长子网划分（10分）

已知：172.16.0.0/16，需要8个等长子网

（1）需要从主机位借几位？（2分）

**步骤**：2^3 = 8，需要借3位

**答案**：借3位

（2）新的子网掩码/前缀长度（2分）

**步骤**：原/16 + 借3位 = /19

**答案**：/19（掩码255.255.224.0）

（3）每个子网可用主机地址数（3分）

**步骤**：
- 剩余主机位 = 32 - 19 = 13
- 地址总数 = 2^13 = 8192
- 可用 = 8192 - 2 = 8190

**答案**：8190个

（4）第一个子网的网络地址和广播地址（3分）

**步骤**：
- 块大小 = 2^(32-19) = 2^13 = 8192（即增量步长）
- 或按/19计算：第三个八位组后3位为子网位

按步长计算：
- 子网0的网络地址：172.16.0.0/19
- 子网0的广播地址：172.16.31.255

**答案**：网络地址172.16.0.0/19，广播地址172.16.31.255

> 验证：/19 第三个八位组 前3位为子网位，子网0=000 00000→0，广播=000 11111→31

### 10. RIP更新（10分）

R6原路由表：
| 目的 | 距离 | 下一跳 |
|---|---|---|
| Net1 | 4 | R4 |
| Net2 | 2 | R5 |
| Net3 | 1 | 直接 |

R4发来的更新（R4到各网络的距离）：
| 目的 | R4距离 |
|---|---|
| Net1 | 3 |
| Net2 | 4 |
| Net4 | 2 |

**更新步骤**（距离向量算法）：

到Net1：R4距离3 → R6经R4到Net1 = 3+1 = 4，与原距离4相同 → 保持R4
到Net2：R4距离4 → R6经R4到Net2 = 4+1 = 5，原经R5距离2更短 → 保持R5
到Net3：R4的更新中没有Net3 → 保持不变
到Net4：新网络，R4距离2 → R6经R4到Net4 = 2+1 = 3

**答案**：

| 目的网络 | 距离 | 下一跳 |
|---|---|---|
| Net1 | 4 | R4 |
| Net2 | 2 | R5 |
| Net3 | 1 | 直接 |
| Net4 | 3 | R4 |

### 12. 综合简答（10分）

（1）跨网络通信时，源主机ARP查询的是谁的MAC地址？为什么？（5分）

**答案**：
ARP查询的是**默认网关（下一跳路由器接口）的MAC地址**，而不是最终目的主机的MAC地址。

原因：ARP只在同一局域网内有效，不能跨网络解析远程主机的MAC地址。当目的主机不在同一网络时，源主机先将数据帧发送给默认网关，每一跳路由器根据路由表转发，每段链路重新用ARP解析下一跳的MAC地址。帧中的目的MAC地址每一跳都会变化（变成下一跳的MAC），而目的IP地址保持不变。

（2）NAT的主要作用是什么？（5分）

**答案**：
NAT（网络地址转换）的主要作用：
- 将内部网络的私有IP地址和端口转换为公网IP地址和端口，使使用私有地址的内网主机能够访问Internet。
- 缓解IPv4地址不足的问题，多个内网主机可以共用一个公网IP地址访问互联网。
- 对外隐藏内部网络结构，提供一定的安全性。
- NAT路由器维护地址转换表，记录每个内部连接映射到哪个公网地址和端口。
