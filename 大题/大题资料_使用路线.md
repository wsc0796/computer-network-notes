# 2. 最省时间的做法

## 第一轮：只做，不翻笔记

```text
每张图限时 8~12 分钟。
不会也要写出已知条件、公式、下一步。
不要边看答案边抄。
```

推荐顺序：

```text
3 -> 4 -> 6 -> 2 -> 5 -> 1 -> 7
```

原因：

```text
先练子网，因为网络层大题最容易反复考；
再练 CRC/RIP/路由；
最后练 IP 分片和 TCP 拥塞控制。
```

## 第二轮：对照笔记订正

订正时只问 3 个问题：

```text
1. 我是概念不会，还是公式不会？
2. 我是哪一步算错？
3. 下次看到同类题，固定模板是什么？
```

不要把整张图抄进笔记，只把“错因 + 模板”写下来。

## 第三轮：闭卷复做错题

```text
当天晚上复做一次。
第二天早上只复做错题。
三天后再随机抽 2 张复做。
```

# 3. 每类题必须形成的模板

## 子网划分

```text
1. 看前缀 /x，主机位 = 32 - x。
2. 可用主机数 = 2^主机位 - 2。
3. 块大小 = 256 - 掩码最后一个非255字节。
4. 找 IP 落在哪个块。
5. 写网络地址、广播地址、可用主机范围。
```

## 路由聚合

```text
1. 把变化的字节写成二进制。
2. 找最长公共前缀。
3. 公共前缀长度就是聚合前缀。
4. 后面全部置0得到聚合地址。
```

## RIP更新

```text
收到邻居 X 的表：
新距离 = 邻居到目标网络的距离 + 1
下一跳 = X

更新规则：
原来没有 -> 加入
原下一跳就是 X -> 更新
原下一跳不是 X 但新距离更短 -> 更新
否则不变
```

## IP分片

```text
每片数据长度必须是 8 字节的整数倍，最后一片除外。
片偏移 = 该片数据起始位置 / 8。
MF = 1 表示后面还有分片。
MF = 0 表示最后一片。
每个分片都有自己的 IP 首部。
```

## TCP拥塞控制

```text
慢开始：cwnd 每 RTT 翻倍。
拥塞避免：cwnd 每 RTT +1。
超时：ssthresh = 原 cwnd / 2，cwnd = 1。
3 个重复 ACK：ssthresh = 原 cwnd / 2，进入快重传/快恢复。
```

# 4. 这批资料的优先级

| 优先级 | 题型 | 原因 |
---|---|---|
| A | 子网划分、路由聚合、RIP更新 | 网络层大题高频，学习通也考 |
| A | IP分片 | 公式固定，容易出大题 |
| A | TCP拥塞控制曲线 | 图题高频，和课堂图高度相关 |
| B | CRC | 已有模板，但要保证手算熟 |
| B | 路由表查找 | 选择/填空/大题都可能出现 |
