# 计算机网络全课程重点梳理

这份笔记是对整门《计算机网络：自顶向下方法》的复习总纲。它不是按原文逐字整理，而是按考试、面试和知识理解的价值重新分层。

重点等级说明：

| 等级 | 含义 |
|---|---|
| 必会 | 高频考点，必须能用自己的话讲清楚 |
| 重点 | 需要理解原理，能应对常见追问 |
| 了解 | 知道是什么、属于哪一层、解决什么问题即可 |

一条总主线：

> 应用产生数据，传输层负责进程到进程通信，网络层负责主机到主机转发，链路层负责相邻节点传帧，物理层负责把比特变成信号。

## 一、全课程知识地图

| 模块 | 核心问题 | 重点程度 |
|---|---|---|
| 网络概述 | 互联网由什么组成？分组如何传？为什么需要分层？ | 必会 |
| 应用层 | 应用程序如何使用网络？HTTP、DNS、邮件、FTP 怎么工作？ | 必会 |
| 传输层 | TCP/UDP 如何为应用进程提供通信？TCP 为什么可靠？ | 必会 |
| 网络层 | IP 如何寻址、转发、路由？路由器如何工作？ | 必会 |
| 数据链路层 | 同一链路/局域网内如何传帧？MAC、ARP、交换机怎么工作？ | 重点 |
| 物理层 | 比特如何变成真实物理信号？ | 了解 |

建议复习顺序：

```text
分层模型
-> 应用层 HTTP/DNS
-> 传输层 TCP/UDP
-> 网络层 IP/路由
-> 链路层 MAC/ARP/交换机
-> 物理层和补充细节
```

## 二、网络概述

### 1. 互联网组成

必会：

- 主机/端系统：运行网络应用程序。
- 通信链路：光纤、同轴电缆、无线、卫星等。
- 分组交换设备：路由器、交换机。
- 协议：规定报文格式、顺序和收发动作。
- 网络边缘：主机和应用程序。
- 网络核心：由路由器组成的网状网络。

重点理解：

- 从具体构成看，互联网是“主机 + 链路 + 交换设备”的集合。
- 从服务角度看，互联网为分布式应用提供通信服务。
- 协议不是软件名字，而是一套通信规则。

了解：

- ISP 层次结构。
- 网络发展历史年份。
- 各种接入网技术的细节。

常见考法：

- “什么是协议？”
- “互联网从组成角度和服务角度分别怎么理解？”
- “路由器和交换机在互联网中分别起什么作用？”

### 2. 分组交换与电路交换

必会：

| 对比 | 电路交换 | 分组交换 |
|---|---|---|
| 是否预留资源 | 预留专用线路 | 不预留，按需共享 |
| 典型场景 | 传统电话网 | 互联网 |
| 优点 | 性能稳定 | 资源利用率高 |
| 缺点 | 空闲时浪费资源 | 可能排队、丢包、乱序 |

重点理解：

- 分组交换把数据切成一个个分组，通过存储转发传输。
- 路由器需要收到完整分组后再转发。
- 分组可能经历排队延迟，也可能因缓存满而丢失。

了解：

- 虚电路的工作方式。
- 频分、时分、波分等资源划分方式。

常见考法：

- “为什么互联网采用分组交换而不是电路交换？”
- “分组交换为什么可能产生排队延迟和丢包？”

### 3. 时延、丢包、吞吐量

必会：

| 概念 | 含义 |
|---|---|
| 处理时延 | 节点检查分组头、决定转发方向的时间 |
| 排队时延 | 分组在路由器队列中等待发送的时间 |
| 传输时延 | 把整个分组推到链路上所需时间 |
| 传播时延 | 信号在链路中传播所需时间 |
| 丢包 | 队列缓存满，新到分组被丢弃 |
| 吞吐量 | 单位时间内实际传输的数据量 |
| 瓶颈链路 | 限制端到端吞吐量的链路 |

重点理解：

- 传输时延取决于分组长度和链路带宽。
- 传播时延取决于距离和信号传播速度。
- 排队时延最不稳定，拥塞时会显著增加。

常见考法：

- “端到端时延由哪些部分组成？”
- “传输时延和传播时延有什么区别？”
- “什么是瓶颈链路？”

### 4. 协议分层和服务模型

必会：

| 层级 | 核心作用 | 典型协议/技术 |
|---|---|---|
| 应用层 | 为应用进程提供网络服务 | HTTP, HTTPS, DNS, FTP, SMTP |
| 传输层 | 进程到进程通信 | TCP, UDP |
| 网络层 | 主机到主机寻址、路由、转发 | IP, ICMP, 路由协议 |
| 链路层 | 相邻节点之间传帧 | Ethernet, Wi-Fi, ARP |
| 物理层 | 在线路上传输比特 | 光纤、双绞线、无线电 |

必会概念：

- 服务：下层向上层提供的通信能力。
- SAP：服务访问点，传输层典型 SAP 是端口。
- SDU：上层交给本层的数据。
- PDU：本层加上头部后的协议数据单元。
- 封装：向下传输时逐层加头部。
- 解封装：向上传输时逐层去头部。

PDU 名称：

| 层级 | 数据单位 |
|---|---|
| 应用层 | Message / 报文 |
| 传输层 | Segment / TCP 段；Datagram / UDP 数据报 |
| 网络层 | Packet 或 IP Datagram |
| 链路层 | Frame / 帧 |
| 物理层 | Bit Stream / 比特流 |

常见考法：

- “为什么计算机网络要分层？”
- “五层模型每层解决什么问题？”
- “封装和解封装过程是什么？”

## 三、应用层（重要协议：fip,smtp,pop3,http,dns）

#### 功能：
应用层的核心问题：应用层对应用程序的通信通过服务

> 应用程序之间如何定义报文格式、交互顺序和应用语义。

### 1. Web 和 HTTP

必会：

- HTTP 是 Web 的应用层协议。
- HTTP 使用 C/S 模式：浏览器是客户端，Web 服务器是服务端。
- HTTP 基于 TCP。
- HTTP 是无状态协议：服务器默认不保存客户历史请求状态。
- URL 用来定位资源。
- HTTP 请求报文：请求行、请求头、空行、请求体。
- HTTP 响应报文：状态行、响应头、空行、响应体。
- 常见状态码：200、301/302、400、403、404、500、505。

重点理解：

- 非持久 HTTP：一个 TCP 连接最多传一个对象。
- 持久 HTTP：多个对象复用同一个 TCP 连接。
- HTTP/1.1 默认使用持久连接。
- HTTP 报文边界由 HTTP 自己定义，例如 `Content-Length`，不是 TCP 负责。

补充必会：

- HTTP 明文传输，默认端口 80。
- HTTPS = HTTP + TLS/SSL，默认端口 443。
- HTTPS 提供加密、身份认证和完整性保护。

常见考法：

- “HTTP 和 HTTPS 的区别？”
- “HTTP 为什么说是无状态的？”
- “非持久连接和持久连接有什么区别？”
- “HTTP 报文由哪些部分组成？”

### 2. DNS

必会：

- DNS 的作用：把域名转换为 IP 地址。
- DNS 是应用层协议。
- DNS 通常使用 UDP，端口 53。
- DNS 采用分布式、层次化数据库。
- 域名结构：根域、顶级域、权威域名服务器、本地域名服务器。
- 常见记录：A、NS、CNAME、MX。

重点理解：

- 本地域名服务器通常先查缓存。
- 没有缓存时，会向根服务器、顶级域服务器、权威服务器逐级查询。
- DNS 缓存能减少查询延迟和服务器压力。

常见考法：

- “DNS 为什么不能只用一个集中式服务器？”
- “DNS 解析过程是什么？”
- “A、CNAME、MX、NS 记录分别是什么？”

### 3. FTP、Email

重点：

- FTP 用于文件传输。
- FTP 使用控制连接和数据连接。
- FTP 是有状态协议。
- SMTP 用于邮件发送，使用 TCP，端口 25。
- MIME 用来支持多媒体邮件内容。
- POP3、IMAP、HTTP Mail 用于邮件访问。

了解：

- FTP 具体命令。
- 邮件系统各组件的完整流程。

常见考法：

- “FTP 为什么需要控制连接和数据连接？”
- “SMTP 和 POP3/IMAP 的区别？”

### 4. P2P、BitTorrent、CDN

重点：

- C/S 模式中服务器压力集中。
- P2P 中节点既可以是客户端，也可以是服务器。
- P2P 适合文件分发、流媒体等场景。
- BitTorrent 通过多个 peer 互相交换文件块。
- CDN 通过地理分布式服务器缓存内容，降低延迟和源站压力。

了解：

- P2P 的集中式、完全分布式、混合式实现。
- CDN 的深入调度策略。

常见考法：

- “P2P 相比 C/S 有什么优缺点？”
- “CDN 解决了什么问题？”

### 5. Socket

必会：

- Socket 是应用进程和传输层之间的接口。
- TCP Socket 面向连接。
- UDP Socket 无连接。
- 端口用于区分同一主机上的不同应用进程。

重点理解：

- TCP 连接通常由四元组标识：源 IP、源端口、目的 IP、目的端口。
- UDP 接收端主要根据目的 IP 和目的端口交付给对应进程。

了解：

- Python socket 代码细节。

常见考法：

- “Socket 在网络编程中起什么作用？”
- “端口号为什么必要？”

## 四、传输层

传输层的核心问题：

> 如何在不同主机上的应用进程之间提供逻辑通信。

### 1. 多路复用与解复用

必会：

- 复用：发送端从多个 socket 收集数据，加上头部后交给网络层。
- 解复用：接收端根据头部信息把数据交给正确 socket。
- 端口号是传输层区分进程的关键。

重点理解：

- TCP 解复用依赖源 IP、源端口、目的 IP、目的端口。
- UDP 解复用主要依赖目的 IP、目的端口。

常见考法：

- “传输层如何把收到的数据交给正确的应用程序？”

### 2. UDP

必会：

- UDP 无连接。
- UDP 不保证可靠、不保证有序、不保证不丢失。
- UDP 头部简单，开销小。
- UDP 支持校验和。
- UDP 适合实时音视频、DNS、流媒体等场景。

重点理解：

- UDP 不负责重传和排序，不代表应用不能自己实现可靠性。
- UDP 的优势是简单、延迟低、控制权交给应用。

常见考法：

- “UDP 和 TCP 有什么区别？”
- “为什么 DNS 常用 UDP？”
- “UDP 是否一定不可靠？”

### 3. 可靠数据传输 RDT

重点：

- RDT 是理解 TCP 可靠性的理论基础。
- 停止等待协议一次只发送一个分组，效率低。
- ACK 表示确认收到。
- NAK 表示否定确认。
- 序号用于区分新分组和重复分组。
- 超时重传用于处理丢包。

重点理解：

- RDT 1.0：可靠信道，不需要额外机制。
- RDT 2.0：处理比特差错，引入 ACK/NAK。
- RDT 2.2：不用 NAK，只用 ACK。
- RDT 3.0：处理丢包，引入超时重传。

了解：

- 状态机图的每个状态。
- 每个版本的完整伪代码。

常见考法：

- “为什么需要序号？”
- “为什么仅靠 ACK 不够，还需要超时重传？”

### 4. 流水线、滑动窗口、GBN、SR

重点：

- 流水线协议允许多个分组同时在网络中传输。
- 滑动窗口限制未确认分组数量。
- GBN：累计确认，接收窗口通常为 1，出错后回退重传。
- SR：选择确认，可以缓存乱序分组，只重传丢失分组。

对比：

| 对比 | GBN | SR |
|---|---|---|
| 接收窗口 | 1 | 大于 1 |
| 接收方式 | 只接收按序分组 | 可缓存乱序分组 |
| 确认方式 | 累计确认 | 单独确认 |
| 重传方式 | 回退重传多个分组 | 只重传丢失分组 |
| 实现复杂度 | 较低 | 较高 |

常见考法：

- “GBN 和 SR 的区别？”
- “为什么流水线能提高信道利用率？”

### 5. TCP

必会：

- TCP 面向连接。
- TCP 提供可靠、有序的字节流服务。
- TCP 是全双工通信。
- TCP 使用序号和确认号。
- TCP 使用累计确认。
- TCP 使用超时重传和快速重传。
- TCP 使用流量控制和拥塞控制。
- TCP 是字节流协议，不保留应用层消息边界。

TCP 连接管理必会：

- 三次握手：建立连接，确认双方收发能力，防止历史连接造成混乱。
- 四次挥手：释放两个方向的连接，因为 TCP 是全双工。

TCP 可靠性必会：

| 机制 | 作用 |
|---|---|
| 序号 | 标识字节流位置，支持排序和去重 |
| ACK | 确认已收到的数据 |
| 超时重传 | ACK 超时未到时重传 |
| 快速重传 | 收到多个重复 ACK 时提前重传 |
| 校验和 | 检测传输错误 |
| 接收缓存 | 支持乱序暂存 |
| 流量控制 | 防止压垮接收方 |
| 拥塞控制 | 防止压垮网络 |

重点理解：

- 流量控制看接收方窗口 RWND。
- 拥塞控制看网络拥塞程度和拥塞窗口 CongWin。
- RTO 根据 EstimatedRTT 和 DevRTT 动态估算。
- TCP 结合了 GBN 和 SR 的一些思想。

常见考法：

- “TCP 如何保证可靠传输？”
- “三次握手为什么不是两次？”
- “四次挥手为什么不是三次？”
- “流量控制和拥塞控制有什么区别？”
- “TCP 粘包/拆包是什么？为什么会发生？”

### 6. 拥塞控制

必会：

- 拥塞表现：丢包、超时、延迟增大。
- 拥塞控制是为了保护网络。
- TCP 通过调整拥塞窗口控制发送速率。
- 超时通常表示严重拥塞。
- 三个重复 ACK 通常表示轻微拥塞。

重点理解：

- 慢启动：拥塞窗口指数增长。
- 拥塞避免：拥塞窗口线性增长。
- 快速重传：收到多个重复 ACK 后不等超时，直接重传。
- 快速恢复：轻微拥塞时降低窗口但不完全从头开始。

了解：

- 公平性公式。
- AIMD 的严格推导。

常见考法：

- “慢启动为什么叫慢启动？”
- “超时和三个重复 ACK 触发的拥塞控制有什么区别？”

## 五、网络层

网络层的核心问题：

> 如何把分组从源主机送到目标主机。

### 1. 转发与路由

必会：

- 转发：数据平面功能，根据转发表把分组从输入端口送到输出端口。
- 路由：控制平面功能，计算从源到目的地的路径。
- 路由器检查 IP 数据报头部，根据目的 IP 地址转发。

重点理解：

- 转发是局部动作。
- 路由是全局路径选择。
- 数据平面负责快，控制平面负责算。

常见考法：

- “转发和路由的区别？”
- “数据平面和控制平面的区别？”

### 2. IP 协议

必会：

- IP 提供尽力而为服务。
- IP 不保证可靠、不保证顺序、不保证不丢包。
- IP 地址标识主机或路由器的接口。
- 路由器通常有多个接口，因此通常有多个 IP 地址。
- IP 数据报中有源 IP、目的 IP、TTL、协议字段等。
- TTL 防止分组在网络中无限循环。

重点理解：

- 大 IP 数据报可能因 MTU 限制被分片。
- 分片重组通常只在目的主机进行。
- IP 本身不负责重传，可靠性通常交给 TCP 或应用层。

了解：

- IP 头部每个字段的位数。
- 分片偏移的详细计算。

常见考法：

- “IP 为什么是不可靠服务？”
- “IP 地址到底标识主机还是接口？”
- “TTL 的作用是什么？”

### 3. IP 地址、子网、CIDR

必会：

- IP 地址分为网络部分和主机部分。
- 子网掩码用于区分网络部分和主机部分。
- 同一子网内主机通常可以不经过路由器直接通信。
- CIDR 使用斜线表示法，例如 `192.168.1.0/24`。
- CIDR 解决传统分类地址浪费和路由表膨胀问题。
- 路由聚集可以减少路由表项。

重点理解：

- 主机位全 0 通常表示网络地址。
- 主机位全 1 通常表示广播地址。
- 私有地址不会在公网中直接路由。

常见私有地址：

| 范围 | 说明 |
|---|---|
| 10.0.0.0/8 | 私有地址 |
| 172.16.0.0/12 | 私有地址 |
| 192.168.0.0/16 | 私有地址 |
| 127.0.0.1 | 回环地址 |

常见考法：

- “什么是子网？”
- “CIDR 有什么作用？”
- “给定 IP 和掩码，求网络地址/广播地址/可用主机数。”

### 4. DHCP、NAT、IPv6

必会 DHCP：

```text
Discover -> Offer -> Request -> Ack
```

DHCP 作用：

- 自动分配 IP 地址。
- 分配子网掩码、默认网关、DNS 服务器等信息。
- 支持移动用户加入网络。

必会 NAT：

- NAT 在私有网络和公网之间转换 IP 地址和端口。
- 多台内网设备可以共享一个公网 IP。
- NAT 维护转换表。

重点 IPv6：

- IPv6 地址为 128 位。
- IPv6 主要为了解决 IPv4 地址不足和路由表膨胀问题。
- IPv6 头部更简化，取消头部校验和。

了解：

- IPv4 到 IPv6 的隧道技术。
- IPv6 报文头字段细节。

常见考法：

- “DHCP 的四步过程是什么？”
- “NAT 为什么能缓解 IPv4 地址不足？”
- “IPv6 相比 IPv4 有哪些变化？”

### 5. 路由算法和路由协议

重点：

- 路由协议目标：找到好的路径，适应网络变化，避免环路。
- LS 链路状态算法：每个节点掌握全局拓扑，常用 Dijkstra。
- DV 距离向量算法：节点只和邻居交换距离向量，逐步收敛。

LS 与 DV：

| 对比 | LS | DV |
|---|---|---|
| 信息范围 | 全局拓扑 | 邻居距离信息 |
| 算法 | Dijkstra | Bellman-Ford |
| 收敛 | 较快 | 可能较慢 |
| 复杂度 | 较高 | 较低 |
| 问题 | 通信和存储开销 | 可能出现路由环路、计数到无穷 |

路由协议：

| 协议 | 类型 | 范围 | 重点 |
|---|---|---|---|
| RIP | 距离向量 | AS 内部 | 小型网络，跳数限制 |
| OSPF | 链路状态 | AS 内部 | 大型网络，收敛快 |
| BGP | 路径向量 | AS 之间 | 策略路由，互联网核心 |

常见考法：

- “LS 和 DV 有什么区别？”
- “RIP 和 OSPF 有什么区别？”
- “BGP 为什么用于 AS 之间？”

### 6. SDN

了解到重点之间：

- SDN 将控制平面和数据平面分离。
- SDN 控制器集中管理网络控制逻辑。
- OpenFlow 是 SDN 交换机和控制器之间的通信协议。
- 通用转发可以按多种字段匹配，而不是只按目的 IP。

常见考法：

- “SDN 的核心思想是什么？”
- “传统路由和 SDN 有什么区别？”

## 六、数据链路层和物理层

链路层核心问题：

> 如何在相邻节点之间可靠或尽力地传输帧。

物理层核心问题：

> 如何把比特变成真实信号传输。

### 1. 链路层服务

必会：

- 成帧：把网络层数据报封装成帧。
- 链路接入：在共享介质上决定谁可以发送。
- 错误检测：发现比特错误。
- 可靠数据传输：在高差错链路中更常见。
- 流量控制：协调收发速度。

重点理解：

- 链路层传输的是一跳。
- 网络层负责端到端路径上的主机到主机转发。
- 链路层使用 MAC 地址，网络层使用 IP 地址。

常见考法：

- “链路层和网络层的区别？”
- “为什么链路层还可能需要可靠传输？”

### 2. 差错检测

重点：

- 奇偶校验：简单，能检测部分错误。
- 二维奇偶校验：可以纠正单比特错误。
- Internet 校验和：用于 TCP/UDP。
- CRC：链路层常用，检错能力强。

了解：

- CRC 多项式计算过程。
- 各种校验方式的严格数学推导。

常见考法：

- “CRC 为什么检错能力更强？”
- “校验和与 CRC 分别常出现在哪些层？”

### 3. 多点接入和 MAC 协议

重点：

- 多点接入要解决多个节点共享信道时谁先发送的问题。
- MAC 协议分为信道划分、随机访问、轮流三类。

分类：

| 类型 | 例子 | 特点 |
|---|---|---|
| 信道划分 | TDMA, FDMA, CDMA | 把信道按时间/频率/编码划分 |
| 随机访问 | ALOHA, CSMA/CD, CSMA/CA | 允许竞争和冲突 |
| 轮流协议 | 轮询、令牌传递 | 按顺序获得发送权 |

重点协议：

- CSMA/CD：用于传统有线以太网，发送时检测碰撞。
- CSMA/CA：用于 Wi-Fi，尽量避免碰撞。

了解：

- ALOHA 效率公式。
- 轮询和令牌传递的完整流程。

常见考法：

- “CSMA/CD 和 CSMA/CA 的区别？”
- “为什么无线网络更适合碰撞避免而不是碰撞检测？”

### 4. LAN、MAC、ARP、交换机

必会：

- MAC 地址是链路层地址，通常 48 位。
- IP 地址用于网络层寻址，MAC 地址用于局域网内一跳传输。
- ARP 作用：根据 IP 地址找到同一局域网内的 MAC 地址。
- ARP 请求通常是广播。
- ARP 响应是目标设备返回自己的 MAC 地址。
- ARP 结果会缓存。

交换机必会：

- 交换机工作在链路层。
- 交换机根据 MAC 地址表转发帧。
- 交换机通过学习源 MAC 地址建立 MAC 地址表。
- 目标 MAC 未知时会泛洪。
- STP 用于避免交换网络中的环路。

交换机与路由器：

| 设备 | 层级 | 看什么地址 | 作用 |
|---|---|---|---|
| 交换机 | 链路层 | MAC 地址 | 同一局域网内转发帧 |
| 路由器 | 网络层 | IP 地址 | 不同网络之间转发数据报 |

常见考法：

- “ARP 的工作过程是什么？”
- “交换机如何学习 MAC 地址？”
- “交换机和路由器有什么区别？”

### 5. 物理层

了解：

- 物理层负责比特传输。
- 物理介质包括双绞线、光纤、无线电、卫星链路等。
- 曼彻斯特编码通过位中间跳变帮助时钟同步。
- 4B/5B 编码通过 5 位表示 4 位，减少连续 0 或 1。
- 调制把数字数据映射到适合介质传输的物理信号。

常见考法：

- “物理层和链路层的区别？”
- “曼彻斯特编码为什么便于同步？”

## 九、答题总模板

遇到任何协议类问题，可以按这个顺序回答：

1. 它在哪一层。
2. 它解决什么问题。
3. 它依赖下层提供什么服务。
4. 它向上层提供什么能力。
5. 它的典型机制是什么。
6. 它和相邻协议有什么区别。

例子：

> TCP 位于传输层，解决不同主机上应用进程之间可靠通信的问题。它依赖网络层 IP 提供的主机到主机尽力而为传输服务，在此基础上通过序号、ACK、重传、流量控制和拥塞控制，为应用层提供可靠、有序、面向连接的字节流服务。它和 UDP 的区别是，UDP 无连接、开销小、不保证可靠和有序，而 TCP 更复杂但提供可靠传输。
