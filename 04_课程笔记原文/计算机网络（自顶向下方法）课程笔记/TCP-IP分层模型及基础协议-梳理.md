# TCP/IP 分层模型及基础协议：复习梳理

这篇是对原始课程笔记的复习版整理。目标不是把所有细节都背下来，而是先抓住主线：

> 网络通信 = 应用数据逐层封装，经过网络传输，到达对方后再逐层解封装。

也就是：

```text
应用数据
-> 传输层加 TCP/UDP 头
-> 网络层加 IP 头
-> 链路层加帧头/帧尾
-> 物理层变成比特信号传出去
```

## 一、TCP/IP 五层模型

| 层级 | 英文 | 核心作用 | 典型协议/技术 | 必要程度 |
|---|---|---|---|---|
| 应用层 | Application | 规定应用程序之间如何交换数据 | HTTP, HTTPS, DNS, FTP, SMTP | 必须掌握 |
| 传输层 | Transport | 负责进程到进程的数据传输，用端口区分应用进程 | TCP, UDP | 必须掌握 |
| 网络层 | Network | 负责主机到主机的寻址、路由和转发 | IP, ICMP, RIP, OSPF, BGP | 必须掌握主线 |
| 链路层 | Link | 负责相邻节点之间传输帧 | Ethernet, Wi-Fi, ARP, MAC | 必须掌握基础 |
| 物理层 | Physical | 把比特变成电信号、光信号或无线信号 | 双绞线、光纤、无线电、编码方式 | 了解即可 |

最重要的一句话：

> 应用层关心“应用消息”，传输层关心“进程”，网络层关心“主机/路由”，链路层关心“相邻节点”，物理层关心“比特信号”。

## 二、每层为什么存在

应用层存在，是为了让具体应用能使用网络。比如浏览网页需要 HTTP，查域名需要 DNS，发邮件需要 SMTP。它定义的是应用之间交换什么内容、用什么格式交换。

传输层存在，是为了让不同主机上的进程能够通信。一台主机上可能同时运行浏览器、微信、游戏，所以需要端口区分不同进程。TCP 提供可靠、有序的字节流，UDP 提供简单、低开销的数据报传输。

网络层存在，是为了解决跨网络传输问题。数据不一定在同一个局域网里，可能要经过很多路由器，所以需要 IP 地址、路由选择和转发机制。

链路层存在，是为了解决一跳之内的传输问题。网络层知道目标 IP，但真正每一跳发送时，需要用 MAC 地址、帧、交换机、Wi-Fi 或以太网把数据送到相邻设备。

物理层存在，是为了让 0 和 1 真正在物理介质上传出去。比如通过电信号、光信号、无线电波传输。

## 三、PDU 名称要记

| 层级 | 数据单位 |
|---|---|
| 应用层 | 报文 / Message |
| 传输层 | TCP 段 / Segment；UDP 数据报 / Datagram |
| 网络层 | IP 数据报 / Packet 或 Datagram |
| 链路层 | 帧 / Frame |
| 物理层 | 比特流 / Bit Stream |

这个表很容易考，因为它能检查你是否真的理解了“逐层封装”。

## 四、基础协议怎么放进分层模型

### 1. 应用层协议

| 协议 | 作用 | 必要程度 |
|---|---|---|
| HTTP | Web 应用层协议，浏览器和服务器交换请求/响应报文 | 必须掌握 |
| HTTPS | HTTP + TLS/SSL，加密、认证、完整性保护 | 必须掌握 |
| DNS | 域名解析，把域名转换为 IP 地址 | 必须掌握 |
| FTP | 文件传输协议 | 知道作用即可 |
| SMTP | 邮件发送协议 | 知道作用即可 |

HTTP/HTTPS 重点：

- HTTP 是明文传输，默认端口 80。
- HTTPS 是 HTTP 加 TLS/SSL，默认端口 443。
- HTTPS 多了 TLS 握手和证书验证。
- HTTP 本身是无状态的。
- HTTP 基于 TCP，但 HTTP 报文边界由 HTTP 自己定义，不是 TCP 帮它定义。

DNS 重点：

- 作用是域名到 IP 地址的转换。
- 通常使用 UDP，端口 53。
- DNS 是应用层协议，但它服务于整个网络访问流程。

### 2. 传输层协议

| 协议 | 特点 | 常见应用 |
|---|---|---|
| TCP | 面向连接、可靠、有序、字节流、带流量控制和拥塞控制 | HTTP/HTTPS, FTP, SMTP |
| UDP | 无连接、不可靠、不保证有序、开销小、面向数据报 | DNS, 流媒体, 实时音视频 |

TCP 必须掌握：

- 三次握手：建立连接，确认双方收发能力。
- 四次挥手：关闭两个方向的连接，因为 TCP 是全双工。
- 可靠性保证：序号、确认 ACK、超时重传、快速重传、校验和、流量控制、拥塞控制。
- 流量控制：照顾接收方，避免接收缓冲区被撑爆。
- 拥塞控制：照顾网络，避免网络中数据过多导致丢包和延迟。
- 粘包/拆包：TCP 是字节流，不保留应用层消息边界。

UDP 必须掌握：

- 不建立连接，直接发送。
- 不保证可靠、不保证顺序、不负责重传。
- 速度快、开销小，适合实时性要求高、能容忍部分丢失的场景。

### 3. 网络层协议和机制

| 内容 | 作用 | 必要程度 |
|---|---|---|
| IP | 提供主机/接口寻址和数据报转发 | 必须掌握 |
| ICMP | 传递网络错误和诊断信息，ping/traceroute 会用到 | 知道作用 |
| 路由协议 | 让路由器学习路径 | 掌握 RIP/OSPF/BGP 区别 |
| DHCP | 自动分配 IP 地址等网络配置 | 必须掌握基础 |
| NAT | 私有地址和公网地址转换，缓解 IPv4 地址不足 | 必须掌握基础 |

IP 重点：

- IP 提供的是尽力而为服务，不保证可靠、不保证顺序、不保证不丢包。
- IP 地址标识的是主机或路由器的接口，不是简单地标识“一台机器”。
- 路由器根据目标 IP 地址查转发表，决定下一跳。
- TTL 用来限制数据报在网络中无限循环。
- IP 分片用于处理数据报大于链路 MTU 的情况，重组通常在目的主机完成。

DHCP 重点：

```text
Discover -> Offer -> Request -> Ack
```

它解决的是：主机如何自动获得 IP 地址、子网掩码、默认网关、DNS 服务器等配置。

NAT 重点：

- 内网设备使用私有 IP。
- NAT 设备把内网 IP/端口映射成公网 IP/端口。
- 多台设备可以共享一个公网 IP 访问互联网。

路由协议重点：

| 协议 | 类型 | 适用范围 |
|---|---|---|
| RIP | 距离向量，按跳数 | 小型网络，了解 |
| OSPF | 链路状态，算最短路径 | AS 内部路由，重点理解 |
| BGP | 路径向量，策略路由 | AS 之间路由，知道地位 |

### 4. 链路层协议和机制

| 内容 | 作用 | 必要程度 |
|---|---|---|
| Ethernet | 有线局域网常用链路层技术 | 必须知道 |
| Wi-Fi / 802.11 | 无线局域网技术 | 必须知道 |
| MAC 地址 | 局域网内标识网卡/接口 | 必须掌握 |
| ARP | 根据 IP 地址找到同一局域网内的 MAC 地址 | 必须掌握 |
| 交换机 | 根据 MAC 地址表转发帧 | 必须掌握基础 |

ARP 重点：

- 作用：IP 地址 -> MAC 地址。
- 只在同一局域网内工作。
- ARP 请求通常是广播。
- ARP 响应告诉请求方目标 IP 对应的 MAC 地址。
- 结果会被缓存到 ARP 表中。

交换机重点：

- 工作在链路层。
- 学习源 MAC 地址和端口的对应关系。
- 根据目标 MAC 地址转发帧。
- 不知道目标 MAC 时会泛洪。

路由器和交换机的区别：

| 设备 | 主要依据 | 所在层级 | 核心作用 |
|---|---|---|---|
| 交换机 | MAC 地址 | 链路层 | 在局域网内转发帧 |
| 路由器 | IP 地址 | 网络层 | 在不同网络之间转发数据报 |

### 5. 物理层内容

物理层在这部分只需要抓住一句话：

> 物理层负责把比特变成真实物理信号，并在传输介质上传送。

曼彻斯特编码、4B/5B 编码、调制方式等内容，如果是课程考试可能会考细节；如果是面试或通用复习，知道它们属于物理层/编码方式即可。

## 五、哪些内容是必要的

### 必须掌握

- TCP/IP 五层模型：每层作用、典型协议、为什么需要这一层。
- 封装和解封装过程。
- PDU 名称：Message、Segment、Datagram/Packet、Frame、Bit。
- TCP 和 UDP 的区别。
- TCP 三次握手、四次挥手、可靠性保证。
- TCP 粘包/拆包的原因：TCP 是字节流，不保留消息边界。
- HTTP 和 HTTPS 的区别。
- DNS 的作用和端口。
- IP 地址、子网、CIDR、默认网关的基本概念。
- 路由器转发的基本逻辑。
- ARP 的作用：IP 到 MAC。
- DHCP 的作用：自动分配网络配置。
- NAT 的作用：私网地址转换为公网地址。
- 交换机和路由器的区别。

### 重点理解，但不需要死背所有细节

- TCP 超时重传时间的计算公式。
- RDT、GBN、SR 和 TCP 可靠传输之间的关系。
- 拥塞控制的慢启动、拥塞避免、快速恢复。
- IP 分片和重组。
- RIP、OSPF、BGP 的详细算法过程。
- CSMA/CD 和 CSMA/CA 的工作过程。
- CRC、校验和、奇偶校验的原理。

这些内容的价值在于帮你理解协议为什么这样设计，但答题时通常不需要展开到所有公式。

### 了解即可

- OSI 七层模型中会话层、表示层的细节。
- 互联网发展历史年份。
- ALOHA 的效率公式。
- 曼彻斯特编码、4B/5B 编码的完整码表。
- IP 头部每一个字段的位数。
- 各类传统 A/B/C 类地址的完整范围。

这些内容如果不是课程考试重点，可以先压缩成“知道它是什么、属于哪一层、解决什么问题”。

## 六、容易混淆的点

1. HTTPS 不是简单换端口，而是 HTTP + TLS/SSL。
2. TCP 可靠不等于网络不丢包，而是丢了能发现并重传。
3. TCP 是字节流，所以应用层必须自己处理消息边界。
4. UDP 不可靠不代表没用，它适合低延迟、可容忍丢失的场景。
5. IP 地址更准确地说是接口地址，路由器通常有多个 IP 地址。
6. ARP 只解决同一局域网内的 IP 到 MAC 映射。
7. 交换机看 MAC 地址，路由器看 IP 地址。
8. 流量控制是保护接收方，拥塞控制是保护网络。
9. DNS 是应用层协议，但它通常服务于几乎所有网络访问。
10. HTTP 报文边界靠 HTTP 协议格式解决，不靠 TCP。

## 七、答题模板

如果题目问“请说明 TCP/IP 分层模型及基础协议”，可以这样答：

> TCP/IP 协议栈通常按五层理解：应用层、传输层、网络层、链路层、物理层。应用层直接为应用程序提供网络服务，典型协议有 HTTP、HTTPS、DNS、FTP、SMTP；传输层负责进程到进程通信，典型协议有 TCP 和 UDP，其中 TCP 提供可靠、有序、面向连接的字节流服务，UDP 提供无连接、低开销的数据报服务；网络层负责主机到主机的寻址、路由和转发，核心协议是 IP，相关协议有 ICMP、RIP、OSPF、BGP；链路层负责相邻节点之间传输帧，典型技术有 Ethernet、Wi-Fi、ARP、MAC 地址和交换机；物理层负责把比特转换成电信号、光信号或无线信号在介质上传输。
>
> 这套分层模型的意义在于降低网络系统复杂度。每一层只完成自己的职责，并向上一层提供服务。发送数据时从应用层向下逐层封装，接收数据时从物理层向上逐层解封装。

## 八、复习顺序建议

1. 先背五层模型表格：层级、作用、协议。
2. 再理解一次浏览器访问网站的过程：DNS -> TCP/TLS -> HTTP -> IP -> ARP/MAC -> 物理传输。
3. 接着重点看 TCP：三握四挥、可靠性、流量控制、拥塞控制、粘包拆包。
4. 然后看 IP：地址、子网、路由、NAT、DHCP。
5. 最后看链路层：MAC、ARP、交换机、Ethernet/Wi-Fi。

复习时优先保证自己能回答三个问题：

1. 这一层解决什么问题？
2. 这一层的典型协议是什么？
3. 这一层给上一层提供了什么服务？
