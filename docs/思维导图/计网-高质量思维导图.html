01 选择判断：触发词到答案

ARP请求广播；ARP响应单播；路由器不转发ARP广播
跨网通信MAC查默认网关MAC，不是最终目的主机MAC
最长前缀匹配多条路由都匹配时，选择前缀最长的一条
RIP跳数最大有效15跳；16表示不可达
OSPF算法链路状态协议；Dijkstra最短路径
BGP范围AS之间；路径向量；策略优先
TCP确认号期望收到的下一个字节序号
TIME_WAIT主动关闭方进入；等待2MSL
端口号长度16位；HTTP 80，HTTPS 443，DNS常用UDP 53
CRC作用差错检测，不是自动纠错
交换机学习根据源MAC地址动态学习
以太网最小帧64字节；MAC地址48位

简答DNS解析

  • 作用：域名转换成IP地址
  • 路径：缓存 -> 本地DNS -> 根 -> TLD -> 权威
  • 陷阱：根DNS通常不给最终IP
定义一句话
过程三到五步
最后补陷阱

简答TCP三次握手

  • SYN seq=x
  • SYN+ACK seq=y ack=x+1
  • ACK ack=y+1
  • 原因：确认双方收发能力，防止失效SYN

简答TCP四次挥手

  • FIN -> ACK -> FIN -> ACK
  • 原因：TCP全双工，两个方向独立关闭
  • TIME_WAIT：保证最后ACK到达，让旧报文消失

03 网络层大题：看到输入就走模板

子网划分

h=32-前缀；总地址数2^h；可用主机2^h-2；网络地址看步长或做IP & 掩码。

CIDR/聚合

先写二进制，找最长公共前缀，公共前缀后置0。查路由时多条匹配选最长前缀。

IP分片

数据长度=总长度-首部；非最后片数据长度取8的倍数；片偏移=数据起点/8；最后MF=0。

RIP更新

新距离=邻居通告距离+1；没有就加，原下一跳是该邻居就更新，更短就更新，16不可达。

04 TCP专题：画图、状态、为什么

P1三次握手

  • C -> S：SYN
  • S -> C：SYN+ACK
  • C -> S：ACK
  • 客户端：CLOSED -> SYN_SENT -> ESTABLISHED
  • 服务端：LISTEN -> SYN_RCVD -> ESTABLISHED

P1四次挥手

  • FIN -> ACK -> FIN -> ACK
  • 主动方最后进TIME_WAIT
  • 等待2MSL
TCP高分点
不能只背名词

判断可靠传输

  • TCP按字节编号
  • ACK表示期望下一个字节
  • 超时重传，重复ACK触发快重传
  • 发送窗口= min(rwnd, cwnd)

判断控制机制

  • 流量控制保护接收方：rwnd
  • 拥塞控制保护网络：cwnd
  • 慢开始指数，拥塞避免线性
