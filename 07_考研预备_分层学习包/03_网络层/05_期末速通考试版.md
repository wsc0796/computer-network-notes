## 第二部分：网络层考试版笔记

## A. 必背概念

### 1. 网络层的作用

【考点】网络层功能  
【一句话理解】网络层负责把分组从源主机送到目的主机，中间靠路由器转发。  
【考试怎么问】网络层的主要功能是什么？  
【标准答案】网络层向运输层提供主机到主机的逻辑通信服务，主要负责 IP 编址、路由选择、分组转发、拥塞相关控制辅助，以及通过 ARP、ICMP、NAT 等机制支持分组在不同网络之间传输。  
【易错点】网络层不是进程到进程通信；进程到进程是运输层，网络层是主机到主机。

### 2. 转发 vs 路由

【考点】转发与路由选择  
【一句话理解】转发是“一个包来了往哪发”，路由是“路由表怎么来的”。  
【考试怎么问】转发和路由有什么区别？  
【标准答案】转发是路由器收到分组后，根据路由表把分组从合适接口送出的局部动作；路由选择是通过路由算法或路由协议生成、维护路由表的全局过程。  
【易错点】查路由表转发分组叫转发；RIP/OSPF/BGP 维护路由信息叫路由。

### 3. 数据报服务 vs 虚电路服务

【考点】网络层服务模型  
【一句话理解】数据报像寄普通信，每个包独立走；虚电路像先建专线，再沿固定路线走。  
【考试怎么问】比较虚电路服务和数据报服务。  
【标准答案】数据报服务不预先建立连接，每个分组独立携带目的地址并独立选路，可能乱序、丢失或重复，因而灵活但不保证可靠；虚电路服务通信前先建立逻辑连接，分组沿已建立的虚电路转发，分组首部可携带虚电路号，顺序性较好但建立和维护连接开销较大。  
【易错点】互联网 IP 层采用数据报服务，不是虚电路服务。

### 4. IP 地址

【考点】IPv4 地址  
【一句话理解】IP 地址是网络层逻辑地址，用来标识网络中的接口位置。  
【考试怎么问】IPv4 地址多少位？作用是什么？  
【标准答案】IPv4 地址是 32 位地址，通常写成点分十进制形式，用于标识主机或路由器的网络接口，并支持网络层寻址和路由选择。  
【易错点】IP 地址不是网卡硬件地址；MAC 地址才是数据链路层硬件地址。

### 5. 分类 IP 地址

【考点】A/B/C/D/E 类地址  
【一句话理解】早期按第一个字节范围把 IP 地址分成固定类别。  
【考试怎么问】某个 IP 属于哪一类？默认掩码是多少？  
【标准答案】A 类第一个字节 1~126，默认 /8；B 类 128~191，默认 /16；C 类 192~223，默认 /24；D 类 224~239 用于组播；E 类 240~255 保留。  
【易错点】127.0.0.0/8 是回环地址，不作为普通 A 类网络使用；0.0.0.0 也有特殊含义。

### 6. 子网掩码

【考点】子网掩码作用  
【一句话理解】子网掩码告诉你 IP 地址前多少位是网络部分，后多少位是主机部分。  
【考试怎么问】子网掩码有什么作用？  
【标准答案】子网掩码用于区分 IP 地址中的网络号和主机号。将 IP 地址与子网掩码逐位 AND，可得到该 IP 所属网络的网络地址。  
【易错点】掩码中连续的 1 表示网络位，连续的 0 表示主机位；不要把主机位全 0 或全 1 当可用主机地址。

### 7. CIDR

【考点】无分类域间路由 CIDR  
【一句话理解】CIDR 用 `/前缀长度` 表示网络范围，不再死守 A/B/C 类。  
【考试怎么问】`192.168.1.0/26` 表示什么？  
【标准答案】CIDR 使用 `a.b.c.d/x` 表示地址块，其中 x 是网络前缀长度，剩余 `32-x` 位为主机位。CIDR 支持更灵活的地址分配和路由聚合。  
【易错点】`/26` 比 `/24` 范围更小，但匹配更具体。

### 8. 网络地址、广播地址、可用主机范围

【考点】地址块三件套  
【一句话理解】主机位全 0 是网络地址，全 1 是广播地址，中间才是可用主机。  
【考试怎么问】给定 IP/掩码，求网络地址、广播地址、可用范围。  
【标准答案】网络地址由 IP 与子网掩码逐位 AND 得到；广播地址是在网络前缀不变的情况下将主机位全部置 1；可用主机范围通常是网络地址 +1 到广播地址 -1。  
【易错点】可用主机数通常为 `2^主机位数 - 2`，点到点链路等特殊场景另说，期末一般按减 2。

### 9. ARP

【考点】地址解析协议 ARP  
【一句话理解】ARP 解决“已知下一跳 IP，查它的 MAC 地址”。  
【考试怎么问】ARP 的作用和过程是什么？跨网络通信时 ARP 查谁？  
【标准答案】ARP 用于在同一局域网内根据 IP 地址解析对应 MAC 地址。主机先查 ARP 缓存，若没有则广播 ARP 请求，目标设备单播 ARP 响应，发送方将映射写入缓存。跨网络通信时，主机 ARP 查询的是默认网关/下一跳路由器接口的 MAC 地址，而不是最终目的主机的 MAC 地址。  
【易错点】ARP 不跨网络查询远程主机 MAC；跨网络时帧的目的 MAC 是下一跳 MAC。

### 10. ICMP

【考点】网际控制报文协议 ICMP  
【一句话理解】ICMP 用来报告网络层差错和做网络诊断。  
【考试怎么问】ping 使用什么协议？ICMP 有什么作用？  
【标准答案】ICMP 封装在 IP 数据报中，用于差错报告和网络诊断。例如 TTL 超时、目的不可达等情况可发送 ICMP 差错报文；ping 通常使用 ICMP 回显请求和回显应答。  
【易错点】ICMP 不是运输层协议；它服务于网络层控制和诊断。

### 11. 路由表与最长前缀匹配

【考点】路由器转发  
【一句话理解】多条路由都能匹配时，选前缀最长、最具体的那条。  
【考试怎么问】给路由表和目的 IP，问从哪个接口转发。  
【标准答案】路由器取出目的 IP 地址，与路由表项逐项匹配。如果有多条路由匹配，选择网络前缀最长的表项；若没有具体匹配，可使用默认路由 `0.0.0.0/0`；若无匹配，则丢弃并可能发送 ICMP 差错报文。  
【易错点】不是选第一条匹配，也不是选范围最大，而是选前缀长度最大的匹配项。

### 12. 静态路由

【考点】静态路由  
【一句话理解】管理员手动写死的路线。  
【考试怎么问】静态路由有什么优缺点？  
【标准答案】静态路由由管理员手工配置，优点是简单、稳定、开销小、可控性强；缺点是不能自动适应拓扑变化，网络变化时需要人工维护，适合小型、稳定网络。  
【易错点】静态路由不等于默认路由；默认路由可以手工配置，也可以由协议获得。

### 13. RIP

【考点】RIP 路由协议  
【一句话理解】RIP 按跳数选路，小网络够用，大网络不行。  
【考试怎么问】RIP 的特点是什么？为什么适合小型网络？  
【标准答案】RIP 是内部网关协议，基于距离向量算法，以跳数为度量，最大有效跳数为 15，16 表示不可达。RIP 周期性与邻居交换路由信息，协议简单但收敛较慢，适合小型网络。  
【易错点】RIP 的“距离”不是带宽或时延，而是跳数。

### 14. OSPF

【考点】OSPF 路由协议  
【一句话理解】OSPF 让路由器掌握拓扑，用最短路径算法算路线。  
【考试怎么问】OSPF 与 RIP 有什么不同？  
【标准答案】OSPF 是内部网关协议，基于链路状态算法，路由器洪泛链路状态信息，使区域内路由器形成一致拓扑数据库，再用 Dijkstra 算法计算最短路径。OSPF 收敛快、适合大型网络。  
【易错点】OSPF 的度量通常是开销/带宽，不是跳数；OSPF 用于 AS 内部。

### 15. BGP

【考点】BGP  
【一句话理解】BGP 是 AS 之间的路由协议，选路受策略影响，不只是最短。  
【考试怎么问】BGP 为什么不一定选择最短路径？  
【标准答案】BGP 是外部网关协议，用于自治系统 AS 之间，采用路径向量思想。互联网 AS 之间存在商业、管理和策略约束，因此 BGP 路由选择不只考虑最短路径，还要考虑策略。  
【易错点】RIP/OSPF 是 IGP，BGP 是 EGP；BGP 不是普通局域网内部路由协议。

### 16. IPv4 分组格式

【考点】IPv4 首部字段  
【一句话理解】IPv4 首部告诉路由器怎么转发、还能不能分片、上层交给谁。  
【考试怎么问】TTL、协议字段、标识、MF、片偏移分别有什么作用？  
【标准答案】TTL 每经过一个路由器减 1，用于防止分组无限循环；协议字段标识上层协议，如 ICMP=1、TCP=6、UDP=17；标识字段用于标记同一原始数据报的各分片；MF=1 表示后面还有分片，MF=0 表示最后一片；片偏移以 8 字节为单位表示该片数据在原始数据报中的位置。  
【易错点】IPv4 首部最小 20 字节；片偏移单位是 8 字节，不是 1 字节。

### 17. IP 分片与重组

【考点】IPv4 分片  
【一句话理解】大 IP 包过不了小 MTU 链路时，IPv4 可以拆成多个片，最后由目的主机重组。  
【考试怎么问】什么时候分片？谁负责重组？片偏移怎么算？  
【标准答案】当 IPv4 数据报长度超过下一链路 MTU 且 DF=0 时，可进行分片。各分片具有相同标识字段，除最后一片外，每片数据长度必须是 8 字节的整数倍；片偏移以 8 字节为单位；MF=1 表示后续还有分片，最后一片 MF=0。分片由目的主机重组，中间路由器不重组。  
【易错点】只要一个分片丢失，整个原始数据报无法正常重组；最后一片数据长度可以不是 8 的倍数。

### 18. NAT

【考点】网络地址转换 NAT  
【一句话理解】NAT 让多个内网私有地址主机共用公网地址访问互联网。  
【考试怎么问】NAT 的作用是什么？基本原理是什么？  
【标准答案】NAT 在内部私有 IP 地址/端口与公网 IP 地址/端口之间建立映射转换表，使多个内网主机可以共享一个或少量公网 IP 访问 Internet，缓解 IPv4 地址不足，也隐藏了内网结构。  
【易错点】NAT 不是 DNS；DNS 是域名到 IP，NAT 是内网地址和公网地址转换。

## B. 选择/判断高频点

- 网络层提供主机到主机的逻辑通信，运输层提供进程到进程通信。
- IPv4 地址是 32 位，MAC 地址通常是 48 位。
- IP 层采用尽最大努力交付，不保证可靠、顺序、不重复。
- CIDR 的 `/x` 表示网络前缀长度，x 越大网络越小、匹配越具体。
- 可用主机数通常等于 `2^主机位 - 2`。
- ARP 是 IP 地址到 MAC 地址，DNS 是域名到 IP 地址。
- 跨网络通信时，ARP 查下一跳网关的 MAC，不查最终目的主机的 MAC。
- ICMP 封装在 IP 数据报中，ping 使用 ICMP。
- 路由器根据目的 IP 和路由表转发，不根据目的 MAC 做跨网路由。
- 最长前缀匹配不是按路由表顺序选，而是在所有匹配项中选前缀最长的。
- 默认路由 `0.0.0.0/0` 匹配所有地址，但前缀最短。
- RIP 最大 15 跳，16 表示不可达。
- RIP 是距离向量，OSPF 是链路状态，BGP 是路径向量。
- IPv4 首部最小 20 字节，最大 60 字节。
- TTL 每经过路由器减 1，减到 0 时丢弃并可发送 ICMP 超时报文。
- IP 首部协议字段：ICMP=1，TCP=6，UDP=17。
- 同一原始数据报的分片标识字段相同。
- 片偏移单位是 8 字节。
- 分片由目的主机重组，不由中间路由器重组。
- NAT 可缓解 IPv4 地址不足，但会影响端到端透明性。

## C. 简答题高频点

### 1. 网络层主要功能

【考点】网络层功能  
【一句话理解】编地址、选路径、转发包。  
【考试怎么问】简述网络层的主要功能。  
【标准答案】网络层主要负责主机到主机的逻辑通信，通过 IP 地址进行逻辑寻址，通过路由选择确定分组传输路径，通过转发将分组从合适接口送出，并通过 ICMP、ARP、NAT 等协议或机制辅助网络互连。  
【易错点】不要把端口号、多路复用写成网络层功能；那是运输层。

### 2. ARP 工作过程

【考点】ARP  
【一句话理解】广播问“这个 IP 是谁”，目标单播回答 MAC。  
【考试怎么问】简述 ARP 的工作过程。  
【标准答案】主机发送数据前先查 ARP 缓存，若没有目标 IP 与 MAC 的映射，则在同一局域网内广播 ARP 请求。拥有该 IP 的主机收到后单播 ARP 响应，告知自己的 MAC 地址。发送方把映射写入 ARP 缓存，再封装成帧发送。若目的主机不在同一子网，则 ARP 查询默认网关的 MAC。  
【易错点】ARP 请求通常广播，ARP 响应通常单播。

### 3. ICMP 作用

【考点】ICMP  
【一句话理解】IP 网络的差错报告和诊断工具。  
【考试怎么问】ICMP 有什么作用？ping 使用什么协议？  
【标准答案】ICMP 用于在 IP 网络中报告差错和进行网络诊断，常见报文包括目的不可达、超时、回显请求和回显应答等。ping 使用 ICMP 回显请求和回显应答检测连通性。  
【易错点】ICMP 只报告错误，不负责纠正错误，也不保证可靠传输。

### 4. NAT 作用

【考点】NAT  
【一句话理解】内网地址出门时换成公网地址。  
【考试怎么问】简述 NAT 的作用和基本原理。  
【标准答案】NAT 将内部私有 IP 地址和端口转换为公网 IP 地址和端口，并维护转换表，使多个内网主机共享公网 IP 访问互联网，缓解 IPv4 地址短缺，同时隐藏内网结构。返回报文到达 NAT 设备后，再根据转换表还原给对应内网主机。  
【易错点】NAT 常转换 IP 和端口，不只是单纯换 IP。

### 5. 路由器转发分组过程

【考点】路由表转发  
【一句话理解】取目的 IP，查最长前缀，改 TTL，重新封装转发。  
【考试怎么问】路由器收到 IP 分组后如何处理？  
【标准答案】路由器收到帧后取出 IP 数据报，检查首部，取目的 IP 地址查路由表，按最长前缀匹配选择下一跳或输出接口；将 TTL 减 1，若 TTL 为 0 则丢弃并可发送 ICMP 超时报文；重新计算 IP 首部校验和，再封装到下一链路层帧中发送。  
【易错点】每一跳链路层帧头会变化，IP 目的地址通常不变。

### 6. RIP vs OSPF

【考点】路由协议对比  
【一句话理解】RIP 看跳数，OSPF 看链路状态。  
【考试怎么问】比较 RIP 和 OSPF。  
【标准答案】RIP 基于距离向量算法，以跳数为度量，最大 15 跳，周期性与邻居交换路由表，协议简单但收敛慢，适合小型网络。OSPF 基于链路状态算法，以链路开销为度量，通过洪泛链路状态信息建立拓扑数据库，再用 Dijkstra 算法求最短路径，收敛快，适合大型网络。  
【易错点】OSPF 不是外部网关协议；BGP 才用于 AS 之间。

## 高频对比表

### 1. IP 地址 vs MAC 地址

| 对比项 | IP 地址 | MAC 地址 |
|---|---|---|
| 层次 | 网络层 | 数据链路层 |
| 长度 | IPv4 32 位，IPv6 128 位 | 通常 48 位 |
| 作用 | 跨网络逻辑寻址和路由 | 同一链路/局域网内帧交付 |
| 是否每跳变化 | 源/目的 IP 通常不变 | 每一跳源/目的 MAC 会变化 |
| 设备依据 | 路由器看 IP 转发 | 交换机看 MAC 转发 |

### 2. ARP vs ICMP

| 对比项 | ARP | ICMP |
|---|---|---|
| 作用 | IP 地址解析为 MAC 地址 | 差错报告和网络诊断 |
| 常见场景 | 发帧前查下一跳 MAC | ping、TTL 超时、目的不可达 |
| 范围 | 同一局域网内解析 | 封装在 IP 中跨网络传输 |
| 易错点 | 不是域名解析 | 不是运输层协议 |

### 3. 数据报服务 vs 虚电路服务

| 对比项 | 数据报服务 | 虚电路服务 |
|---|---|---|
| 是否连接 | 不预先建立连接 | 通信前建立虚电路 |
| 每个分组 | 独立选路 | 沿虚电路转发 |
| 首部信息 | 携带完整目的地址 | 可携带虚电路号 |
| 顺序性 | 可能乱序 | 顺序性较好 |
| 典型 | IP | X.25、ATM 等思想 |

### 4. 分类 IP vs CIDR

| 对比项 | 分类 IP | CIDR |
|---|---|---|
| 地址划分 | A/B/C 类固定边界 | 任意前缀长度 |
| 表示方式 | 默认掩码 /8、/16、/24 | `a.b.c.d/x` |
| 灵活性 | 低，容易浪费 | 高，支持聚合 |
| 考试重点 | 判断类别和默认掩码 | 子网、聚合、最长前缀 |

### 5. RIP vs OSPF

| 对比项 | RIP | OSPF |
|---|---|---|
| 类型 | IGP | IGP |
| 算法 | 距离向量 | 链路状态 |
| 度量 | 跳数 | 开销/带宽 |
| 最大范围 | 15 跳，16 不可达 | 无 RIP 式跳数限制 |
| 更新 | 周期性与邻居交换 | 洪泛链路状态，触发更新为主 |
| 收敛 | 慢 | 快 |
| 适用 | 小型网络 | 大型 AS 内部网络 |

### 6. 静态路由 vs 动态路由

| 对比项 | 静态路由 | 动态路由 |
|---|---|---|
| 配置 | 人工配置 | 路由协议自动学习 |
| 优点 | 简单、稳定、开销小 | 能适应拓扑变化 |
| 缺点 | 维护麻烦 | 占用资源，配置复杂 |
| 适用 | 小型稳定网络 | 大型复杂网络 |

### 7. 网络地址 vs 广播地址

| 对比项 | 网络地址 | 广播地址 |
|---|---|---|
| 主机位 | 全 0 | 全 1 |
| 含义 | 表示整个网段 | 表示该网段所有主机 |
| 是否可分配给普通主机 | 通常不可 | 通常不可 |
| 例子 | 192.168.1.0/24 | 192.168.1.255 |

### 8. 子网掩码 vs 前缀长度

| 对比项 | 子网掩码 | 前缀长度 |
|---|---|---|
| 表示 | 255.255.255.0 | /24 |
| 含义 | 1 为网络位，0 为主机位 | 网络位个数 |
| 转换 | 24 个连续 1 = 255.255.255.0 | /26 = 255.255.255.192 |

### 9. NAT 的内网地址 vs 公网地址

| 对比项 | 内网地址 | 公网地址 |
|---|---|---|
| 范围 | 私有地址，如 10/8、172.16/12、192.168/16 | 全球唯一可路由地址 |
| 是否能直接上互联网 | 不能直接在公网路由 | 可以在公网路由 |
| NAT 中角色 | 被转换的一方 | 对外显示的一方 |
| 常见题眼 | 多台主机共享公网 IP | NAT 转换表记录映射 |

## 第三部分：计算题模板

## 1. 已知 IP 和子网掩码，求网络地址

【题型】给 IP 地址和子网掩码，求网络地址。  
【解题步骤】  
1. 把 IP 和掩码写成二进制，或只看发生变化的那一段。  
2. IP 与子网掩码逐位 AND。  
3. 得到网络地址。  

【例题】IP=`192.168.1.130`，掩码=`255.255.255.192`，求网络地址。  
【答案】`255.255.255.192` 是 `/26`，最后一段块大小为 `256-192=64`，子网段为 `0,64,128,192`。130 落在 128~191，所以网络地址是 `192.168.1.128/26`。  
【易错点】不要把主机 IP 当网络地址；要找它所在块的起点。

## 2. 已知 CIDR，求可用主机数

【题型】给 `a.b.c.d/x`，求地址总数和可用主机数。  
【解题步骤】  
1. 主机位数 = `32 - x`。  
2. 地址总数 = `2^主机位数`。  
3. 可用主机数通常 = `2^主机位数 - 2`。  

【例题】`192.168.1.0/26` 有多少可用主机地址？  
【答案】主机位数 `32-26=6`，地址总数 `2^6=64`，可用主机数 `64-2=62`。  
【易错点】减 2 是减网络地址和广播地址；不要把 `/26` 误认为 26 个主机。

## 3. 已知网段，划分若干等长子网

【题型】给一个网段，要求划分为若干个等长子网。  
【解题步骤】  
1. 看需要多少个子网，找最小 n 使 `2^n >= 子网数`。  
2. 从原主机位中借 n 位作为子网位，新前缀 = 原前缀 + n。  
3. 每个子网块大小 = `2^(剩余主机位)`。  
4. 依次列网络地址、广播地址、可用范围。  

【例题】把 `192.168.1.0/24` 划分为 4 个等长子网。  
【答案】需要 4 个子网，借 2 位，新前缀 `/26`，块大小 64。  

| 子网 | 网络地址 | 广播地址 | 可用主机范围 |
|---|---|---|---|
| 1 | 192.168.1.0/26 | 192.168.1.63 | 192.168.1.1~62 |
| 2 | 192.168.1.64/26 | 192.168.1.127 | 192.168.1.65~126 |
| 3 | 192.168.1.128/26 | 192.168.1.191 | 192.168.1.129~190 |
| 4 | 192.168.1.192/26 | 192.168.1.255 | 192.168.1.193~254 |

【易错点】每个子网的起点按块大小递增；广播地址是下一个子网网络地址减 1。

## 4. 求每个子网的网络地址、广播地址、主机范围

【题型】给 CIDR 地址块，要求列出某个子网的三件套。  
【解题步骤】  
1. 根据前缀长度求主机位数。  
2. 块大小 = `2^主机位数`。  
3. 网络地址 = 所在块起点。  
4. 广播地址 = 网络地址 + 块大小 - 1。  
5. 可用范围 = 网络地址 + 1 到广播地址 - 1。  

【例题】`172.16.10.77/28` 的网络地址、广播地址、可用范围是什么？  
【答案】`/28` 主机位 4 位，块大小 16。最后一段块起点为 `0,16,32,48,64,80...`，77 落在 64~79。  
网络地址：`172.16.10.64/28`。  
广播地址：`172.16.10.79`。  
可用范围：`172.16.10.65~172.16.10.78`。  
【易错点】`/28` 块大小是 16，不是 28。

## 5. VLSM 不等长子网划分

【题型】一个大网段给多个部门，主机数需求不同。  
【解题步骤】  
1. 按主机需求从大到小排序。  
2. 对每个需求找最小主机位 h，使 `2^h - 2 >= 主机数`。  
3. 前缀长度 = `32 - h`。  
4. 从低地址开始依次分配，每次按块大小对齐。  

【例题】用 `192.168.1.0/24` 给 A=100 台、B=50 台、C=20 台、D=10 台划分。  
【答案】  

| 部门 | 需求 | 前缀 | 网络地址 | 可用范围 | 广播地址 |
|---|---:|---|---|---|---|
| A | 100 | /25 | 192.168.1.0 | 192.168.1.1~126 | 192.168.1.127 |
| B | 50 | /26 | 192.168.1.128 | 192.168.1.129~190 | 192.168.1.191 |
| C | 20 | /27 | 192.168.1.192 | 192.168.1.193~222 | 192.168.1.223 |
| D | 10 | /28 | 192.168.1.224 | 192.168.1.225~238 | 192.168.1.239 |

【易错点】必须从大需求开始分；否则后面大块可能放不下。

## 6. 路由表最长前缀匹配题

【题型】给路由表和目的 IP，判断出口/下一跳。  
【解题步骤】  
1. 判断目的 IP 是否落入每条路由的地址范围。  
2. 找出所有匹配项。  
3. 在匹配项中选前缀长度最大的。  
4. 若无匹配，使用默认路由；若无默认路由，则丢弃。  

【例题】路由表如下，目的 IP 为 `192.168.1.130`，从哪个接口转发？

| 目的网络 | 接口 |
|---|---|
| 192.168.1.0/24 | 0 |
| 192.168.1.128/25 | 1 |
| 192.168.0.0/16 | 2 |
| 0.0.0.0/0 | 3 |

【答案】`192.168.1.130` 同时匹配 `/24`、`/25`、`/16` 和 `/0`，最长前缀是 `/25`，所以从接口 1 转发。  
【易错点】默认路由能匹配所有 IP，但前缀最短，通常最后才用。

## 7. IP 分片题

【题型】给 IP 数据报总长度、首部长度、MTU，求各片总长度、数据长度、片偏移、MF。  
【解题步骤】  
1. 原始数据长度 = IP 总长度 - IP 首部长度。  
2. 每片最大数据长度 = MTU - IP 首部长度。  
3. 除最后一片外，每片数据长度必须取不超过最大数据长度的 8 的整数倍。  
4. 片偏移 = 该片前面已有数据长度 / 8。  
5. 最后一片 MF=0，其余分片 MF=1。  
6. 每片总长度 = 该片数据长度 + IP 首部长度。  

【例题】IP 数据报总长度 1500 字节，首部 20 字节，经过 MTU=620 字节链路，求分片。  
【答案】原始数据长度 `1500-20=1480`。每片最大数据长度 `620-20=600`，600 是 8 的倍数。分为 3 片：

| 分片 | 数据长度 | 总长度 | 片偏移 | MF |
|---|---:|---:|---:|---:|
| 1 | 600 | 620 | 0 | 1 |
| 2 | 600 | 620 | 75 | 1 |
| 3 | 280 | 300 | 150 | 0 |

【易错点】片偏移单位是 8 字节，所以第二片偏移是 `600/8=75`；最后一片数据长度可以不是 8 的倍数。

## 8. RIP 距离向量更新题

【题型】收到邻居路由表，更新本路由器路由表。  
【解题步骤】  
1. 邻居发来的所有距离先 +1。  
2. 新网络：添加。  
3. 距离更短：更新。  
4. 来自同一下一跳邻居：即使距离变大也更新。  
5. 距离达到 16：表示不可达。  

【例题】R 当前表：N1 距离 3 下一跳 R2；N2 距离 5 下一跳 R3；N3 距离 2 下一跳 R1。收到 R2 表：N1 距离 2；N2 距离 4；N4 距离 3。求更新。  
【答案】R2 表距离 +1 后：N1=3，N2=5，N4=4。N1 来自同一邻居 R2，更新后仍为 3/R2；N2 距离不更短且不是同一邻居，不更新；N4 是新网络，添加 4/R2。  
【易错点】“来自同一邻居”要更新，这是 RIP 表更新题最容易漏的规则。

## 考前 20 分钟背诵版

网络层负责主机到主机的逻辑通信，核心任务是 IP 编址、路由选择和分组转发。转发是路由器收到一个分组后查路由表并从某个接口发出；路由是通过路由算法或路由协议生成、维护路由表。IP 层提供尽最大努力交付，不保证可靠、顺序和不重复，可靠性主要由上层 TCP 等协议负责。互联网 IP 层采用数据报服务，每个分组独立选路；虚电路服务则需要先建立逻辑连接，分组沿虚电路转发。

IPv4 地址为 32 位，通常写成点分十进制。IP 地址是网络层逻辑地址，用于跨网络寻址和路由；MAC 地址是数据链路层地址，用于当前链路上的帧交付。跨网络通信时，源/目的 IP 通常不变，但每一跳的源/目的 MAC 会变化。分类地址中 A 类默认 /8，B 类 /16，C 类 /24；CIDR 用 `a.b.c.d/x` 表示地址块，x 是网络前缀长度，打破了 A/B/C 类固定边界。子网掩码用于区分网络位和主机位，IP 与掩码做 AND 得到网络地址。网络地址是主机位全 0，广播地址是主机位全 1，可用主机数通常为 `2^主机位 - 2`。

子网划分必须会算：给 IP/掩码，先看前缀长度和块大小；网络地址是所在块起点，广播地址是块末尾，可用范围是中间地址。等长划分时，需要多少子网就借多少位；VLSM 不等长划分必须从主机需求最大的子网开始分配。最长前缀匹配必须单独记：目的 IP 可能匹配多条路由，不能选第一条，也不能选范围最大，而是在所有匹配项中选择前缀长度最大的；`/25` 比 `/24` 更具体，默认路由 `0.0.0.0/0` 只是保底。

ARP 用于已知 IP 查 MAC。主机先查 ARP 缓存，未命中则广播 ARP 请求，目标单播响应。高频陷阱：ARP 只查同一局域网内的 MAC；跨网络通信时，源主机查的是默认网关/下一跳路由器接口的 MAC，不是最终目的主机的 MAC。ICMP 封装在 IP 中，用于差错报告和网络诊断，ping 使用 ICMP。NAT 在内网私有 IP/端口和公网 IP/端口之间建立映射，使多个内网主机共享公网 IP 访问互联网，缓解 IPv4 地址不足。

IPv4 首部最小 20 字节。TTL 每过一个路由器减 1，减到 0 丢弃并可发 ICMP 超时报文；协议字段标识上层协议，ICMP=1、TCP=6、UDP=17；标识字段用于同一原始数据报分片归组；DF=1 禁止分片；MF=1 表示后面还有分片；片偏移单位是 8 字节。分片题要先算数据长度，再算每片最大数据长度，除最后一片外数据长度必须是 8 的倍数，最后一片 MF=0，分片由目的主机重组。

RIP 是内部网关协议，距离向量算法，以跳数为度量，最大 15 跳，16 表示不可达，适合小型网络。OSPF 是内部网关协议，链路状态算法，洪泛链路状态并用 Dijkstra 算最短路径，收敛快，适合大型 AS 内部网络。BGP 是外部网关协议，用于 AS 之间，路径选择受策略影响，不一定选最短路径。

## 第五部分：自测题 + 答案解析

## 一、选择/判断题

### 1. 网络层提供的是哪种通信？

A. 进程到进程  
B. 主机到主机  
C. 比特到比特  
D. 用户到用户  

答案：B。  
解析：网络层负责主机到主机通信，运输层负责进程到进程通信。

### 2. IPv4 地址长度是 48 位。对还是错？

答案：错。  
解析：IPv4 是 32 位，MAC 地址通常是 48 位。

### 3. CIDR 记法 `192.168.1.0/24` 中，24 表示主机位长度。对还是错？

答案：错。  
解析：/24 表示网络前缀长度，主机位是 8 位。

### 4. 以下哪个协议用于 IP 地址到 MAC 地址的解析？

A. DNS  
B. ARP  
C. ICMP  
D. NAT  

答案：B。  
解析：ARP 是 IP 到 MAC；DNS 是域名到 IP。

### 5. ping 通常使用哪个协议？

A. TCP  
B. UDP  
C. ICMP  
D. ARP  

答案：C。  
解析：ping 使用 ICMP 回显请求和回显应答。

### 6. RIP 的最大有效跳数是？

A. 8  
B. 15  
C. 16  
D. 255  

答案：B。  
解析：RIP 最大 15 跳，16 表示不可达。

### 7. 最长前缀匹配中，`/26` 比 `/24` 更具体。对还是错？

答案：对。  
解析：前缀长度越大，网络范围越小，匹配越具体。

### 8. 普通 IP 数据报分片由中间路由器重组。对还是错？

答案：错。  
解析：IPv4 分片由目的主机重组，中间路由器不重组。

### 9. IPv4 首部中 TTL 的作用是防止数据报在网络中无限循环。对还是错？

答案：对。  
解析：TTL 每过路由器减 1，减到 0 后丢弃。

### 10. BGP 主要用于自治系统 AS 之间。对还是错？

答案：对。  
解析：BGP 是外部网关协议，用于 AS 之间，策略驱动。

### 11. `192.168.1.1` 是私有地址，访问互联网通常需要 NAT。对还是错？

答案：对。  
解析：`192.168.0.0/16` 是私有地址范围。

### 12. 目的 IP 匹配默认路由 `0.0.0.0/0`，说明它一定会选默认路由。对还是错？

答案：错。  
解析：默认路由匹配所有 IP，但只有没有更长匹配时才选它。

## 二、简答题

### 1. 简述网络层的作用。

答案：网络层负责主机到主机的逻辑通信，主要进行 IP 编址、路由选择和分组转发，使分组能够经过多个路由器从源主机到达目的主机。  
解析：关键词是“主机到主机、IP、路由、转发”。

### 2. 比较数据报服务和虚电路服务。

答案：数据报服务不预先建立连接，每个分组独立携带目的地址并独立选路，可能乱序或丢失；虚电路服务通信前先建立逻辑连接，分组沿虚电路转发，顺序性较好，但需要建立和维护连接。  
解析：互联网 IP 层采用数据报服务。

### 3. 简述 ARP 的工作过程，并说明跨网络通信时查谁的 MAC。

答案：主机先查 ARP 缓存，若没有映射，则在同一局域网内广播 ARP 请求，目标设备单播 ARP 响应，发送方缓存映射后发送帧。若目的主机不在同一子网，ARP 查询的是默认网关/下一跳路由器接口的 MAC，而不是最终目的主机的 MAC。  
解析：跨网络 ARP 是高频陷阱。

### 4. 简述 NAT 的作用。

答案：NAT 将内网私有 IP 地址和端口转换为公网 IP 地址和端口，并维护映射表，使多个内网主机可以共享公网 IP 访问互联网，缓解 IPv4 地址不足，也能隐藏内网结构。  
解析：答题时写“地址和端口映射”更完整。

### 5. 比较 RIP、OSPF、BGP。

答案：RIP 是 IGP，基于距离向量，以跳数为度量，最大 15 跳，适合小型网络；OSPF 是 IGP，基于链路状态，以开销为度量，洪泛链路状态并用 Dijkstra 计算，适合大型 AS 内部网络；BGP 是 EGP，用于 AS 之间，基于路径向量，选路受策略影响。  
解析：IGP/EGP、算法、度量、场景是四个得分点。

## 三、计算/应用题

### 1. 子网计算题

题目：已知 IP 地址 `192.168.10.77`，子网掩码 `255.255.255.240`。求网络地址、广播地址、可用主机范围和可用主机数。

答案：掩码 `255.255.255.240` 是 `/28`，主机位 4 位，块大小 `16`。最后一段 77 落在 64~79。  
网络地址：`192.168.10.64/28`。  
广播地址：`192.168.10.79`。  
可用范围：`192.168.10.65~192.168.10.78`。  
可用主机数：`2^4-2=14`。  
解析：块大小 = 256 - 240 = 16。

### 2. 最长前缀匹配题

题目：路由表如下，目的 IP 为 `10.1.2.130`，应从哪个接口转发？

| 目的网络 | 接口 |
|---|---|
| 10.0.0.0/8 | A |
| 10.1.0.0/16 | B |
| 10.1.2.0/24 | C |
| 10.1.2.128/25 | D |
| 0.0.0.0/0 | E |

答案：`10.1.2.130` 匹配 `/8`、`/16`、`/24`、`/25` 和 `/0`，最长前缀是 `/25`，所以从接口 D 转发。  
解析：不是选最先匹配的 A，也不是默认路由 E，而是选最具体的 D。

### 3. IP 分片题

题目：IPv4 数据报总长度为 4000 字节，首部长度为 20 字节，经过 MTU=1500 字节的链路，DF=0。求各分片的数据长度、总长度、片偏移和 MF。

答案：原始数据长度 `4000-20=3980`。每片最大数据长度 `1500-20=1480`，且 1480 是 8 的倍数。  
分片如下：

| 分片 | 数据长度 | 总长度 | 片偏移 | MF |
|---|---:|---:|---:|---:|
| 1 | 1480 | 1500 | 0 | 1 |
| 2 | 1480 | 1500 | 185 | 1 |
| 3 | 1020 | 1040 | 370 | 0 |

解析：第二片偏移 `1480/8=185`；第三片偏移 `(1480+1480)/8=370`；最后一片 MF=0，数据长度可以不是 8 的倍数。

### 4. VLSM 应用题

题目：用 `192.168.2.0/24` 给三个部门划分子网：A 需要 60 台，B 需要 30 台，C 需要 12 台。给出每个子网的网络地址、广播地址和可用范围。

答案：按需求从大到小分。A 需 60 台，`2^6-2=62`，用 `/26`；B 需 30 台，`2^5-2=30`，用 `/27`；C 需 12 台，`2^4-2=14`，用 `/28`。

| 部门 | 网络地址 | 广播地址 | 可用范围 |
|---|---|---|---|
| A | 192.168.2.0/26 | 192.168.2.63 | 192.168.2.1~62 |
| B | 192.168.2.64/27 | 192.168.2.95 | 192.168.2.65~94 |
| C | 192.168.2.96/28 | 192.168.2.111 | 192.168.2.97~110 |

解析：VLSM 必须先分大网段，再分小网段，并且每个子网起点要按块大小对齐。
