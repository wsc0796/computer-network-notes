## 2. 必背公式与规则

```text
地址总数 = 2^主机位数
可用主机数 = 2^主机位数 - 2
片偏移单位 = 8 字节
最长前缀匹配：多个匹配时选前缀最长者
RIP 最大跳数：15，16 表示不可达
路由聚合：找所有地址的"最长公共前缀"
```

## 3. D1：子网划分题（选填计算常出）

**公式速记：**

```text
已知 IP 和掩码 → 求网络地址
  ① 把 IP 和掩码写成二进制
  ② IP AND 掩码 = 网络地址

已知 网络地址/前缀 → 求广播地址
  ① 主机位全部置 1
  ② 网络地址 + (2^主机位数 - 1)

可用主机范围 = 网络地址+1 ~ 广播地址-1
可用主机数 = 2^主机位数 - 2
```

**例题1：**

> IP = 192.168.1.130，掩码 = 255.255.255.192（/26），求网络地址、广播地址、可用主机范围。

```text
步骤：
  192.168.1.130        → 11000000.10101000.00000001.10 000010
  255.255.255.192      → 11111111.11111111.11111111.11 000000
  ──────────────────────────────────────────────────────────
  AND结果（网络地址）   → 11000000.10101000.00000001.10 000000
                       = 192.168.1.128/26

  广播地址：主机位（后6位）全置 1
            11000000.10101000.00000001.10 111111
            = 192.168.1.191

  可用范围：192.168.1.129 ~ 192.168.1.190
  可用主机数：2^6 - 2 = 62
```

**例题2：**

> 某公司使用 192.168.1.0/24，需要划分 4 个等长子网，求每个子网的网络地址、广播地址、可用范围。

```text
步骤：
  需要 4 个子网 → 借 2 位（2^2 = 4）
  原 /24 → /26，子网掩码 255.255.255.192
  每个子网有 2^6 - 2 = 62 个可用地址

  子网1：192.168.1.0/26   广播 192.168.1.63    可用 1~62
  子网2：192.168.1.64/26  广播 192.168.1.127   可用 65~126
  子网3：192.168.1.128/26 广播 192.168.1.191   可用 129~190
  子网4：192.168.1.192/26 广播 192.168.1.255   可用 193~254
```

## 4. D2：路由聚合题

**规则：找所有地址的"最长公共前缀"**

**例题：**

> 将以下 4 条路由聚合成一条：172.16.0.0/24、172.16.1.0/24、172.16.2.0/24、172.16.3.0/24

```text
步骤：
  172.16.0.0/24 → 172.16.000000 00.0
  172.16.1.0/24 → 172.16.000000 01.0
  172.16.2.0/24 → 172.16.000000 10.0
  172.16.3.0/24 → 172.16.000000 11.0
                  └──── 前22位相同 ────┘
  聚合结果：172.16.0.0/22

  验证：/22 有 2^(24-22) = 4 个 /24，恰好覆盖这4条。
```

## 5. D2：最长前缀匹配（路由表转发题）

**三步法：**
```text
① 目的IP 与每条路由的掩码做 AND（或用 /x 判断范围）
② 落在网络地址范围内 → 这条匹配
③ 多条匹配 → 选 /x 最大的那个
```

**例题：**

路由表：
| 目的网络 | 下一跳/接口 |
|---------|-----------|
| 192.168.1.0/24 | 接口0 |
| 192.168.1.128/25 | 接口1 |
| 192.168.0.0/16 | 接口2 |
| 0.0.0.0/0 | 接口3 |

目的IP = **192.168.1.130**，从哪个接口转发？

```text
逐条检查：
  192.168.1.0/24   → 范围 192.168.1.0~255      ✅ 匹配 /24
  192.168.1.128/25 → 范围 192.168.1.128~255    ✅ 匹配 /25 ← 最长
  192.168.0.0/16   → 范围 192.168.0.0~255.255  ✅ 匹配 /16
  0.0.0.0/0        → 全部匹配                  ✅ 匹配 /0

选最长前缀 → 接口1
```

## 6. D3：RIP 距离向量更新题（大题常考）

**更新三规则：**
```text
收到邻居的更新表，邻居的所有距离 +1，然后：
① 新网络 → 添加
② 距离更短 → 更新
③ 来自同一邻居 → 强制更新（不管距离变大变小）
```

**例题：**

路由器 R 当前路由表：
| 目的网络 | 距离 | 下一跳 |
|---------|:----:|:-----:|
| N1 | 3 | R2 |
| N2 | 5 | R3 |
| N3 | 2 | R1 |

收到邻居 R2 发来的路由表：
| 目的网络 | 距离 |
|---------|:----:|
| N1 | 2 |
| N2 | 4 |
| N4 | 3 |

求更新后的路由表。

```text
R2 的每条距离 +1 → N1:3, N2:5, N4:4

逐条对比：
  N1：R2 发来距离 3，当前也是 3（下一跳 R2，同一邻居）
      → 同一邻居 → 强制更新（即使相等也更新）
      → N1: 3, R2 ✓（不变）

  N2：R2 发来距离 5，当前是 5（下一跳 R3）
      → 新距离 5 ≥ 当前 5，且不是同一邻居 → 不更新
      → N2: 5, R3 ✓（不变）

  N4：新网络，当前路由表没有
      → 直接添加 → N4: 4, R2

更新后：
| 目的网络 | 距离 | 下一跳 |
|---------|:----:|:-----:|
| N1 | 3 | R2 |
| N2 | 5 | R3 |
| N3 | 2 | R1 |
| N4 | 4 | R2 |
```

## 7. D3：RIP vs OSPF 对比模板

| 对比项 | RIP | OSPF |
|-------|-----|------|
| 算法 | 距离向量（Bellman-Ford） | 链路状态（Dijkstra） |
| 度量 | 跳数 | 带宽/开销 |
| 最大跳数 | 15（16=不可达） | 无限制 |
| 更新方式 | 定期广播整个路由表 | 触发洪泛链路状态 |
| 收敛速度 | 慢（可能计数到无穷） | 快 |
| 复杂度 | 简单 | 复杂 |
| 适用 | 小型网络 | 大型企业/ISP |

**一句话记死：**
```text
RIP   = 小区快递（15跳极限 + 定期广播 = 只能小网络）
OSPF = 顺丰全国网（无跳数限制 + 触发更新 = 可到任何AS内部）
BGP  = 国际物流（谈策略、谈价格 = 互联网AS之间）
```

**适用场景的底层逻辑：**
```text
RIP 的 15 跳极限和定期广播机制 → 只能在小网络里玩
OSPF 的全网拓扑一致性要求     → 只能在 AS 内部用
BGP 的策略驱动特性             → 必须在 AS 之间用
```

**简答模板：**
```text
RIP 基于距离向量算法，以跳数为度量，最大 15 跳，16 跳表示不可达。
路由器每 30 秒向邻居广播整个路由表，收敛较慢，适合小型网络。

OSPF 基于链路状态算法，以链路带宽为度量，无跳数限制。
路由器通过洪泛链路状态信息让全网拓扑一致，收敛快，适合大型网络。
```

## 8. D4：IP 分片计算题

**公式：**
```text
数据长度 = 总长度 - IP首部（通常20字节）
每片数据长度 ≤ (MTU - 20)，且必须是 8 的整数倍
片偏移 = 前一片的数据累计长度 / 8
MF：1 表示后面还有分片，0 表示最后一片
```

**例题：**

> IP 数据报总长 1500 字节（含 20 字节首部），需经过 MTU=620 字节的链路，求分片情况。

```text
原始数据长度 = 1500 - 20 = 1480 字节
MTU = 620，每片最大数据长度 = 620 - 20 = 600 字节
600 / 8 = 75，是 8 的倍数，符合要求

分片数 = ceil(1480 / 600) = 3 片

分片1：数据 0~599（600字节），偏移 0，  MF=1
分片2：数据 600~1199（600字节），偏移 75，MF=1
分片3：数据 1200~1479（280字节），偏移 150，MF=0
                                                  ↑ 最后一片 MF=0

| 分片 | 总长度 | 数据长度 | 片偏移 | MF |
|:----:|:-----:|:--------:|:-----:|:--:|
| 1 | 620 | 600 | 0 | 1 |
| 2 | 620 | 600 | 75 | 1 |
| 3 | 300 | 280 | 150 | 0 |
```

## 9. D4：简答模板库

### ARP 解析过程
```text
主机已知目标 IP 但不知道目标 MAC 时，
在同一局域网内广播 ARP 请求。
目标主机收到后单播回复 ARP 响应，告知自己的 MAC 地址。
发送方随后将 IP 数据报封装成帧发往该 MAC 地址。
```

### NAT 的作用
```text
NAT 在内网地址和公网地址之间做映射转换，
使多个内网主机共享一个公网 IP 访问互联网。
它缓解了 IPv4 地址短缺，但破坏了端到端透明性。
```

### DHCP 的 DORA 过程
```text
Discover：主机广播"谁给我 IP？"（源IP=0.0.0.0，目的IP=255.255.255.255）
Offer：DHCP 服务器可广播或单播（取决于客户端状态和广播标志位）
Request：主机通常广播"我要了这个 IP"（通知其他DHCP服务器）
ACK：服务器可广播或单播确认，租期开始
```
> 基础题常按"全程广播"简化处理；以老师课堂口径为准。DHCP本身是应用层协议（UDP端口67/68），因其直接服务于IP地址配置，常与网络层一起学习。

### 路由器转发分组过程
```text
① 收到 IP 数据报
② 取出目的 IP 地址
③ 查路由表，最长前缀匹配找到下一跳/端口
④ TTL 减 1，重新计算首部校验和
⑤ 将分组从对应端口转发出去
⑥ 若无匹配 → 丢弃，发 ICMP 差错报文
```

### 静态路由 vs 动态路由
```text
静态路由由管理员手动配置，不依赖路由协议。
优点是简单、可控、不占用网络带宽；
缺点是拓扑变化时需要人工调整，适合小型稳定网络。

动态路由通过路由协议自动交换信息，
能自适应拓扑变化，适合大型复杂网络。
```

**考试常见考法：**

|------|------|

| 首部长度最小/最大？ | 最小 20 字节（5×4），最大 60 字节（15×4） |
| 同一原始数据报的分片哪个字段相同？ | **标识（Identification）** |

## 12. 补丁3：ARP 跨网络陷阱（大题常出）

**场景题：**

> 主机 A（192.168.1.2/24）访问互联网服务器 B（8.8.8.8）。
> 请问：① A 会 ARP 查询谁的 MAC？② 数据帧中的目的 MAC 是什么？

```text
答：
① A 发现 B 不在同一子网，不会查 B 的 MAC，
   而是查默认网关（路由器接口 192.168.1.1）的 MAC。
② 帧中的目的 MAC = 网关接口的 MAC，不是 B 的 MAC。
```

> **一句话：跨网络通信时，ARP 永远查下一跳路由器的 MAC，不是最终目的主机的 MAC。**

## 13. 补丁4：VLSM 不等长子网划分

**公式：**
```text
核心原则：从大到小分配
① 最大需求的子网先分
② 每分完一个，剩余地址块继续分
```

**例题：**

> 使用 192.168.1.0/24 地址块，按以下需求划分子网：
> 部门A：100台 | 部门B：50台 | 部门C：20台 | 部门D：10台

```text
部门A 需 100 台 → 需要 2^7 - 2 = 126 ≥ 100，主机位 7 位 → /25
  子网1：192.168.1.0/25，可用 1~126
  剩余地址：192.168.1.128/25

部门B 需 50 台 → 需要 2^6 - 2 = 62 ≥ 50，主机位 6 位 → /26
  子网2：192.168.1.128/26，可用 129~190
  剩余地址：192.168.1.192/26

部门C 需 20 台 → 需要 2^5 - 2 = 30 ≥ 20，主机位 5 位 → /27
  子网3：192.168.1.192/27，可用 193~222
  剩余地址：192.168.1.224/27

部门D 需 10 台 → 需要 2^4 - 2 = 14 ≥ 10，主机位 4 位 → /28
  子网4：192.168.1.224/28，可用 225~238
  剩余地址：192.168.1.240/28（留用）
```

## 14. 补丁5：IP 分片完整规则

**在现有分片计算基础上，补充以下概念题：**

```text
① 同一原始数据报的各分片，标识字段相同 → 重组时归组依据
② DF=1 时禁止分片，超 MTU 则丢弃+ICMP
③ 最后一片的 MF=0，且数据长度可以不是 8 的整数倍
④ 分片由目的主机重组，不是中间路由器（IPv6 路由器也不分片）
⑤ 只要有一个分片丢失，整个原始数据报无法正常重组
```

**概念判断题：**

| 题干 | 答案 |
|------|------|
| 同一原始数据报的各分片 IP 首部完全相同 | ❌ 片偏移和 MF 不同 |
| 分片由中间路由器重组 | ❌ 由目的主机重组 |
| 最后一片的 MF=1 | ❌ MF=0 |
| 所有分片的数据长度都必须是 8 的倍数 | ❌ 最后一片可以不是 |
| 一个分片丢失，其他分片可以正常用 | ❌ 整个数据报无法重组 |
