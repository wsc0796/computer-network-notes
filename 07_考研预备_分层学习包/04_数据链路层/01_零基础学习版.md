## 1. 数据链路层解决什么问题

数据链路层解决的是：

```text
相邻节点之间如何可靠/有效地传输帧。
```

它不负责端到端，也不负责跨多个网络的路由。它关注的是一段链路上的传输。

可以这样理解：

```text
网络层关心 IP 分组往哪走。
数据链路层关心这一跳怎么把帧发过去。
```

## 2. 数据链路层主线

```text
把网络层数据报封装成帧
    ↓
解决帧边界和透明传输
    ↓
进行差错检测
    ↓
在共享信道中解决介质访问控制
    ↓
用 MAC 地址和交换机实现局域网通信
```

## 3. 三个基本问题

数据链路层常考三个基本问题：

```text
封装成帧
透明传输
差错检测
```

封装成帧：

```text
在数据前后加首部和尾部，使接收方能识别帧边界。
```

透明传输：

```text
无论数据中出现什么比特组合，都不能被误认为控制信息或帧边界。
```

差错检测：

```text
检测传输过程中是否发生比特错误。
常用方法是 CRC。
```

## 4. CRC

CRC 是循环冗余校验，用于差错检测。

做题流程：

```text
1. 看生成多项式对应的二进制位串。
2. 设生成多项式位数为 k，则在原数据后补 k-1 个 0。
3. 用模 2 除法除以生成多项式。
4. 得到余数，把余数接到原数据后面作为校验序列。
```

注意：

```text
CRC 主要用于检错，不是纠错。
模 2 加减法等价于异或，不进位、不借位。
```

## 5. PPP

PPP 是点对点协议，常用于点对点链路。

需要记：

```text
PPP 不提供可靠传输。
PPP 支持封装成帧、透明传输和差错检测。
```

## 6. 介质访问控制与 CSMA/CD

在共享信道中，多个站点可能同时发送数据，产生冲突。

CSMA/CD：

```text
载波监听多点接入/碰撞检测。
```

核心思想：

```text
先听再发，边发边听，冲突停止，随机退避后重发。
```

常见结论：

```text
最坏情况下检测到冲突需要 2 倍端到端传播时延。
CSMA/CD 主要用于传统共享式以太网。
```

## 7. MAC 地址与交换机

MAC 地址：

```text
数据链路层地址，通常 48 位。
```

交换机工作在：

```text
数据链路层。
```

交换机 MAC 表学习：

```text
交换机根据收到帧的源 MAC 地址和进入端口动态学习 MAC 表。
转发时根据目的 MAC 地址查表。
查不到时泛洪。
```
