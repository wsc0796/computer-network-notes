# 网络概述 —— 易错概念对比

> 408选择题的陷阱集中地
> 建议：遮住右侧先自己判断，再看解析

---

## 1. 协议 vs 服务

| 对比维度 | 协议 | 服务 |
|:---------|:-----|:-----|
| 作用对象 | **对等层**之间 | **上下层**之间 |
| 方向 | **水平**方向 | **垂直**方向 |
| 能看见什么 | 对等实体之间的交互 | 下层为上层提供的能力 |
| 具体内容 | 语法、语义、时序 | 服务原语（请求/指示/响应/证实） |

### 易错判断题

| 陈述 | 答案 | 解释 |
|:-----|:----|:-----|
| "协议是下层为上层提供的功能" | ❌ | 那是**服务**的定义 |
| "服务是对等实体间通信的规则" | ❌ | 那是**协议**的定义 |
| "协议是水平的，服务是垂直的" | ✅ | 这是最核心的区分 |

## 2. PDU vs SDU vs PCI

| 缩写 | 全称 | 含义 | 谁加/谁看 |
|:-----|:-----|:-----|:---------|
| **SDU** | 服务数据单元 | 上层交给下层的数据 | 下层**不解析**SDU内容 |
| **PCI** | 协议控制信息 | 本层加的首部/尾部 | 对等层之间约定 |
| **PDU** | 协议数据单元 | SDU + PCI = 本层要发的东西 | 对等层交换的单位 |

### 关系图

```text
上一层交下来的数据 = 本层的 SDU
本层的 SDU + 本层首部(PCI) = 本层的 PDU
本层的 PDU = 下一层的 SDU

简言之：PDU = PCI + SDU
       本层PDU = 下层SDU
```

### 易错

| 陈述 | 答案 | 解释 |
|:-----|:----|:-----|
| "数据链路层的PDU叫分组" | ❌ | 是**帧**，分组是网络层PDU |
| "物理层的PDU是帧" | ❌ | 是**比特**，帧是数据链路层PDU |
| "运输层的PDU叫数据报" | ⚠️ | TCP的叫**报文段**(Segment)，UDP的叫**用户数据报**(Datagram) |

## 3. 发送时延 vs 传播时延

| 对比维度 | 发送时延 | 传播时延 |
|:---------|:---------|:---------|
| 取决于 | 数据量 + 发送速率 | 距离 + 信号速率 |
| 比喻 | 水管口径 × 水量 | 水管长度 |
| 可否用高速链路缩短 | ✅ 可以 | ❌ 不能（受光速限制） |
| 典型量级 | ms级（数据量大时可达秒级） | μs～ms级 |

### 易混淆场景

| 场景 | 谁更大 |
|:-----|:-------|
| 近距离（机房内）+ 大数据量 | 发送时延 >> 传播时延 |
| 远距离（卫星）+ 小数据量 | 传播时延 >> 发送时延 |
| 光纤跨洋 + 大文件 | 两者都可能很大 |

### 判断题

| 陈述 | 答案 |
|:-----|:----|
| "数据量越大，传播时延越大" | ❌ 传播时延与数据量无关 |
| "发送速率越高，发送时延越小" | ✅ |
| "传播时延可以通过增大带宽来减小" | ❌ 传播时延与带宽无关 |
| "使用光纤代替铜线，传播时延减半" | ❌ 光纤中光速约2×10⁸，铜线约2.3×10⁸，差不多 |

## 4. OSI七层 vs TCP/IP四层

### 层数别记反

```text
OSI   : 7层（物 链 网 运 会 表 应）
TCP/IP: 4层（网络接口、网际、运输、应用）
教学  : 5层（物 链 网 运 应）
```

### 易错

| 陈述 | 答案 |
|:-----|:----|
| "OSI模型有5层" | ❌ 7层 |
| "TCP/IP模型有5层" | ❌ 通用说法是4层（五层是教学模型） |
| "表示层和会话层是TCP/IP的层" | ❌ 只在OSI里才有 |
| "OSI最高层是运输层" | ❌ 最高层是**应用层** |

## 5. 电路交换 vs 报文交换 vs 分组交换

| 特征 | 电路交换 | 报文交换 | 分组交换 |
|:-----|:---------|:---------|:---------|
| 建立连接 | 需要 | 不需要 | 不需要 |
| 独占资源 | 是 | 否 | 否 |
| 时延 | 固定（建立后） | 可变 | 可变 |
| 灵活性 | 低 | 中 | 高 |
| 代表 | 传统电话 | 电报 | 互联网 |

### 为什么互联网用分组交换

1. **统计复用**：不独占资源，多个用户共享链路
2. **灵活**：分组可走不同路由
3. **容错**：某条链路故障时分组可绕行
4. **适合突发数据**：计算机通信通常是间歇性的

### 分组交换的时延来源

```text
总时延 = 处理时延 + 排队时延 + 发送时延 + 传播时延
```

其中**排队时延**是分组交换特有的（电路交换没有）。

## 6. 网络分类核心区分

### LAN vs WAN

| 维度 | LAN | WAN |
|:-----|:----|:----|
| 覆盖范围 | 小（楼内/校园） | 大（城市/国家） |
| 所有者 | 通常单一组织 | 通常是服务商 |
| 技术 | 以太网为主 | 多种（帧中继、ATM、MPLS） |
| 广播 | 支持 | 不支持 |

**关键点**：判断LAN/WAN的首要标准不是地理范围，而是**使用的技术**。校园网即使跨多栋楼仍是LAN。

### 集线器 vs 交换机 vs 路由器

| 设备 | 工作层 | 隔离冲突域 | 隔离广播域 | 转发依据 |
|:-----|:------|:----------|:----------|:---------|
| 集线器(Hub) | 物理层 | ❌ | ❌ | 向所有端口转发 |
| 交换机(Switch) | 数据链路层 | ✅ | ❌ | MAC地址表 |
| 路由器(Router) | 网络层 | ✅ | ✅ | IP路由表 |

## 7. 封装过程：向上还是向下

### 容易搞反的场景

```
发送方：从上往下（应用→物理），每层加首部
接收方：从下往上（物理→应用），每层去首部
```

### 易错判断题

| 陈述 | 答案 | 解释 |
|:-----|:----|:-----|
| "发送方数据链路层收到网络层的PDU后，添加本层首部" | ✅ | 向下封装 |
| "接收方物理层将数据交给网络层" | ❌ | 物理层交给**数据链路层**，逐层向上 |
| "路由器在转发分组时，会解封装到应用层" | ❌ | 路由器只解封到**网络层** |

## 8. 408常考一句话判断汇总

| 陈述 | 答案 |
|:-----|:----|
| 网络提供面向连接的服务就意味着通信可靠 | ❌ 面向连接≠可靠，如ATM面向连接但可能丢信元 |
| 分组交换一定比电路交换慢 | ❌ 分组交换在轻载时可能更快（无需连接建立） |
| 时延带宽积就是排队时延 | ❌ 时延带宽积=传播时延×带宽 |
| 物理层是中继器和集线器的所在层 | ✅ |
| 运输层PDU称为数据报 | ⚠️ TCP的PDU是Segment，UDP的PDU是Datagram |
| OSI的上三层是会话层、表示层、应用层 | ✅ |
| TCP/IP的网络接口层对应OSI的数据链路层和物理层 | ✅ |
| 交换机隔离广播域 | ❌ 路由器才隔离广播域 |

## 9. 自查清单

闭卷回答以下问题：

1. PDU的英文全称是什么？中文叫什么？
2. SDU和PDU的关系是什么？
3. 发送方封装时第N层的PDU变成第N-1层的什么？
4. OSI有几层？TCP/IP有几层？
5. 集线器、交换机、路由器各工作在哪一层？
6. 哪个设备隔离广播域？
7. 分组交换比电路交换多了哪种时延？

<details>
<summary>答案</summary>

1. Protocol Data Unit，协议数据单元
2. 本层PDU = 本层PCI + 本层SDU（其中SDU来自上层PDU）
3. 第N层PDU = 第N-1层SDU
4. OSI 7层，TCP/IP 4层
5. 集线器=物理层，交换机=数据链路层，路由器=网络层
6. 路由器
7. 排队时延

</details>
