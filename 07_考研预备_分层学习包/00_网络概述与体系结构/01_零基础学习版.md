## 1. 什么是计算机网络

### 一句话定义

> 计算机网络 = 用通信线路把独立自治的计算机系统连起来，按协议实现资源共享和数据通信。

两个关键词：**自治**（每台主机独立运作）和**互联**（通过通信链路连接）。

### 两种视角理解网络

| 视角 | 核心问题 | 回答 |
|:-----|:---------|:-----|
| 组成角度 | "网络由什么构成？" | 端系统（主机） + 分组交换机（路由器/交换机） + 通信链路 |
| 服务角度 | "网络能做什么？" | 为分布式应用提供通信服务的基础设施 |

### 网络的三大功能

1. **数据通信** — 最基本的功能，把数据从A送到B
2. **资源共享** — 硬件（打印机）、软件、数据
3. **分布式处理** — 多台计算机协同完成一个任务

## 2. 网络的分类

### 按覆盖范围

| 类型 | 覆盖范围 | 典型场景 |
|:-----|:---------|:---------|
| **个域网 PAN** | 10m左右 | 蓝牙耳机连手机 |
| **局域网 LAN** | 几米～几公里 | 教室、宿舍、公司内部 |
| **城域网 MAN** | 几十公里 | 一个城市 |
| **广域网 WAN** | 几百～几千公里 | 互联网 |

**考试陷阱**：局域网不一定范围小！一个分布在多个楼内的校园网仍然是LAN（因其使用局域网技术）。判断LAN/WAN的关键是**使用的技术**，不全是地理范围。

### 按拓扑结构

```
星型    ── 中心节点连接所有主机（以太网交换机）
总线型  ── 所有主机共享一根总线（传统以太网）
环型    ── 主机首尾相连成环（令牌环）
网状    ── 任意节点间可能直连（广域网）
树型    ── 星型的扩展，分级连接
```

### 按交换方式

| 交换方式 | 原理 | 特点 |
|:---------|:-----|:-----|
| **电路交换** | 建立独占物理连接 | 通话期间独占，适合电话网 |
| **报文交换** | 整条报文存储转发 | 不需建立连接，延迟大 |
| **分组交换** | 数据分组后逐跳转发 | 灵活高效，互联网的基础 |

## 3. 分层结构：为什么要分层

### 分层的四个好处

1. **模块化**：复杂问题拆成小问题
2. **独立演进**：每层内部改变不影响其他层
3. **标准化**：层间接口固定，不同厂商可各自实现
4. **复用**：上层直接使用下层提供的服务

### 形象比喻：寄快递

```
你写地址封箱    ← 应用层：决定"寄什么内容"
快递公司分拣   ← 运输层：保证"完整到达"
转运中心规划路线 ← 网络层：决定"走哪条路"
卡车逐站运输   ← 数据链路层：逐段传送
公路/铁路本身   ← 物理层：实际的物理介质
```

## 4. 五层体系结构（考试标准）

### 五层及各自功能

| 层号 | 名称 | 功能一句话 | PDU名称 |
|:-----|:-----|:-----------|:--------|
| 5 | **应用层** | 为用户应用程序提供网络服务 | 报文 Message |
| 4 | **运输层** | 进程到进程的通信 | 报文段 Segment |
| 3 | **网络层** | 主机到主机的通信，路由选择 | 分组/数据报 Packet |
| 2 | **数据链路层** | 相邻节点之间的帧传输 | 帧 Frame |
| 1 | **物理层** | 比特流的透明传输 | 比特 Bit |

### 必须能默写

```text
应 → 传 → 网 → 链 → 物
（应用层→运输层→网络层→数据链路层→物理层）

口诀：应传网链物
```

### OSI七层与TCP/IP四层对比

```text
OSI七层                TCP/IP四层        五层教学模型
───────               ─────────         ────────────
应用层 ─┐                              应用层
表示层 ─┤── 应用层     应用层            ───────
会话层 ─┘                              运输层
运输层 ─── 运输层     运输层            ───────
网络层 ─── 网络层     网际层            网络层
数据链路层             网络接口层        数据链路层
物理层 ─┘                              物理层
```

## 5. 协议、接口、服务：三个必须区分的概念

### 核心定义

| 概念 | 定义 | 一句话 |
|:-----|:-----|:-------|
| **协议** | 对等层之间通信规则的集合 | "同层之间怎么说话" |
| **接口** | 相邻层之间交换信息的地方 | "上下层之间怎么交数据" |
| **服务** | 下层通过接口向上层提供的功能 | "下层给上层做什么事" |

### 协议三要素

| 要素 | 含义 | 类比电话 |
|:-----|:-----|:---------|
| **语法** | 数据格式、编码方式 | 拨号号码的格式（区号+号码） |
| **语义** | 各部分数据的含义 | 振铃声="等待接听"，忙音="对方正在通话" |
| **时序** | 事件发生的顺序 | 先拨号→等振铃→对方摘机→开始通话 |

### 服务访问点 SAP

> SAP（Service Access Point）= 相邻两层实体交换信息的逻辑位置。

| 层 | SAP |
|:---|:----|
| 运输层 → 应用层 | 端口号（Port） |
| 网络层 → 运输层 | 协议号（Protocol Number） |
| 数据链路层 → 网络层 | 类型字段（EtherType） |
| 物理层 → 数据链路层 | 网络接口卡（NIC） |

## 6. 封装与解封装

### 发送方：从上到下加包头

```text
应用层数据
  ↓ 运输层加 TCP/UDP 首部
运输层报文段
  ↓ 网络层加 IP 首部
IP数据报
  ↓ 数据链路层加帧头、帧尾
帧
  ↓ 物理层
比特流
```

### 接收方：从下到上去包头

```text
比特流
  ↓ 物理层
帧 → 去帧头帧尾
  ↓ 网络层
IP数据报 → 去IP首部
  ↓ 运输层
运输层报文段 → 去TCP/UDP首部
  ↓ 应用层
应用层数据
```

### 各层协议数据单元（PDU）汇总

| 层 | PDU名称 | 别称 |
|:---|:--------|:-----|
| 应用层 | Message | 报文 |
| 运输层 | Segment (TCP) / Datagram (UDP) | 报文段 / 用户数据报 |
| 网络层 | Packet / Datagram | 分组 / IP数据报 |
| 数据链路层 | Frame | 帧 |
| 物理层 | Bit | 比特 |

## 7. 各层主要设备

| 层 | 设备 | 操作单元 | 功能 |
|:---|:-----|:---------|:-----|
| 物理层 | 中继器、集线器(Hub) | 比特 | 信号放大/再生 |
| 数据链路层 | 交换机、网桥 | 帧 | 按MAC地址转发帧 |
| 网络层 | 路由器 | 分组/IP数据报 | 选择路径，转发分组 |
| 应用层 | 网关 | 报文 | 协议转换，连接异构网络 |

### 关键结论

- **交换机**隔离**冲突域**，但不隔离**广播域**
- **路由器**隔离**冲突域**和**广播域**
- **集线器**既不隔离冲突域也不隔离广播域

<details>
<summary>答案</summary>

2. 帧 Frame
3. 语法、语义、时序
4. 协议是**对等层**之间通信的规则；服务是**下层通过接口向上层**提供的功能
5. 网络层
6. 不隔离（路由器才隔离广播域）
7. 从上到下：应用层 → 运输层 → 网络层 → 数据链路层 → 物理层，每层加包头
