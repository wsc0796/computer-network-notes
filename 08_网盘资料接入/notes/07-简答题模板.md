# 07 简答题模板

## 1. 五层模型与封装

```text
应用层产生报文；
运输层加TCP/UDP首部形成报文段/用户数据报；
网络层加IP首部形成IP数据报；
数据链路层加帧头帧尾形成帧；
物理层把帧转换为比特流传输。
```

## 2. TCP三次握手

```text
客户端发送SYN；
服务器回复SYN+ACK；
客户端回复ACK。
三次握手可以确认双方收发能力，并防止失效连接请求导致服务器误建立连接。
```

## 3. TCP四次挥手

```text
主动关闭方发送FIN；
被动关闭方回复ACK；
被动关闭方数据发完后发送FIN；
主动关闭方回复ACK并进入TIME_WAIT。
四次挥手的原因是TCP全双工通信，两个方向需要分别关闭。
```

## 4. 流量控制与拥塞控制

```text
流量控制是防止发送方发送过快，导致接收方缓冲区溢出，核心依据是接收窗口rwnd。
拥塞控制是防止过多数据进入网络，导致路由器拥塞和丢包，核心依据是拥塞窗口cwnd。
实际发送窗口取 min(rwnd, cwnd)。
```

## 5. DNS解析过程

```text
浏览器先查缓存；
未命中则询问本地DNS服务器；
本地DNS再向根DNS、顶级域DNS、权威DNS逐级查询；
最终得到目标主机IP地址并返回给客户端。
```

## 6. ARP过程

```text
主机先查ARP缓存；
若没有目标IP对应MAC，则在局域网广播ARP请求；
目标主机单播ARP响应；
发送方写入ARP缓存。
跨网通信时，ARP查询的是默认网关MAC。
```
