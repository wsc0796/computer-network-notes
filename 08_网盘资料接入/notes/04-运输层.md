## TCP vs UDP

| 对比 | TCP | UDP |
|---|---|---|
| 连接 | 面向连接 | 无连接 |
| 可靠性 | 可靠传输 | 尽力而为 |
| 数据形式 | 字节流 | 报文 |
| 典型场景 | HTTP、文件传输 | DNS、实时音视频 |

## 三次握手

```text
1. 客户端 -> 服务器：SYN, seq=x
2. 服务器 -> 客户端：SYN+ACK, seq=y, ack=x+1
3. 客户端 -> 服务器：ACK, ack=y+1
```

为什么不是两次：

```text
防止历史失效SYN导致服务器误建立连接；
确认双方发送和接收能力。
```

## 四次挥手

```text
1. 主动关闭方发送 FIN
2. 被动关闭方回复 ACK
3. 被动关闭方发送 FIN
4. 主动关闭方回复 ACK，进入 TIME_WAIT
```

TIME_WAIT作用：

```text
保证最后一个ACK能到达；
让旧连接的残留报文自然消失。
```

## 可靠传输与控制机制

```text
可靠传输：序号、确认、超时重传、滑动窗口。
流量控制：保护接收方，核心变量 rwnd。
拥塞控制：保护网络，核心变量 cwnd。
发送窗口 = min(rwnd, cwnd)。
```

拥塞控制：

```text
慢开始：cwnd指数增长。
拥塞避免：cwnd线性增长。
快重传：收到重复ACK后尽快重传。
快恢复：避免直接回到慢开始的最低点。
```
