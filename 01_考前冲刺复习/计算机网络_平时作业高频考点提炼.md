# 2. 分层模型与协议/服务

## 2.1 协议与服务区别

协议是控制两个对等实体进行通信的规则集合，包含语法、语义和时序。服务是下层通过层间接口向上层提供的功能。

一句话：

```text
协议是“同层之间怎么说话”，服务是“下层给上层做什么”。
```

## 2.2 PDU

PDU 是协议数据单元，是对等层实体之间交换的数据单位。

| 层次 | PDU |
|---|---|
| 应用层 | 报文 Message |
| 传输层 | 报文段 Segment / 用户数据报 |
| 网络层 | 分组 Packet / 数据报 Datagram |
| 数据链路层 | 帧 Frame |
| 物理层 | 比特 Bit |

作业里明确考了：

```text
数据链路层的PDU是帧 Frame。
```

## 2.3 OSI与TCP/IP

OSI七层：

```text
物理层、数据链路层、网络层、传输层、会话层、表示层、应用层
```

TCP/IP四层：

```text
应用层、传输层、网际层、网络接口层
```

# 3. 时延与传输效率

## 3.1 四种时延

| 时延 | 含义 | 公式 |
|---|---|---|
| 处理时延 | 节点处理分组所需时间 | 通常题目给出 |
| 排队时延 | 分组排队等待转发时间 | 通常题目给出 |
| 发送时延/传输时延 | 把数据全部推到链路上的时间 | 数据长度 / 发送速率 |
| 传播时延 | 电磁波在链路上传播的时间 | 距离 / 传播速率 |

## 3.2 作业原型题

题型：

```text
收发两端距离1000km，信号传播速度2×10^8 m/s，数据长度10^7 bit，发送速率100 kbit/s。
求发送时延和传播时延。
```

答案：

```text
发送时延 = 10^7 / (100×10^3) = 100s
传播时延 = 1000km / (2×10^8m/s)
        = 10^6m / (2×10^8m/s)
        = 0.005s
```

## 3.3 传输效率

作业题型：

```text
应用层数据400B，每层增加首部20B，若有若干层首部，求有效数据效率。
```

常用公式：

```text
效率 = 有效数据 / 总发送数据 × 100%
```

例：

```text
有效数据400B，总数据400+20+20+60 = 500B
效率 = 400 / 500 = 80%
```

# 4. 数据链路层

## 4.1 数据链路层三个基本问题

```text
封装成帧
透明传输
差错检测
```

解释：

```text
封装成帧：给网络层数据加帧头帧尾，形成帧。
透明传输：无论数据中出现什么比特组合，都能作为普通数据传输。
差错检测：检测传输过程中是否出现差错，常用CRC。
```

## 4.2 透明传输

作业考点：

```text
在数据中遇到控制字符时，必须采取适当措施，使接收方不会把数据误认为控制信息。
这体现透明传输。
```

常用方法：

```text
字符填充、比特填充。
```

## 4.3 PPP协议

PPP是点到点协议，常用于点到点链路。它支持封装成帧、透明传输和差错检测，但不提供可靠传输。

判断题：

```text
PPP协议使数据链路层实现可靠传输。（错）
```

# 5. CRC 差错检测

## 5.1 基本思想

发送方把数据看成二进制多项式，用生成多项式进行模2除法，得到余数作为CRC校验码。接收方用同样生成多项式检查，若余数不为0，说明可能出错。

## 5.2 做题步骤

```text
1. 生成多项式最高次为r，就在数据后补r个0。
2. 用生成多项式对应的二进制数进行模2除法。
3. 得到r位余数。
4. 发送数据 = 原数据 + CRC余数。
```

## 5.3 作业原型

题型：

```text
数据为1011110，生成多项式 P(x)=x^3+1。
求CRC余数。
```

答案：

```text
P(x)=x^3+1 对应二进制 1001。
最高次为3，因此数据后补3个0：1011110000。
用1001做模2除法，余数为011。
```

## 5.4 另一个作业结论

```text
二进制位串与生成多项式 x^5+x^3+x^2+1 对应的位串为：101101。
```

# 6. 局域网、CSMA/CD、以太网

## 6.1 局域网层次

作业结论：

```text
局域网通常只包含物理层和数据链路层，不包含网络层。
```

## 6.2 介质访问控制

| 方法 | 含义 |
|---|---|
| CDMA | 码分多址 |
| TDMA | 时分多址 |
| FDMA | 频分多址 |
| CSMA | 载波监听多路访问 |
| CSMA/CD | 载波监听多路访问/冲突检测 |

作业结论：

```text
带冲突检测的载波监听多路访问技术是 CSMA/CD。
可能发生冲突的是 CSMA。
```

## 6.3 CSMA/CD争用期

考点：

```text
最坏情况下，一个站发现冲突所需时间为2t。
```

其中 t 是端到端传播时延。

## 6.4 以太网

作业结论：

```text
快速以太网：100Base-T
以太网使用广播式总线型拓扑。
以太网数据字段最小有效长度：46字节。
```

## 6.5 MAC地址

```text
MAC地址是数据链路层地址，长度为48位。
```

注意：

```text
网卡唯一标识符是MAC地址，不是IP地址。
```

## 6.6 交换机与网桥

| 设备 | 层次 | 依据 |
|---|---|---|
| 网桥 | 数据链路层 | MAC地址 |
| 交换机 | 数据链路层 | MAC地址表 |
| 路由器 | 网络层 | IP地址 |

作业结论：

```text
交换机的MAC地址表是在数据转发过程中动态学习建立的。
```
