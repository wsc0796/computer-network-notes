# 1. 时延与交换方式计算

## 1.0 四种时延

```text
发送时延   = 数据长度 ÷ 发送速率         ← 把数据"推"到链路上的时间
传播时延   = 链路距离 ÷ 信号传播速率       ← 信号在线缆/光纤上"跑"的时间
排队时延   = 分组在路由器缓存中等待的时间   ← 取决于网络拥塞程度
处理时延   = 路由器查表、转发的时间         ← 通常很小，考试常忽略

总时延 = 发送时延 + 传播时延 + 排队时延 + 处理时延
```

## 1.1 三种交换方式通用公式

```text
设：经过 k 条链路，报文长度 L bit，分成 n 个等长分组（含首部），
    每个分组 P bit，链路速率均为 R，每条链路传播时延 Tp。

报文交换（每跳收完再发）：总时间 = k × L/R + k × Tp
电路交换（先建连，再发送）：总时间 = 建连时延 + L/R + k × Tp
分组交换（流水线）：      总时间 = (k + n - 1) × P/R + k × Tp
```

> **本题按通信计算常用口径 1 KB = 1000 Byte。若题目明确 1 KB = 1024 Byte，按题目要求。**

## 1.2 例题

**题 1-1（基础时延）：**
> 数据长度 10 KB，发送速率 1 Mbps，链路距离 1000 km，传播速率 2×10⁸ m/s。
> 求发送时延和传播时延。

```text
发送时延 = (10 × 8 × 1000) bit ÷ (1 × 10⁶) bps = 80000 ÷ 10⁶ = 0.08 s = 80 ms

传播时延 = (1000 × 10³) m ÷ (2 × 10⁸) m/s = 5 × 10⁻³ s = 5 ms
```

---

**题 1-2（分组交换—通用公式验证）：**
> 报文长 6000 字节，分成 2 个分组，每经过一个路由器存储转发。
> 链路速率 1 Mbps，传播时延每条链路 2 ms。
> 经过 2 条链路（3 节点），求总传输时间。

```text
每个分组长度 = 6000 ÷ 2 = 3000 字节 = 24000 bit
每跳发送时延 P/R = 24000 ÷ 10⁶ = 24 ms

用通用公式（k=2 条链路，n=2 个分组）：
总时间 = (k + n - 1) × P/R + k × Tp
       = (2 + 2 - 1) × 24 + 2 × 2
       = 72 + 4 = 76 ms
```

---

**题 1-3（含首部开销 + 多速率链路）：**
> 报文 6000 Byte，分成 4 个分组，每个分组首部 20 Byte。
> 链路 1 速率 1 Mbps，链路 2 速率 2 Mbps，传播时延各 2 ms。
> 求总时间。

```text
总数据 = 6000 Byte，4 个分组，首部共 4×20 = 80 Byte
分组数据总长 = 6000 + 80 = 6080 Byte，每分组 = 1520 Byte = 12160 bit

链路1发送时延：12160 ÷ 1×10⁶ = 12.16 ms
链路2发送时延：12160 ÷ 2×10⁶ = 6.08 ms

注意：不能直接用统一速率！第二条链路更快，瓶颈在第一条。

总发送时延 ≈ (k + n - 1) × 12.16 + (n-1) × 6.08  ← 取瓶颈为准
         ≈ (2+4-1)×12.16 + 2×2 ≈ 60.8 + 4 ≈ 64.8 ms

实际中更精确的计算需要逐分组分析每跳的发送结束时间。
```

---

**题 1-4（有效传输效率）：**
> 以太网帧数据字段最小 46 字节，最大 1500 字节。帧首部+尾部共 18 字节。
> 传输 46 字节数据时，有效传输效率是多少？

```text
总帧长 = 46 + 18 = 64 字节
有效数据 = 46 字节
传输效率 = 46 ÷ 64 ≈ 71.9%
```

# 2. CSMA/CD 计算题

## 2.0 核心公式

```text
帧发送时间 ≥ 2 × 单程传播时延（争用期）

即：最小帧长 / 发送速率 ≥ 2 × 距离 / 传播速率

变形公式：
  最小帧长 = 2 × 距离 × 发送速率 / 传播速率
  最大距离 = 最小帧长 × 传播速率 / (2 × 发送速率)
```

## 2.1 例题

**题 2-1（求最小帧长）：**
> 以太网速率 10 Mbps，最远两端距离 2500 m，传播速率 2×10⁸ m/s。
> 求最小帧长。

```text
单程传播时延 = 2500 ÷ (2×10⁸) = 1.25 × 10⁻⁵ s = 12.5 μs

争用期（往返时间） = 2 × 12.5 = 25 μs

最小帧长 = 25 × 10⁻⁶ × 10 × 10⁶ = 250 bit ≈ 31.25 字节

实际最小帧长取 64 字节（512 bit），大于理论值，留有余量。
```

---

**题 2-2（求最大距离）：**
> 以太网速率 100 Mbps，最小帧长 64 字节（512 bit），传播速率 2×10⁸ m/s。
> 求最大网络距离。

```text
公式：距离 ≤ 最小帧长 × 传播速率 / (2 × 发送速率)

距离 ≤ 512 × 2×10⁸ / (2 × 100×10⁶) = 512 × 2×10⁸ / 2×10⁸ = 512 m

实际以太网标准距离 < 计算值，因为还有中继器延迟等因素。
```

---

## 2.2 CSMA/CD 简答模板

```text
CSMA/CD 基本过程：
① 先听后发：站点发送前先监听信道，忙则等待，空闲则发送。
② 边发边听：发送过程中持续检测碰撞。
③ 碰撞停止：若检测到碰撞，立即停止发送，发阻塞信号。
④ 随机退避：按退避算法等待随机时间后重试。

争用期 = 2 倍端到端传播时延。
帧发送时间不得小于争用期：保证最远端发生碰撞时，
发送站仍在发送，从而能够检测到碰撞。
简言之：帧发送时间 ≥ 争用期（不能过早发完）。
```

# 3. TCP 序号与确认号计算

## 3.0 核心规则

```text
① 确认号 ack = 期望收到的下一个字节序号
                = 已连续收到的最后一个字节序号 + 1

② SYN 占用一个序号，FIN 占用一个序号
   纯 ACK（不带数据、不带SYN/FIN）不消耗序号

③ 发送数据后，下一段的 seq = 当前 seq + 已发送数据字节数
```

## 3.1 正常传输例题

**题 3-1：**
> 主机 A 向 B 发送一个 TCP 段，seq=100，数据长度 500 字节。
> B 正确接收后回复 ACK，问 ack 是多少？
> A 紧接着发下一段，seq 是多少？

```text
B 收到的最后一个字节序号 = 100 + 500 - 1 = 599
ack = 599 + 1 = 600  （期望下一个字节是 600）

A 下一段的 seq = 100 + 500 = 600
```

---

**题 3-2（累计确认）：**
> A 先后发送三段：
> seq=100, 数据 200 字节
> seq=300, 数据 200 字节
> seq=500, 数据 200 字节
>
> 第二段丢失，B 收到第一和第三段后回复什么 ACK？

```text
第一段到达：B 回复 ack=300（期望 300）
第三段到达：300~499 的缺口还在，TCP 是累计确认
           → B 累计确认号仍为 300，发送重复 ACK 300
           → 若支持 SACK，可在选项中额外告知已收到 500~699 的乱序数据
```

---

## 3.2 三次握手 & 四次挥手序号题

**题 3-3（三次握手）：**
> 客户端发 SYN(seq=x=100)，服务器回 SYN+ACK(ack=?, seq=y=500)。
> 客户端最后回 ACK(ack=?)。

```text
服务器回复：ack = x+1 = 101
客户端最后：ack = y+1 = 501
```

---

**题 3-4（四次挥手序号）：**
> 主动方发 FIN(seq=u=200)，被动方回 ACK(ack=?)
> 被动方最后发 FIN(seq=v=800, ack=?)  ← 中间还发了数据

```text
被动方回第一个 ACK：ack = u+1 = 201
被动方发 FIN：seq=v=800, ack=u+1=201（未变）
主动方最后回复 ACK：
  seq=u+1=201, ack=v+1=801
```

**关键规则：**
```text
TCP 双方各有独立的序号空间。主动方的 seq 和被动方的 seq 不能累加。

通用公式：
  下一序号 = 当前seq + 数据字节数 + SYN(占1) + FIN(占1)
```

# 4. GBN/SR 窗口与信道利用率

## 4.0 ACK 编号约定（必须在做题前明确）

```text
本文件统一采用：ACK n 表示"帧 n 及之前的所有帧均已正确收到"。
即 ACK 确认的是累计已收到的最高序号帧。
不同教材可能使用不同约定，考试以题目说明为准。
```

## 4.1 信道利用率公式

```text
停止等待：U = T帧 / (T帧 + 2×T传播 + T_ACK)
          忽略 ACK 发送时延时：U ≈ T帧 / (T帧 + 2×T传播)

滑动窗口：U = min(1, W × T帧 / (T帧 + 2×T传播))
          即：窗口内填满了管道 → U=1；填不满 → U 正比于窗口大小
```

## 4.2 例题

**题 4-1（停止等待信道利用率）：**
> 帧长 1000 Byte，发送速率 1 Mbps，单程传播时延 20 ms。
> ACK 帧发送时延忽略。求停止等待协议的信道利用率。

```text
帧发送时延 T帧 = 1000×8 ÷ 10⁶ = 8 ms

U ≈ 8 / (8 + 2×20) = 8 / 48 ≈ 16.7%
```

---

**题 4-2（GBN 窗口最大利用率）：**
> 帧发送时延 1 ms，传播时延 250 ms（RTT=500 ms）。
> 序号 3 位（n=3），用 GBN。求能达到的最大信道利用率。

```text
GBN 最大窗口：W_max = 2³ - 1 = 7
每个发送+等待周期 = 1 + 500 = 501 ms
窗口 7 帧的总发送时间 = 7 ms

利用率 = 7 / 501 ≈ 1.4%

若要填满管道（U=100%），至少需要窗口：
  W ≥ (1 + 500) / 1 = 501
但 3 位序号最多 7 帧，远不够 → 利用率极低
```

---

---

**题 4-3（GBN 超时重传范围）：**
> GBN 发送窗口=4，当前已发送帧 3,4,5,6（均未确认）。
> 收到 ACK3 后，帧 4 超时。问重传哪些帧？

```text
ACK3 含义：帧 0~3 均已正确收到。
当前最早未确认帧 = 4。
帧 4 超时 → 重传帧 4,5,6（从最早未确认帧开始全部重传）。
```

---

**题 4-4（GBN 窗口滑动——还能发多少帧）：**
> GBN 序号 0~7 循环，发送窗口=4。当前窗口 [2,3,4,5]。
> 已发送 2,3,4，收到 ACK2。问窗口如何滑动，还能发送哪些帧？

```text
ACK2 确认帧 2 → 窗口滑动一位
新窗口：[3,4,5,6]
已发送未确认：3,4 → 可以发 5,6
```



**题 4-5（SR 窗口大小限制）：**
> 序号 3 位（n=3），采用 SR 协议，等长窗口。求最大发送窗口。

```text
SR 等长窗口：W_s = W_r ≤ 2^(n-1) = 2² = 4

若窗口取 5（超过 4），序号只有 0~7 循环 8 个值，
新旧帧序号会重叠，接收方无法区分重传旧帧和新轮次新帧。

所以 SR 最大窗口 = 4  ← 序号空间的一半
```

---

**题 4-6（SR 丢帧重传范围判断）：**
> SR 协议，发送窗口=4（序号 0~7），当前窗口 [3,4,5,6]。
> 帧 4 超时，哪些帧会被重传？

```text
SR：只重传帧 4（每个帧独立计时器），帧 3、5、6 不受影响。
这是 SR 和 GBN 最核心的区别。
```

# 5. 交换机转发表分析题

## 5.0 三步法

```text
每收到一帧，严格按顺序：
① 学习：记下【源MAC → 入端口】
② 查表：查【目的MAC】对应的端口
    - 找到且不是入端口 → 从该端口转发
    - 找到但是入端口 → 丢弃（过滤）
    - 没找到 → 泛洪（除入端口外的所有端口）
```

## 5.1 例题

**题 5-1：**
> 交换机端口 1 接主机 A，端口 2 接 B，端口 3 接 C。初始 MAC 表为空。
> 依次发生：
> (a) A 向 B 发送帧
> (b) B 向 A 发送帧
> (c) C 向 B 发送帧
> 填写每一步的行为。

```text
(a) A→B：
    学习：A → 端口 1
    查目的 B：未知 → 向端口 2、3 泛洪

(b) B→A：
    学习：B → 端口 2
    查目的 A：端口 1 → 从端口 1 转发（单播）

(c) C→B：
    学习：C → 端口 3
    查目的 B：端口 2 → 从端口 2 转发（单播）

最终 MAC 表：A→1, B→2, C→3
```

---

**题 5-2（广播帧）：**
> MAC 表已有 A→1, B→2, C→3。A 向广播地址 FF-FF-FF-FF-FF-FF 发送帧。
> 交换机如何处理？

```text
广播帧的目的 MAC 是全 F（广播地址），交换机按广播转发规则，
向除入端口 1 外的所有端口（2、3）泛洪。
与"未知单播泛洪"结果相同，但原因不同——
广播因目的地址类型决定，未知单播因 MAC 表查不到决定。
```

---

**题 5-3（同端口过滤）：**
> 交换机端口 1 接集线器，主机 A 和 D 都连在该集线器上。
> MAC 表中 A→端口1, D→端口1。A 向 D 发送帧。

```text
学习：源 MAC A → 端口 1（已有，刷新计时器）
查目的 D → 端口 1，与入端口相同 → 过滤，不转发
```

---

## 5.2 多交换机级联题

**题 5-4（两台交换机）：**
> S1 的端口1接A，端口2接S2的端口1。
> S2 的端口1接S1，端口2接B。
> 初始 MAC 表均空。依次发生：A→B, B→A。

```text
(a) A→B：
  S1：A→端口1（学习），目的B未知→泛洪到端口2
  S2：A→端口1（学习，从S1来），目的B未知→泛洪到端口2
  B 收到帧

(b) B→A：
  S2：B→端口2（学习），目的A→端口1→从端口1单播给S1
  S1：B→端口2（学习，从S2来），目的A→端口1→从端口1单播给A

最终：
  S1表：A→1, B→2
  S2表：A→1, B→2
```

## 5.3 CSMA/CD 退避算法

```text
二进制指数退避：
  第 i 次碰撞后，k = min(i, 10)
  从 [0, 2^k - 1] 中随机选整数 r
  等待时间 = r × 争用期
  最多重试 16 次，仍失败则放弃并向上层报告。

例：第 3 次碰撞 → k=min(3,10)=3 → r∈[0,7]
    等待 r 个争用期后重发
```

# 6. 速背卡

```text
┌──────────────┬──────────────────────────────────────────────┐
│ 时延         │ 发送时延=数据/速率；传播时延=距离/传播速率        │
│ 分组流水线   │ k条链路、n个分组：总发送阶段=(k+n-1)×P/R       │
│ 停止等待     │ U≈T帧/(T帧+2T传播)，ACK时延不忽略时要加入       │
│ CSMA/CD      │ 帧发送时间 ≥ 争用期 = 2t                     │
│              │ 最小帧长 = 2×距离×速率/传播速率                │
│ TCP 确认     │ ack=最后字节+1；SYN/FIN各占一个序号            │
│ TCP 下一序号 │ seq新=seq旧+数据长度+SYN+FIN                   │
│ 三次握手     │ seq=x(→), ack=x+1(←), ack=y+1(→)            │
│ GBN          │ W≤2ⁿ-1，累计确认，丢弃乱序，超时从最早未确认全重传 │
│ SR           │ W≤2ⁿ⁻¹，缓存乱序，独立ACK/计时器，只重传超时帧    │
│ 交换机       │ 源MAC学习，目的MAC查表；未知单播与广播均泛洪       │
│              │ 但前者因查不到，后者因广播地址规则                │
└──────────────┴──────────────────────────────────────────────┘
```
