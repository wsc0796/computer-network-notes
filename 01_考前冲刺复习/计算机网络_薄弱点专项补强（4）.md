# 1. 物理层：编码识图与计算

## 1.1 为什么物理层会是薄弱点

现有资料只给了奈奎斯特/香农公式，但2025江西农大B卷明确出现了**NRZ/曼彻斯特编码识图**和**10Base含义**。物理层编码题不需要计算，关键在**看图辨类型**。

## 1.2 四种数字编码（考试核心）

数字数据变成数字信号，有四种编码方式。记住它们的**波形特征**：

### NRZ（非归零编码）

```
规则：高电平=1，低电平=0，整个位周期保持不变

位序列：   1    0    1    1    0    0    1
           ┌─┐      ┌─┐  ┌─┐
           │ │      │ │  │ │
     ──────┘ └──────┘ └──┘ └──────┐
                                   │
                                   └──────

特征：不归零，遇到连续0或1波形不变。
```

考试判据：
```text
NRZ：看到连续相同的位，波形是一条水平线。
     连续8个1 → 8个位周期的高电平直线。
```

| 优点 | 缺点 |
|------|------|
| 实现简单 | 无自同步能力（连续0/1时接收方不知道这是几个位） |

### NRZ-I（非归零 inverted / 见1就翻）

```
规则：每个位开始处：如果是1就翻转电平，如果是0就保持

位序列：   1    0    1    1    0    0    1
     ──┐  ┌──┐     ┌─┐              ┌──┐
       │  │  │     │ │              │  │
       └──┘  └─────┘ └──────────────┘  └───

特征：只看"有没有跳变"就知道是1还是0。有跳变=1，没跳变=0。
```

考试判据：
```text
NRZ-I：找"跳变"。在位边界处，电平变了 → 这一位是1；电平没变 → 这一位是0。
```

| 优点 | 缺点 |
|------|------|
| 差分编码，抗极性反转 | 连续0时仍无跳变，同步可能丢失 |

### 曼彻斯特编码（Manchester）

```
规则：每个位的正中间必有一次跳变
      中间：高→低 表示 1
      中间：低→高 表示 0

位序列：   1        0        1        1        0
     ──┐  ┌──┐  ┌───┐  ┌──┐  ┌─────┐
       │  │  │  │   │  │  │  │     │
       └──┘  └──┘   └──┘  └──┘     └─────
        └── 从高到低=1
               └── 从低到高=0
```

考试判据：
```text
曼彻斯特：每个位中间一定有跳变（这是它的标志）。
         中间下降沿（高→低）= 1
         中间上升沿（低→高）= 0
         注意：每个位中间都有跳变！
```

口诀：
```text
"曼彻斯特：每个位都翻一次，高→低是1，低→高是0"
```

| 优点 | 缺点 |
|------|------|
| 自带时钟同步（每bit中间必跳变） | 带宽利用率只有50%（波特率=2×比特率） |

### 差分曼彻斯特（Differential Manchester）

```
规则：每个位的正中间必有一次跳变（同曼彻斯特）
      位边界处有跳变 → 0
      位边界处无跳变 → 1

位序列：   1        0        1        1        0
     ──┐     ┌──┐        ┌──┐     ┌─────┐
       │     │  │        │  │     │     │
       └─────┘  └────────┘  └─────┘     └──
          ↑      ↑        ↑      ↑
       无跳变=1 有跳变=0 无跳变=1 无跳变=1
```

考试判据：
```text
差分曼彻斯特：两个特征同时出现：
  1. 每个位中间一定跳变（和曼彻斯特一样）
  2. 位边界有跳变 = 0，无跳变 = 1（注意和NRZ-I相反！）
```

### 一张图区分四种编码

```text
同一个序列 1 0 1 1 0 0 1 的四种编码放在一起看：

NRZ:        ┌──┐  ┌─┐  ┌─┐        ┌──┐
       ────┘  └──┘ └──┘ └────────┘  └────

NRZ-I:   ┌┐ ┌──┐ ┌┐                  ┌┐
       ──┘└─┘  └─┘└──────────────────┘└──
         1  0  1  1  0  0           1

Manchester:┌┐ ┌┐ ┌┐ ┌┐ ┌┐ ┌┐ ┌┐ ┌┐
       ───┘└─┘└─┘└─┘└─┘└─┘└─┘└─┘└─┘└──
          1  0  1  1  0  0  1

Diff Man: ┌┐ ┌┐ ┌┐ ┌┐ ┌┐ ┌┐ ┌┐ ┌┐
       ───┘└─┘└─┘└─┘└─┘└─┘└─┘└─┘└─┘└──
          1  0  1  1  0  0  1
（每个bit中间都跳变 + 边界跳变=0,不跳=1）
```

**考试速判法：**

```text
步骤1：看每个位中间有没有跳变？
       没有 → NRZ 或 NRZ-I（下一步区分）
       有   → 曼彻斯特 或 差分曼彻斯特（下一步区分）

步骤2a（中间无跳变）：看相同连续位时波形是否水平直线？
       是 → NRZ
       否（只有1才跳变）→ NRZ-I

步骤2b（中间有跳变）：看1和0的编码是否固定（高→低永远是1）？
       是 → 曼彻斯特
       否（看位边界跳变）→ 差分曼彻斯特
```

## 1.3 10Base命名规则

10Base-T 这种命名其实是一个公式：

```text
<数据率><传输方式><介质类型/最大距离>

数据率：Mbps，如 10 = 10Mbps, 100 = 100Mbps, 1000 = 1000Mbps
传输方式：Base = 基带传输，Broad = 宽带传输
介质/距离：数字 = 最大段长(×100m)，字母 = 介质类型
```

| 名称 | 含义 |
|------|------|
| 10Base-5 | 10Mbps, 基带, 粗同轴电缆, 最大段长500m |
| 10Base-2 | 10Mbps, 基带, 细同轴电缆, 最大段长185m（实际约200m） |
| 10Base-T | 10Mbps, 基带, 双绞线(Twisted Pair) |
| 10Base-F | 10Mbps, 基带, 光纤(Fiber) |
| 100Base-T | 100Mbps, 基带, 双绞线（= 快速以太网 Fast Ethernet） |
| 100Base-FX | 100Mbps, 基带, 光纤 |
| 1000Base-T | 1000Mbps, 基带, 双绞线（= 千兆以太网 Gigabit Ethernet） |
| 10Broad-36 | 10Mbps, 宽带, 最大段长3600m |

考试常问：
```text
"10Base-5, 10Base-2, 10Base-T 中的 5/2/T 分别代表什么？"
→ 5表示最大段长500m（粗缆），2表示最大段长约200m/185m（细缆），T表示双绞线。
```

## 1.4 奈奎斯特与香农：计算题补强

### 核心区分（先判断场景）

```text
无噪声信道 → 用奈奎斯特 Nyquist
有噪声信道 → 用香农 Shannon
实际信道   → 取两者最小值
```

### 奈奎斯特公式

```text
C = 2W × log₂(V)

C：最大数据率 (bps)
W：信道带宽 (Hz)
V：信号电平数（每个码元可取几种离散值）

常见V值：二进制 V=2 → log₂(2)=1 → C=2W
        四进制 V=4 → log₂(4)=2 → C=4W
        八进制 V=8 → log₂(8)=3 → C=6W
```

例题：
```text
无噪声信道带宽4kHz，采用16种电平，求最大数据率。

C = 2 × 4000 × log₂(16)
  = 8000 × 4
  = 32000 bps = 32 kbps
```

### 香农公式

```text
C = W × log₂(1 + S/N)

C：最大数据率 (bps)
W：信道带宽 (Hz)
S/N：信噪比（比值，不是dB）

若给dB：SNR(dB) = 10 × log₁₀(S/N)
       → S/N = 10^(SNR/10)
```

常见dB换算速记：
```text
10dB  → S/N = 10
20dB  → S/N = 100
30dB  → S/N = 1000
40dB  → S/N = 10000
每增加10dB，S/N乘10。
```

例题：
```text
带宽3kHz，信噪比20dB，求最大数据率。

S/N = 10^(20/10) = 100
C = 3000 × log₂(1 + 100)
  ≈ 3000 × log₂(101)
  ≈ 3000 × 6.66
  ≈ 19980 bps ≈ 20 kbps

考试时若没有计算器，log₂(101)≈6.66，通常取近似值。
log₂(1+S/N) 常用近似：
  S/N=1    → 1
  S/N=3    → 2
  S/N=7    → 3
  S/N=15   → 4
  S/N=31   → 5
  S/N=63   → 6
  S/N=127  → 7
  S/N=255  → 8
  S/N=511  → 9
  S/N=1023 → 10  ← 30dB 最常考
```

### 联合考法

```text
某信道带宽4kHz，信噪比30dB，采用8种电平。
求理论最大数据率。

1. 奈奎斯特：C_nyq = 2×4000×log₂(8) = 8000×3 = 24000 bps
2. 香农：S/N=1000, C_sha = 4000×log₂(1001) ≈ 4000×10 = 40000 bps
3. 实际最大 = min(24000, 40000) = 24000 bps

结论：电平数限制了数据率。
```

### 误码率（Bit Error Rate, BER）

```text
误码率 = 传输出错的比特数 / 传输总比特数

数量级速记：
电话线：约 10⁻⁴ ~ 10⁻⁶
光纤：约 10⁻⁹ ~ 10⁻¹²
```

### 波特率 vs 比特率

```text
波特率（Baud rate）= 每秒传送的码元数
比特率（Bit rate） = 每秒传送的比特数

比特率 = 波特率 × log₂(V)

例：波特率=1000 Baud, V=16 → 比特率=1000×4=4000 bps
```

# 2. 数据链路层：GBN 与 SR 协议

> 现有资料覆盖了停止等待协议，但GBN和SR是期末加油站讲义P29-P30标记的补强区，也是2025 B卷信号。

## 2.1 为什么需要滑动窗口协议

停止等待的问题：
```text
发送一个帧 → 等ACK → 再发下一个
信道利用率 = 传输时延 / (传输时延 + RTT)

如果RTT远大于传输时延，大部分时间在等，信道几乎空转。
```

信道利用率公式：
```text
U = (发送时延) / (发送时延 + 2×传播时延)

例：发送一个帧需1ms，RTT=50ms
U = 1/51 ≈ 2%  ← 98%的时间在空等！
```

**滑动窗口的核心思想：不等待ACK，连续发多个帧。**
窗口越大 → 管道中可同时存在的未确认帧越多 → 利用率越高。

## 2.2 GBN（Go-Back-N，回退N帧）

### 基本原理

```text
发送方：维护一个大小为N的发送窗口。
        窗口内的帧可以连续发送，不用等ACK。
        一个计时器管整个窗口（不是每个帧单独计时！）。

接收方：只接受按序到达的帧。
        来了乱序的帧 → 直接丢弃，不缓存。
        只发"我期望的下一个帧号"这个ACK（累计确认）。
```

> **ACK编号约定**：本资料中 ACK N 表示"期望收到帧 N"，即"帧 N-1 及之前均已正确收到"。各教材可能有不同约定（ACK N = "已收到帧 N"），以老师课堂口径为准，但本文所有例题统一使用"期望帧 N"约定。

### 关键行为

```text
正常情况：
发送方连续发 0,1,2,3,4...
接收方逐个确认 ACK1, ACK2, ACK3...（期望下一帧）
窗口不断滑动。

丢帧情况（帧2丢失）：
发送方发出 0,1,2,3,4
接收方收到 0,1 → 期望帧2，回 ACK2
接收方收到 3 → 不是2 → 丢弃！继续回 ACK2
接收方收到 4 → 不是2 → 丢弃！继续回 ACK2
发送方收到多个 ACK2 → 知道帧2出问题了
计时器到期 → 重传从帧2开始的所有帧：2,3,4 ← "回退N帧"
```

**GBN 的命名来源：丢了第k帧，就要从第k帧开始重传所有已发未确认的帧（回退N帧）。**

### GBN 参数

```text
发送窗口大小：1 ≤ W_s ≤ 2^n - 1（n为序号位数）
接收窗口大小：W_r = 1（永远只接收期望的下一个帧）

典型例子：3位序号（0~7循环），发送窗口最大 = 7
```

### 优缺点

| 优点 | 缺点 |
|------|------|
| 实现简单（接收方不需要缓存） | 一个帧丢了，后面正确接收的全部白传 |
| 只需要一个计时器 | 高误码率时效率低 |

## 2.3 SR（Selective Repeat，选择重传）

### 基本原理

```text
发送方：每个帧独立计时（每个帧有自己的超时定时器）。

接收方：维护一个接收窗口。
        不管序号是否连续，只要落在接收窗口内就收，缓存起来。
        每个正确收到的帧都回独立ACK。
        乱序到达的帧先缓存，等缺失帧到齐后一起交给上层。
```

### 关键行为

```text
丢帧情况（帧2丢失，其他一样）：

发送方发出 0,1,2,3,4
接收方收到 0 → ACK0
接收方收到 1 → ACK1
接收方收到 3 → 缓存！发 ACK3 ← 关键区别
接收方收到 4 → 缓存！发 ACK4
接收方缓存着 [3,4]，等待帧2

发送方：
  收到 ACK0, ACK1 → 窗口滑到2
  帧2的计时器到期 → 只重传帧2！
  
接收方收到重传的帧2 → [2,3,4] 齐了 → 按序交付
```

**SR 的命名来源：只重传"选择性"地丢失的帧，不是全部重传。**

### SR 参数

```text
一般条件：W_s + W_r ≤ 2^n（发送窗口与接收窗口之和不超过序号空间）

当取等长窗口（W_s = W_r）时：
  发送窗口：W_s ≤ 2^(n-1)
  接收窗口：W_r ≤ 2^(n-1)

典型例子：3位序号(n=3) → 2^3=8个序号 → 等长窗口最大 = 4
```

### 优缺点

| 优点 | 缺点 |
|------|------|
| 只重传真正丢的帧，效率高 | 接收方需要缓存乱序帧，实现复杂 |
| 适合高误码率信道 | 每个帧要独立计时器 |

## 2.4 三协议对比表（考试必背）

| | 停止等待 | GBN | SR |
|---|---|---|---| 
| 发送窗口 | 1 | 1 ~ 2^n-1 | W_s+W_r≤2^n；等长时≤2^(n-1) |
| 接收窗口 | 1 | 1 | W_s+W_r≤2^n；等长时≤2^(n-1) |
| 计时器 | 1个 | 1个（管整个窗口） | 每个帧一个 |
| 乱序帧 | 不存在（一个个发） | 丢弃 | 缓存 |
| 确认方式 | 逐一确认 | 累计确认 | 逐一确认 |
| 重传范围 | 丢了只重传1帧 | 回退N帧全重传 | 只重传丢失的帧 |
| 复杂度 | 低 | 中 | 高 |
| 适用场景 | 低速/简单链路 | 低误码率 | 高误码率 |

## 2.5 计算题：最大窗口尺寸

```text
题目：帧序号用3位二进制表示（0~7循环），
     若采用GBN，发送窗口最大为多少？
     若采用SR，发送窗口最大为多少？

GBN：W_max = 2^3 - 1 = 7
SR：W_max = 2^(3-1) = 4

为什么SR窗口更小？
SR的接收窗口也 > 1。如果SR窗口设太大，新旧帧的序号会重叠，
接收方无法区分"这是重传的旧帧"还是"新一轮的新帧"。
```

## 2.6 考试简答模板

```text
GBN协议：
发送方可连续发送窗口内的帧；接收方只接收按序到达的帧，
对乱序帧直接丢弃并重发对上一个正确帧的ACK。
发送方使用单个计时器，超时后重传所有已发未确认的帧。

SR协议：
发送方可连续发送窗口内的帧；接收方缓存落在接收窗口内
的所有帧（包括乱序帧），对每个正确帧回独立ACK。
发送方为每个帧设独立计时器，超时只重传该帧。
相比GBN，SR避免了不必要的重传，但实现更复杂。
```

## 2.7 考试常见题型与易错点

### 题型一：丢帧追踪的坑点提示

丢帧追踪题，最容易扣分的地方不是不会写过程，而是 ACK 序号写错。

```text
题干：发送方发出 0,1,2,3,4，帧2丢失，其余正确到达。

❌ 常见错误答案：
  接收方收到帧3 → 由于帧2还没到，帧3是乱序，所以回 ACK2（期望帧2）
  ↑ 这是 GBN 的行为！GBN 丢弃乱序帧，回上一个正确的 ACK

✅ 正确答案（SR）：
  接收方收到帧3 → 缓存帧3，回 ACK3 ← 独立确认，收到几回几
  接收方收到帧4 → 缓存帧4，回 ACK4
```

```text
记忆口诀：GBN → "等前面的到了我才认" → 累计确认
          SR   → "来了我就收，收了就告诉你" → 逐一独立确认
```

**变体考法对比：**
| 协议 | 帧3乱序到达时接收方行为 |
|------|------------------------|
| GBN | 丢弃帧3，重发 ACK2 |
| SR | 缓存帧3，发 ACK3 |

---

### 题型二：场景选择与原因分析

**典型问法：**
> 在卫星通信（高延迟）和无线局域网（高误码率）中，分别适合使用什么协议？为什么？

**标准答案模板：**

```text
场景                  推荐协议          原因
─────────────────────────────────────────────────────
高延迟（卫星通信）     GBN 或 SR        停止等待协议"发一个等一个"，
                                        信道利用率极低，必须用流水线协议。

高误码率（无线网络）   SR                误码率高→丢帧频繁，
                                        GBN 会回退N帧重传浪费大量带宽，
                                        SR 只重传丢的帧，更高效。

低误码率（光纤骨干）   GBN               GBN 实现简单（接收方无需缓存），
                                        丢帧少时回退重传的浪费可忽略。
```

**万能公式：**
```text
"环境越差（高延迟/高误码）→ 越需要 SR"
"环境越好（低误码/低延迟）→ GBN 性价比更高"
```

### 题型三：核心公式速记三连（考前必背）

```text
┌────────────────────────────────────────────────┐
│  停止等待：W_s = 1,         W_r = 1              │
│                                                │
│  GBN：      W_s ≤ 2ⁿ - 1,   W_r = 1              │
│                                                │
│  SR：       W_s ≤ 2ⁿ⁻¹,     W_r ≤ 2ⁿ⁻¹            │
│                ↑ 这"减1"是 GBN 和 SR 最大区别     │
└────────────────────────────────────────────────┘
```

**考场速记口诀：**
- GBN：窗口 = 序号总数 **减 1**（`2ⁿ - 1`）
- SR：窗口 = 序号总数 **一半**（`2ⁿ⁻¹`）

---

### 易错点合集（这3个坑必考）

**易错1：GBN 的"累计确认"到底怎么理解？**

- 收到帧3的 ACK3 → 意味着"0~3 全部正确确认"
- 但如果帧2丢了，帧3到了 → GBN **不会**回 ACK3，而是丢弃帧3、**重发 ACK2**
- 累计 ≠ 只认最新的，而是"确认这个序号意味着前面的都确认了"

**易错2：SR 的窗口为什么不能超过一半？**

```text
n=3 的序号空间 0~7（共 8 个序号）

窗口设为 5（超过 4 的极限） → 出问题！

时间轴：
1. 发送方发 0,1,2,3,4（窗口内）
2. 接收方全收，窗口滑到 5,6,7,0,1
3. 帧 0 超时 → 发送方重传帧 0
4. 接收方收到帧 0 → 等等，这是重传的"旧帧0"还是"新窗口的帧0"？
   → 无法区分！ ← 这就是窗口不能超过一半的根本原因
```

**易错3：GBN 和 SR 的计时器数量混用**

| | GBN | SR |
|---|---|---|
| 计时器数量 | **1 个**（管整个窗口） | 每个帧各 1 个 |
| 超时后 | 重传**所有**已发未确认帧 | 只重传**该**帧 |

```
考试常设陷阱："SR 只需一个计时器"—— ❌ 错！
SR 每个帧独立计时，需要多个计时器。
与之对比，GBN 才是一个计时器管全部。
```

# 3. 应用层协议细节

> 现有资料覆盖了HTTP/HTTPS/DHCP，以下补充2025 B卷信号中出现的盲区。

## 3.1 DNS：递归查询 vs 迭代查询

现有资料给了DNS过程但没有区分两种查询方式，考试常问区别。

```text
递归查询（Recursive Query）：
  "你帮我查，查不到帮我去问别人，最后给我结果"
  本地DNS服务器 → 根DNS → .com TLD → 权威DNS
  每一跳都是"替我干活"。
  本地DNS服务器通常对客户端提供递归服务。

迭代查询（Iterative Query）：
  "你告诉我下一步该问谁，我自己去问"
  根DNS："我不知道baidu.com，但你可以问.com服务器，地址是xxx"
  本地DNS自己去问.com服务器。
  根DNS和TLD服务器通常只提供迭代查询。
```

考试答法：
```text
递归查询：DNS服务器替请求者完成全部查询，返回最终结果。
迭代查询：DNS服务器返回下一级DNS服务器的地址，由请求者继续查询。
主机向本地DNS服务器通常使用递归查询，
本地DNS服务器向根/顶级/权威的查询通常使用迭代查询。
```

## 3.2 FTP：双连接 + 主动/被动模式

```text
FTP需要两个TCP连接：

控制连接（端口21）：传输命令（登录、列目录、传文件指令）
                  整个会话期间保持连接。

数据连接（端口20 或 随机端口）：传输文件数据
                  每次文件传输新建一个连接，传完就关。
```

为什么需要两个连接：
```text
带外控制（Out-of-band Control）：控制信息与数据分离。
控制和数据走不同的TCP连接，互不干扰。
发送"停止传输"命令时，不需要和文件数据抢同一个连接。
```

### 主动模式 vs 被动模式（2025 B卷有匿名FTP考点）

```text
主动模式（PORT）：
  客户端发送PORT命令告诉服务器"我在xx端口等着"
  服务器从端口20主动连接客户端的这个端口
  问题：客户端在防火墙/NAT后面时，服务器连不过去

被动模式（PASV）：
  客户端发送PASV命令
  服务器回复"我在xx端口等着，你来找我"
  客户端主动连接服务器的这个随机端口
  适用：客户端在防火墙后面（现代Web浏览器都用这个）
```

考试常问：
```text
"匿名FTP" → 用户名 anonymous，密码通常是电子邮件地址或任意字符串
FTP默认端口：控制21，数据20
TFTP（简单文件传输协议）：基于UDP，端口69，用于无盘工作站启动等
```

## 3.3 SMTP、POP3、IMAP：邮件三协议

```text
发邮件 → SMTP（Simple Mail Transfer Protocol）
         端口25（明文）/ 端口465或587（加密）
         发送方→发送方邮件服务器→接收方邮件服务器
         使用TCP，基于"推"（Push）

收邮件 → POP3（Post Office Protocol v3）
         端口110
         把邮件从服务器"拉"到本地，默认拉完就删服务器上的
         适合单一设备收邮件

收邮件 → IMAP（Internet Message Access Protocol）
         端口143
         邮件一直保存在服务器上，多个设备可以同步查看
         适合手机+电脑多设备场景
```

对比表：
| | SMTP | POP3 | IMAP |
|---|---|---|---|
| 作用 | 发邮件 + 服务器间传输 | 收邮件 | 收邮件+在线管理 |
| 端口 | 25 | 110 | 143 |
| 邮件存储 | 不存 | 下载到本地 | 保留在服务器 |
| 多设备 | — | 不方便 | 方便 |
| 方向 | 推(Push) | 拉(Pull) | 拉(Pull) |

邮件发送全过程：
```text
发送方MUA → SMTP → 发送方邮件服务器 → SMTP → 接收方邮件服务器
                                                  ↓
                                          接收方MUA ← POP3/IMAP
```

## 3.4 MIME 与 base64（2025 B卷考点）

### 为什么需要 MIME

```text
SMTP最初只能传7位ASCII文本。不能传：
  - 非英文文字（中文、日文等）
  - 二进制文件（图片、音频、PDF）
  - 长行文本

MIME（Multipurpose Internet Mail Extensions）扩展了邮件格式，
让邮件可以携带多媒体内容。
```

### MIME 做了什么

```text
1. 新增邮件头字段：Content-Type, Content-Transfer-Encoding
2. 定义了内容类型（text/html, image/jpeg, application/pdf...）
3. 定义了编码方式，最常用的是 base64
```

### base64 编码原理

```text
目的：把任意二进制数据变成可打印的ASCII字符

步骤：
1. 把原始数据按24位一组（3字节）分组
2. 每组24位切成4个6位（2^6 = 64，所以叫base64）
3. 每个6位映射到一个可打印字符
   - 0~25  → A~Z
   - 26~51 → a~z
   - 52~61 → 0~9
   - 62 → +
   - 63 → /
4. 最后一组不足3字节时，补0并用 = 填充

原始3字节(24bit) → base64后4字节 → 体积膨胀约33%
```

考试简答：
```text
"简述base64编码的作用"
→ base64将任意二进制数据编码为可打印的ASCII字符，
使二进制数据能通过只支持文本传输的协议（如SMTP）进行传输。
编码后体积膨胀约1/3。
```

## 3.5 Cookie、缓存、CDN

现有资料只提了概念，以下补细节：

### Cookie
```text
目的：HTTP无状态，Cookie用于补状态。

工作：服务器在HTTP响应中加 Set-Cookie 头 → 浏览器存Cookie
     后续每次请求自动带 Cookie 头 → 服务器识别用户

组成：名称、值、域(Domain)、路径(Path)、过期时间(Expires/Max-Age)
```

### Web缓存（代理服务器）
```text
作用：在用户和服务器之间，缓存热门内容
条件GET：请求带 If-Modified-Since 头
         → 没改过 → 304 Not Modified（省带宽）
         → 改过   → 200 OK + 新内容
```

### CDN（内容分发网络）
```text
原理：把内容推到离用户最近的CDN节点
DNS重定向 → 将用户域名解析到最近的CDN节点IP
好处：降低延迟、减少源服务器压力
```

# 4. UDP 首部解析实战（2025 B卷综合应用题）

## 4.1 UDP 首部结构（8字节）

```text
 0      15 16      31
├────────┼────────┤
│ 源端口  │ 目的端口│  字节0-3
├────────┼────────┤
│ 总长度  │ 校验和  │  字节4-7
├────────┴────────┤
│      数据       │
└─────────────────┘

每个字段2字节(16位)，总共8字节。
```

## 4.2 从十六进制解析UDP首部

这是B卷出现过的综合应用题类型：

```text
题目：捕获到一个UDP报文，其首部十六进制为：
      CB84 0035 001C E5A7

      (1) 源端口和目的端口是多少？
      (2) UDP总长度是多少字节？
      (3) 数据部分有多少字节？

解析：

CB84 → 源端口 = 12×16³ + 11×16² + 8×16 + 4
             = 49152 + 2816 + 128 + 4 = 52100

0035 → 目的端口 = 0×16³ + 0×16² + 3×16 + 5
               = 53
               （端口53是DNS！所以这是DNS查询/响应）

001C → UDP总长度 = 0×16³ + 0×16² + 1×16 + 12
                = 28 字节

E5A7 → 校验和（验证传输正确性用）

数据长度 = 总长度 - 首部(8字节) = 28 - 8 = 20 字节
```

速算技巧：
```text
十六进制转十进制速记：
0000→0, 0001→1, 000A→10, 0010→16, 0014→20, 001C→28
0035→53(DNS), 0050→80(HTTP), 01BB→443(HTTPS)
0015→21(FTP控制), 0014→20(FTP数据), 006E→110(POP3)
```

## 4.3 TCP 首部实战要点

TCP 首部比 UDP 复杂，但考试主要关注序号和确认号的计算：

```text
首部中序号和确认号都是4字节(32位)字段。

场景：主机A向B发送数据，seq=100，数据长度=500字节

B的确认：
  如果B收到完整数据 → ack = 100 + 500 = 600
  （"我期望收到从600开始的数据"）

场景：TCP三次握手中：
  客户端发 SYN, seq=x
  服务器回 SYN+ACK, seq=y, ack=x+1
  客户端回 ACK, ack=y+1
```

# 5. IPv6 基础

## 5.1 为什么需要IPv6

```text
IPv4地址32位 → 约43亿个地址，已基本耗尽
IPv6地址128位 → 约3.4×10³⁸个地址
```

## 5.2 IPv6地址表示

```text
128位 = 8组 × 16位，每组用4位十六进制表示，用冒号分隔。

完整形式：2001:0db8:85a3:0000:0000:8a2e:0370:7334

简写规则：
1. 前导零可以省略：00ab → ab, 0000 → 0
   2001:db8:85a3:0:0:8a2e:370:7334
2. 连续的零可以用双冒号(::)替代（但只能用一次）
   2001:db8:85a3::8a2e:370:7334
```

## 5.3 IPv6 vs IPv4 主要区别

| | IPv4 | IPv6 |
|---|---|---|
| 地址长度 | 32位 | 128位 |
| 首部长度 | 可变(20~60B) | 固定40B |
| 分片 | 路由器可以分片 | 路由器不分片 |
| 校验和 | 有首部校验和 | 无（交给上层） |
| 选项 | 在首部中 | 用扩展首部 |
| ARP | 有 | 被NDP(邻居发现协议)替代 |
| 广播 | 有 | 无，用组播替代 |
| NAT | 广泛使用 | 原则上不需要 |

## 5.4 IPv6 分片行为（2025 B卷考点）

```text
考试必背：
IPv6中，路由器不再对数据报进行分片。
若数据报大于路径MTU，路由器直接丢弃，
并向源主机发送ICMPv6 "Packet Too Big"报文。
只有源主机可以进行分片（使用分片扩展首部）。

这和IPv4完全不同！IPv4路由器可以对数据报分片。
```

# 7. 一页速背

```text
编码识图：
  NRZ：高1低0，连续0/1波形不变
  NRZ-I：位边界有跳变=1，无跳变=0（只看跳变！）
  曼彻斯特：每bit中间必跳变，高→低=1，低→高=0
  差分曼彻斯特：每bit中间必跳变 + 边界有跳变=0，无跳变=1

10Base命名：速率-基带/宽带-介质。5=粗缆500m, 2=细缆185m, T=双绞线

奈奎斯特（无噪声）：C=2W·log₂(V)
香农（有噪声）：C=W·log₂(1+S/N)，30dB≈S/N=1000

GBN：发方窗口大，收方窗口=1，乱序丢弃，累计确认，一个计时器
SR：发方窗口≤2^(n-1)，收方缓存乱序帧，每个帧独立计时器

FTP：双连接，控制21+数据20/随机，主动(PORT)被动(PASV)
SMTP：发邮件，TCP 25，推(Push)
POP3：收邮件，TCP 110，拉到本地
IMAP：收邮件，TCP 143，保留在服务器

MIME：扩展邮件支持多媒体；base64：3字节→4字符，膨胀33%

UDP首部8字节：源端口(2) + 目的端口(2) + 长度(2) + 校验和(2)
            数据长度 = 总长度字段值 - 8

IPv6：128位，路由器不分片！MTU过大→丢弃+ICMPv6报告
```
