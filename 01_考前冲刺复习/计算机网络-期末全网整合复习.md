## 1. 网络层（重点）

### 1.1 IP地址与子网划分

#### 核心知识

- IPv4地址是32位，通常写成点分十进制。
- IP地址由网络号和主机号组成。
- 子网掩码或CIDR前缀决定网络号占多少位。
- 网络地址：主机位全0。
- 广播地址：主机位全1。
- 可用主机地址：网络地址+1 到 广播地址-1。

#### 计算规则

```text
主机位 h = 32 - 前缀长度
地址总数 = 2^h
可用主机数 = 2^h - 2
块大小 = 2^h = 256 - 掩码最后一个非255字节
网络地址 = IP地址 & 子网掩码
广播地址 = 网络地址 + 块大小 - 1
```

#### 子网速查表

| 前缀 | 子网掩码 | 块大小 | 可用主机数 |
|---|---|---:|---:|
| /24 | 255.255.255.0 | 256 | 254 |
| /25 | 255.255.255.128 | 128 | 126 |
| /26 | 255.255.255.192 | 64 | 62 |
| /27 | 255.255.255.224 | 32 | 30 |
| /28 | 255.255.255.240 | 16 | 14 |
| /29 | 255.255.255.248 | 8 | 6 |
| /30 | 255.255.255.252 | 4 | 2 |

#### 常见题型

```text
1. 给 IP/prefix，求网络地址、广播地址、可用范围。
2. 给一个 /24，要求等分成若干子网。
3. 给主机数量，反推需要多少主机位和前缀长度。
4. 给若干地址，判断是否在同一子网。
```

#### 易错点

```text
1. 可用主机数通常要减2：网络地址和广播地址不能分配给主机。
2. /31、/32属于特殊场景，普通期末题按常规减2处理。
3. 路由器接口也要占用IP地址。
4. 192.168.x.x 是私有地址，不会直接在公网路由。
```

### 1.2 CIDR

#### 前缀规则

- CIDR写法：`a.b.c.d/x`，`x`表示网络前缀长度。
- 前缀越长，地址块越小，匹配越精确。
- 路由查表遵循最长前缀匹配。

#### 地址块计算

```text
地址块大小 = 2^(32-x)
起始地址 = 主机位全0
结束地址 = 主机位全1
```

例：

```text
192.168.10.64/26
主机位 = 6
块大小 = 64
地址范围 = 192.168.10.64 ~ 192.168.10.127
可用范围 = 192.168.10.65 ~ 192.168.10.126
```

#### 最长前缀匹配

```text
若目的IP同时匹配：
192.168.0.0/16
192.168.10.0/24
192.168.10.64/26

选择 /26，因为前缀最长，匹配最具体。
```

#### 路由聚合

```text
1. 把多个网络地址写成二进制。
2. 找最长公共前缀。
3. 公共前缀之后全部置0。
4. 得到聚合路由。
```

例：

```text
192.168.0.0/24
192.168.1.0/24
192.168.2.0/24
192.168.3.0/24

第三字节：0,1,2,3 = 00000000 ~ 00000011
公共前缀为前22位
聚合结果：192.168.0.0/22
```

### 1.3 路由协议

| 协议 | 类型 | 算法/思想 | 使用范围 | 度量 | 高频考点 |
|---|---|---|---|---|---|
| RIP | IGP | 距离向量 | 自治系统内部小网络 | 跳数 | 15跳最大，16不可达，周期更新 |
| OSPF | IGP | 链路状态 | 自治系统内部较大网络 | 代价 | 泛洪链路状态，Dijkstra最短路径 |
| BGP | EGP | 路径向量 | 自治系统之间 | 路径/策略 | AS间路由，策略优先，不只看最短 |

#### RIP

```text
RIP = 距离向量协议。
距离 = 跳数。
最大有效跳数 = 15。
16 = 不可达。
收到邻居路由表后，新距离 = 邻居通告距离 + 1。
```

RIP更新规则：

```text
原来没有该网络 -> 加入。
原下一跳就是该邻居 -> 必须更新。
原下一跳不是该邻居，但新距离更短 -> 更新。
否则不变。
```

#### OSPF

```text
OSPF = 链路状态协议。
每台路由器维护拓扑数据库。
通过Dijkstra算法计算最短路径树。
适合大型内部网络。
```

#### BGP

```text
BGP = AS之间的路径向量协议。
核心不是单纯最短路径，而是根据AS路径和策略选路。
通常用于互联网骨干路由。
```

### 1.4 IP分片

#### 核心概念

- MTU：链路层一次能承载的最大帧数据长度。
- IPv4数据报超过下一跳MTU时可能分片。
- 每个分片都有自己的IP首部。
- 片偏移单位是8字节。
- MF=1表示后面还有分片；最后一片MF=0。

#### 分片计算步骤

```text
1. 原始数据长度 = IP总长度 - IP首部长度。
2. 每片最大数据长度 = floor((MTU - IP首部长度) / 8) * 8。
3. 分片数量 = 原始数据长度 / 每片最大数据长度，向上取整。
4. 每片总长度 = 该片数据长度 + IP首部长度。
5. 片偏移 = 该片数据起始字节 / 8。
6. MF：最后一片为0，其余为1。
```

#### 易错点

```text
1. 片偏移只看数据部分，不算IP首部。
2. 除最后一片外，每片数据长度必须是8字节整数倍。
3. 每个分片都要带IP首部。
```

## 2. 数据链路层

### 2.1 ARP

> 注意：ARP通常被放在网络层与链路层之间，考试常和IP/MAC、局域网一起考。

```text
作用：已知IP地址，查询同一局域网内目标主机的MAC地址。
请求：广播。
响应：单播。
范围：只在一个局域网内工作，路由器不转发ARP广播。
```

ARP过程：

```text
1. 主机A查ARP缓存。
2. 未命中，广播ARP请求：”谁是这个IP？”
3. 目标主机单播ARP响应，告诉自己的MAC地址。
4. 主机A写入ARP缓存。
```

> **跨网陷阱：** 目的主机不在同一局域网时，ARP不查远程主机的MAC，而是**查默认网关的MAC**。跨网通信时，帧头里的目的MAC是下一跳路由器接口的MAC，不是最终目的主机的MAC。

### 2.2 透明传输

```text
问题：数据部分可能出现和帧定界符相同的比特/字符，导致接收方误判边界。
目标：无论数据内容是什么，都能原样通过链路层。
实现：
  - 字节填充：数据中遇到特殊控制字符时，在前面加转义字符（ESC）。
  - 比特填充：每5个连续1后插入一个0，接收方再删掉（如HDLC/PPP）。
```

### 2.3 字节填充 vs 零比特填充

| 方法 | 使用场景 | 做法 |
|---|---|---|
| 字节填充 | 面向字节协议 | 遇到控制字符，在前面插入转义字符 |
| 零比特填充 | 面向比特协议 | 发送端遇到连续5个1后插入0，接收端删除该0 |

### 2.4 CSMA/CD

```text
全称：载波监听多路访问/冲突检测。
适用：传统共享式以太网。
核心：先听后发，边发边听，冲突则停止并退避重传。
争用期：2倍端到端传播时延，即2τ。
```

高频结论：

```text
MAC地址长度 = 48位。
以太网最小帧长 = 64字节。
以太网数据字段最小 = 46字节。
交换机工作在数据链路层。
交换机MAC表通过源MAC地址动态学习。
普通交换机不隔离广播域，路由器隔离广播域。
```

## 3. 运输层

### 3.1 TCP三次握手

```text
Client -> Server: SYN, seq=x
Server -> Client: SYN+ACK, seq=y, ack=x+1
Client -> Server: ACK, ack=y+1
```

为什么不是两次：

```text
防止历史失效SYN让服务器误以为要建立新连接；
三次握手能确认双方发送和接收能力正常。
```

### 3.2 四次挥手

```text
主动方 -> 被动方: FIN
被动方 -> 主动方: ACK
被动方 -> 主动方: FIN
主动方 -> 被动方: ACK
主动方进入TIME-WAIT，等待2MSL。
```

为什么通常四次：

```text
TCP是全双工通信。
一方发送FIN只表示自己不再发送，另一方可能还有数据要发。
因此ACK和FIN通常分开发送。
```

TIME-WAIT作用：

```text
1. 保证最后一个ACK能被对方收到。
2. 让旧连接残留报文在网络中自然消失。
```

### 3.3 流量控制

```text
目的：防止发送方把接收方缓存撑爆。
核心变量：rwnd，接收窗口。
接收方通过TCP首部窗口字段通告还能接收多少字节。
```

### 3.4 拥塞控制

```text
目的：防止发送方把网络中间路由器/链路撑爆。
核心变量：cwnd，拥塞窗口。
发送窗口 = min(rwnd, cwnd)。
```

| 阶段/事件 | 规则 |
|---|---|
| 慢开始 | cwnd从1开始，每RTT近似翻倍 |
| 拥塞避免 | cwnd每RTT线性+1 |
| 超时 | ssthresh=cwnd/2，cwnd=1 |
| 3个重复ACK | 快重传，ssthresh=cwnd/2，Reno不回到1 |

### 3.5 TCP vs UDP

| 对比 | TCP | UDP |
|---|---|---|
| 连接 | 面向连接 | 无连接 |
| 可靠性 | 可靠、有序 | 不可靠、不保证有序 |
| 数据边界 | 面向字节流 | 面向报文 |
| 首部 | 20~60字节 | 8字节 |
| 控制机制 | 流控、拥控、重传 | 基本没有 |
| 效率 | 开销大 | 开销小、效率高 |

学习通易错：

```text
TCP面向字节流，UDP面向报文。
端口号16位，范围0~65535。
停止等待协议主要缺点是信道利用率低。
```

## 4. 高频选择题总结

### ARP / DHCP

高频考点：

```text
ARP：IP -> MAC；请求广播，响应单播。
DHCP：自动分配IP、掩码、网关、DNS等配置。
DHCP过程：Discover -> Offer -> Request -> ACK，记作DORA。
DHCP常用UDP端口：服务器67，客户端68。
```

常见陷阱：

```text
ARP不是MAC -> IP。
ARP广播不能跨路由器。
DHCP客户端一开始可能没有IP，所以会使用广播。
```

### IP / CIDR

高频考点：

```text
IP地址 = 网络号 + 主机号。
CIDR前缀越长，地址块越小，匹配越精确。
最长前缀匹配优先于默认路由。
私有地址：10/8，172.16/12，192.168/16。
```

常见陷阱：

```text
127.0.0.1是回环地址。
255.255.255.255是受限广播。
网络地址和广播地址通常不能分配给主机。
```

### 路由协议

高频考点：

```text
RIP：距离向量，跳数，16不可达。
OSPF：链路状态，Dijkstra，内部网关协议。
BGP：路径向量，自治系统之间，外部网关协议。
```

常见陷阱：

```text
ICMP不是路由协议。
RIP不是按带宽/时延选路，只按跳数。
BGP不是单纯最短路径，而是策略驱动。
```

### TCP/IP

高频考点：

```text
TCP可靠：序号、确认、重传、流控、拥控。
UDP不可靠但开销小。
TCP首部源端口和目的端口均为16位。
HTTP/1.1和HTTP/2通常基于TCP；HTTP/3基于QUIC/UDP。
```

常见陷阱：

```text
TCP是字节流，不保留应用层消息边界。
UDP面向报文，保留消息边界。
流量控制看接收方，拥塞控制看网络。
```

### NAT / ICMP

高频考点：

```text
NAT：私有IP:端口 <-> 公网IP:端口。
NAT缓解IPv4地址不足。
ICMP用于差错报告和网络探测。
Ping使用ICMP请求/应答。
Traceroute利用TTL和ICMP超时报文。
```

常见陷阱：

```text
NAT不是加密技术。
ICMP不是传输层协议，也不是路由协议。
路由器丢弃TTL为0的数据报时可返回ICMP超时报文。
```

## 5. 大题模板（重点）

### ① IP子网划分模板

```text
题目：给 IP/prefix 或要求划分子网。

1. 写出前缀长度 x，主机位 h=32-x。
2. 地址总数=2^h，可用主机数=2^h-2。
3. 查掩码，求块大小。
4. 找IP落在哪个块：0, block, 2block, ...
5. 写：
   网络地址 = 块起点
   广播地址 = 块终点
   可用范围 = 网络地址+1 ~ 广播地址-1
```

### ② IP分片模板

```text
题目：给原IP总长度、IP首部长度、MTU。

1. 原数据长度 = 原IP总长度 - IP首部长度。
2. 每片最大数据 = floor((MTU-IP首部长度)/8)*8。
3. 依次切分数据。
4. 每片总长度 = 数据长度 + IP首部长度。
5. 片偏移 = 数据起始位置 / 8。
6. MF：最后一片0，其余1。
```

### ③ 路由表最长前缀匹配模板

```text
题目：给目的IP和路由表。

1. 将目的IP与各路由前缀比较。
2. 找所有能匹配的路由项。
3. 选择前缀最长的一项。
4. 若没有匹配项，走默认路由0.0.0.0/0。
5. 若连默认路由也没有，丢弃，可返回ICMP差错。
```

### ④ RIP更新模板

```text
收到邻居X的路由表：
新距离 = 邻居通告距离 + 1
下一跳 = X

更新规则：
1. 原来没有 -> 加入。
2. 原下一跳就是X -> 更新。
3. 原下一跳不是X，但新距离更短 -> 更新。
4. 其他情况 -> 不变。
5. 距离>=16 -> 不可达。
```

## 6. 一页速记总结

```text
【IP公式】
h = 32 - 前缀
地址总数 = 2^h
可用主机 = 2^h - 2
块大小 = 2^h = 256 - 掩码最后非255项
片偏移 = 数据起始字节 / 8

【子网速查】
/24 255.255.255.0   块256 可用254
/25 255.255.255.128 块128 可用126
/26 255.255.255.192 块64  可用62
/27 255.255.255.224 块32  可用30
/28 255.255.255.240 块16  可用14
/29 255.255.255.248 块8   可用6
/30 255.255.255.252 块4   可用2

【ARP/DHCP】
ARP：IP -> MAC，请求广播，响应单播。
DHCP：DORA = Discover, Offer, Request, ACK。

【RIP/OSPF/BGP】
RIP：距离向量，跳数，16不可达，内部小网络。
OSPF：链路状态，Dijkstra，内部大网络。
BGP：路径向量，AS之间，策略选路。

【TCP流程】
三次握手：SYN -> SYN+ACK -> ACK。
四次挥手：FIN -> ACK -> FIN -> ACK。
可靠传输：序号 + ACK + 重传 + 流控 + 拥控。
发送窗口 = min(rwnd, cwnd)。
慢开始翻倍，拥塞避免+1，超时回1，3重复ACK快重传。

【数据链路】
CRC：补0 -> 模2除 -> 余数拼接。
CSMA/CD：先听后发，边发边听，冲突退避。
MAC地址48位，交换机二层，MAC表动态学习。
```
