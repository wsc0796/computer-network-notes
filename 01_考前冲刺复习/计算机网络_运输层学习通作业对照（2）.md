# 3. 需要补强的两个易错点

## 3.1 TCP/UDP面向对象不要背反

考试陷阱：

```text
TCP是面向报文的，UDP是面向字节流的。（错）
```

正确说法：

```text
TCP是面向字节流的。
UDP是面向报文的。
```

为什么：

```text
TCP把应用数据看成连续字节流，不保留应用层消息边界。
UDP一次发送一个报文，接收方一次收一个完整UDP报文，保留消息边界。
```

一句话记：

```text
TCP像水管，按字节流动；UDP像快递，一件一件报文投递。
```

## 3.2 网络层服务类型 vs 运输层运行机制

作业判断题：

```text
网络层提供数据报或虚电路服务不影响运输层的运行机制，但提供不同的服务质量。
```

答案：

```text
对。
```

考试答法：

```text
运输层协议运行在端系统中，依靠端系统的TCP/UDP机制完成进程到进程通信；
网络层在下面提供主机到主机的通信服务，可能是数据报服务，也可能是虚电路服务。
网络层服务类型不会改变运输层协议本身的基本运行机制，
但会影响运输层感受到的丢包、时延、乱序、吞吐率等服务质量。
```

记忆：

```text
机制看运输层自己；质量受网络层影响。
```

# 4. 新增作业后的运输层优先级

## A级：老师作业直接命中

```text
1. 端口号：16位，区分应用进程。
2. 复用与分用：发送端复用，接收端分用。
3. UDP：不可靠、无连接、面向报文、首部8字节、效率高。
4. TCP：可靠、面向连接、面向字节流、有流控和拥塞控制。
5. 停止等待：缺点是信道利用率低。
6. 网络层服务类型影响服务质量，不改变运输层基本机制。
```

## B级：已有资料仍要保留

```text
1. 三次握手、四次挥手。
2. 发送窗口 = min(rwnd, cwnd)。
3. 慢开始、拥塞避免、快重传、快恢复。
4. TCP首部字段：源端口、目的端口、序号、确认号、窗口。
5. UDP首部8字节解析。
```

# 5. 8题速背版

```text
1. 应用进程通过端口与传输层交互。
2. 端口号长度是16位。
3. 传输层不可靠协议是UDP。
4. UDP通常效率高于TCP，因为无连接、首部小、控制少。
5. 停止等待协议主要缺点是信道利用率低。
6. UDP不提供可靠服务。
7. TCP面向字节流，UDP面向报文。
8. 网络层服务类型不改变运输层机制，但影响服务质量。
```
