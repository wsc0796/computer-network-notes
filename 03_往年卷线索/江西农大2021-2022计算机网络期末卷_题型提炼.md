# 5. 简答题模板

## 5.1 TCP和UDP区别

```text
TCP面向连接，可靠传输，有序到达，有流量控制和拥塞控制，开销较大；
UDP无连接，不保证可靠和有序，开销小、实时性好。
TCP适用于Web、文件传输、邮件等；UDP适用于DNS、视频、语音、实时应用等。
```

## 5.2 IP地址和MAC地址区别

```text
IP地址是网络层逻辑地址，用于跨网络寻址和路由选择；
MAC地址是数据链路层物理地址，用于同一链路内帧的交付。
IP地址可随网络变化，MAC地址通常固化在网卡中。
```

## 5.3 三次握手和四次挥手

```text
三次握手：客户端SYN，服务器SYN+ACK，客户端ACK，建立TCP连接。
作用：确认双方收发能力正常，协商初始序号。

四次挥手：一方FIN，对方ACK；对方也准备关闭时发送FIN，原一方ACK。
作用：可靠释放双向连接，保证剩余数据传输完毕。
```

# 6. 综合题答题框架

## 6.1 小型企业网络拓扑

答题结构：

```text
1. 拓扑：路由器连接外网，路由器下接核心交换机，交换机再接各部门主机。
2. 子网：至少划分两个子网，例如办公网和服务器网。
3. 地址：给出每个子网的网段、掩码、网关。
4. 转发：同子网通过交换机通信，跨子网经路由器转发。
5. 管理：可用VLAN隔离不同部门，提高安全性和管理性。
```

## 6.2 Web服务器防火墙配置

答题结构：

```text
1. 只开放必要端口：HTTP 80、HTTPS 443。
2. 关闭不必要端口，默认拒绝外部未授权访问。
3. 若服务器在内网，通过端口转发把公网80/443映射到内网服务器。
4. 配置访问控制列表，限制管理端口只允许管理员IP访问。
5. 启用日志、入侵检测或限速策略，降低攻击风险。
```
