# 5. 必须会的标准答题模板

## 5.1 IPv6数据报过大

```text
IPv6中路由器不对数据报分片。
若数据报过大而无法从输出链路转发，路由器丢弃该数据报，
并可向源主机发送ICMPv6 Packet Too Big报文。
```

## 5.2 停止等待协议死锁

```text
帧或确认帧丢失时，发送方可能一直等待。
解决方法是设置超时计时器，超时后重传。
```

## 5.3 RIP计时器

```text
更新计时器：约30秒，周期性发送路由更新。
失效计时器：约180秒，长时间未收到更新则认为路由不可达。
垃圾收集/删除计时器：约240秒，删除失效路由相关信息。
```

## 5.4 NAT作用

```text
NAT把内部私有IP地址和端口转换为公网IP地址和端口，
使私有地址主机能够访问Internet，并缓解IPv4地址不足。
```

## 5.5 UDP首部解析

```text
UDP首部共8字节：
第1-2字节源端口，第3-4字节目的端口，
第5-6字节总长度，第7-8字节校验和。
数据长度 = UDP总长度 - 8字节。
```
