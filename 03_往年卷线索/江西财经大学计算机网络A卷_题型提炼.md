# 3. 简答题标准答案

## 3.1 什么是路由？路由器的主要功能是什么？

路由是指在网络中为分组从源主机到目的主机选择传输路径的过程。

路由器的主要功能包括：

```text
1. 根据目的IP地址查找路由表。
2. 选择合适的下一跳和输出接口。
3. 转发IP分组。
4. 连接不同网络。
5. 隔离广播域。
6. 必要时丢弃无法转发的分组，并返回ICMP差错报文。
```

## 3.2 IP地址与物理地址的区别，为什么要使用两种地址？

IP地址是网络层逻辑地址，用于在不同网络之间标识主机或接口，支持路由选择和跨网络通信。

物理地址通常指MAC地址，是数据链路层地址，用于同一局域网中标识网卡，支持相邻节点之间的帧传输。

使用两种地址的原因：

```text
IP地址用于全局逻辑寻址和路由选择；
MAC地址用于局域网内实际帧转发。
网络层和数据链路层职责不同，因此需要两种地址配合。
```

一句话：

```text
IP负责跨网络找到主机，MAC负责在当前链路上找到下一跳。
```

## 3.3 简述CSMA/CD工作过程

CSMA/CD即载波监听多路访问/冲突检测，主要用于共享式以太网。

工作过程：

```text
1. 发送前先监听信道。
2. 如果信道空闲，则立即发送。
3. 如果信道忙，则等待。
4. 发送过程中继续检测冲突。
5. 若检测到冲突，立即停止发送并发送干扰信号。
6. 等待随机退避时间后重新发送。
```

口诀：

```text
先听后发，边发边听，冲突停止，随机重发。
```

## 3.4 无连接服务和面向连接服务的主要区别

| 对比 | 面向连接服务 | 无连接服务 |
|---|---|---|
| 是否建连 | 传输前建立连接 | 不建立连接 |
| 过程 | 建立、传输、释放 | 每个分组独立发送 |
| 可靠性 | 通常更容易实现可靠、有序 | 通常不保证可靠、有序 |
| 资源 | 可能维护连接状态 | 不维护连接状态 |
| 例子 | TCP | UDP、IP |

简答模板：

```text
面向连接服务在数据传输前要先建立连接，传输过程中保持连接状态，传输结束后释放连接，适合要求可靠、有序传输的场景。无连接服务不建立连接，每个分组独立携带目的地址并独立转发，开销小、速度快，但通常不保证可靠性和顺序。
```

# 4. 计算题专项

## 4.1 IP地址与子网掩码

### 题1

```text
IP地址 = 197.99.222.139
子网掩码 = 255.255.255.240
```

解：

```text
197开头，属于C类地址。
255.255.255.240 = /28
块大小 = 256 - 240 = 16
139 落在 128~143。
```

答案：

```text
类别：C类
网络地址：197.99.222.128
广播地址：197.99.222.143
可用主机范围：197.99.222.129 ~ 197.99.222.142
主机号：11，即139-128=11
可用主机数：2^(32-28)-2 = 14
```

### 题2

```text
IP地址 = 119.28.188.99
子网掩码 = 255.255.255.0
```

解：

```text
119开头，属于A类地址。
给定掩码为/24。
```

答案：

```text
类别：A类
网络地址：119.28.188.0
广播地址：119.28.188.255
可用主机范围：119.28.188.1 ~ 119.28.188.254
主机号：0.0.0.99 或该子网内主机号99
可用主机数：254
```

## 4.2 超网聚合

题目：

```text
设有6个C类网络：
198.48.1.0
198.48.2.0
198.48.3.0
198.48.4.0
198.48.5.0
198.48.6.0
要求构造为一个超网。
```

说明：

```text
PDF中第一项看起来可能像192.48.1.0，但结合“6个C类网络构造超网”和后续连续地址，按198.48.1.0~198.48.6.0理解更合理。
```

若必须构造为一个超网，需要用一个能覆盖1~6的2的幂边界地址块。

最小单个聚合块可用：

```text
198.48.0.0/21
子网掩码：255.255.248.0
地址范围：198.48.0.0 ~ 198.48.7.255
```

注意：

```text
这个超网会额外覆盖198.48.0.0和198.48.7.0两个/24网络。
如果要求精确覆盖，则不能只用一个CIDR块，需要拆成多个聚合块。
```

## 4.3 分组交换与HDLC封装开销

题目：

```text
网络层分组长度为64字节，其中分组头3字节。
运输层报文长度为549字节。
分组在数据链路层传输时嵌入HDLC信息帧的信息字段。
问：
1. 应分为几个分组？
2. 若每个信息帧独立发送，不计连接和释放开销，总比特流多少位？
```

解：

网络层每个分组可承载运输层数据：

```text
64 - 3 = 61字节
```

运输层报文：

```text
549 / 61 = 9
```

所以应分为9个分组。

若按常见HDLC信息帧开销计算：

```text
标志 1B + 地址 1B + 控制 1B + FCS 2B + 标志 1B = 6B
每帧总长度 = 64 + 6 = 70B
总比特流 = 9 × 70 × 8 = 5040 bit
```

答案：

```text
1. 分为9个分组。
2. 总比特流为5040 bit。
```

# 6. 外校卷新增速背

```text
CSMA/CD：载波监听多路访问/冲突检测。
WAN：覆盖范围较大的网络，跨城市、地区或国家。
ATM：异步传输模式，使用固定长度信元。
集线器：物理层设备，对信号进行再生和广播转发。
拓扑：网络结点和链路的连接结构。
帧中继：主要用于广域网互连。
10BaseT：双绞线以太网。
ping：测试网络连通性。
ARP：IP地址 -> MAC地址。
面向连接：建立、传输、释放。
```
